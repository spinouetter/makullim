# 6. 설정(Settings)

앱 전반의 표시 방식과 데이터 관리를 다룹니다.

## 컬러 테마

어두운 테마 16종 + 밝은 테마("빌리 엘리어트" 그룹 라벨) 10종, 총 26종 중에서 고를 수 있습니다. 버튼을 누르면 즉시 전체 화면 색이 바뀌고 저장됩니다. 시스템 다크/라이트 모드에 따라 자동으로 테마가 바뀌지는 않습니다(사용자가 고른 테마가 고정 적용).

시트맵(좌석 지도) 색만 따로 커스터마이즈하는 "시트맵 색상" 섹션도 별도로 있습니다(등급색·관극 히트맵 색 프리셋, 기본 OFF — 꺼두면 테마 색을 그대로 씁니다).

## 스케줄 컬럼 표시

순번·좌석·티켓·가격·기타 비용·메모·배역·대결 컬럼을 각각 켜고 끕니다. 스케줄 탭 헤더 드롭다운에서 숨긴 컬럼도 전부 여기서 다시 켤 수 있습니다(대결 컬럼은 이 공연에 "대결" 데이터가 정의된 경우에만 이 섹션에 나타납니다).

> 배역 토글은 예외가 하나 있습니다: 막공 표시의 **페어막·주역 페어막으로 선택된 배역**은 여기서 꺼도 스케줄 표에는 항상 표시됩니다(의도된 동작). 이때 토글 버튼 자체는 꺼진 모습으로 보일 수 있지만, 실제 스케줄 표 상태와는 무관합니다.

## 개발 중인 기능 보기

두 개의 독립된 스위치입니다(둘 다 기본 OFF, 이름은 같은 섹션 안에 있지만 서로 무관합니다).

- **Finale 보기**: Finale 탭을 나타나게 합니다.
- **취소 수수료 · Tickets 탭**: Tickets 탭, 티켓 팝오버의 시계(취소료 계산) 아이콘, 취소 시 수수료 자동 입력을 함께 켭니다.

## 다중 티켓 모드

켜면 스케줄 좌석칸에 항상 "+" 버튼이 보여 바로 다중 티켓 관리창을 열 수 있습니다. 꺼져 있으면(기본) 티켓 팝오버 안의 "중복 티켓 관리" 버튼으로만 진입합니다. 어느 쪽이든 맨 위 티켓만 스케줄·통계·좌석 지도에 반영된다는 점은 같습니다.

## 실험 기능 — 세로 스크롤 관성 개선

"세로 스크롤 잠금"이 켜져 있을 때만 의미가 있는 하위 옵션입니다. 기본(클래식) 잠금은 세로 스크롤의 자연스러운 관성(던지듯 스크롤하는 느낌)이 사라지는 부작용이 있는데, 이 실험 옵션을 켜면 그 관성을 최대한 살리는 방식으로 바뀝니다(완벽한 축 고정은 아닐 수 있음).

## 데이터 내보내기 / 가져오기

- **내보내기**: 화면 설정(테마·컬럼 표시·필터·막공 설정 등)과 좌석·티켓·메모·이력을 모두 담은 JSON 파일을 저장합니다.
- **가져오기**: 그 파일을 불러와 지금 이 브라우저 내용을 전부 덮어씁니다.
- **구 버전 방식**(레거시): 좌석·티켓 정보만 담는 예전 형식. 텍스트 붙여넣기도 지원합니다.

## Dropbox 백업

Dropbox 계정에 연결해 수동으로 백업·복원할 수 있습니다. **자동 동기화가 아니라 "수동 백업 → 수동 복원"** 방식이며, 여러 기기에서 각각 편집한 뒤 서로 다른 시점에 백업/복원하면 나중에 복원한 쪽이 이전 데이터를 덮어씁니다(자동 병합 없음). 복원 시 "설정 덮어쓰기"를 체크하지 않으면 좌석·티켓·메모만 덮어쓰고 화면 설정은 그대로 둡니다.

## 여러 공연 전환

등록된 공연이 2개 이상이면 "다른 공연" 섹션이 나타나 목록에서 고를 수 있습니다. 공연이 하나뿐이면 이 섹션은 보이지 않습니다. 공연별로 좌석·메모는 따로 저장됩니다.

## 초기화

- **설정 초기화**: 화면 설정(테마·필터·통계 구성 등)만 원래대로. 좌석·티켓·메모는 남습니다.
- **좌석 데이터 삭제**: 좌석·티켓·메모만 지웁니다. 설정은 남습니다.

## 기타

탭이 많아져(Finale·Tickets 둘 다 켜서 6개) 좁은 화면에 다 안 들어가면 Settings 탭이 자동으로 톱니바퀴(⚙) 아이콘으로 줄어듭니다.
