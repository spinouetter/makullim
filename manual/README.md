# 막울림(makullim) 사용 설명서

이 폴더는 막울림을 처음 쓰는 사람이 화면을 보면서 어떤 기능이 있고 어떻게 쓰는지 확인할 수 있도록 정리한 매뉴얼입니다. 요청 문서(`requests/`), 코드 주석, `HISTORY.md`(작업 히스토리)를 근거로 **현재 배포된 코드가 실제로 하는 동작**을 기준으로 썼습니다. 요청 문서 중에는 나중에 다른 요청으로 재설계되어 지금은 다른 동작을 하는 것도 있는데(예: 막공 표시는 0046→0051→0058→0075→0076→0077 순으로 여러 번 개편됨), 이 매뉴얼은 항상 **최신(현재) 동작**을 기준으로 씁니다.

## 막울림이 무엇인가요

뮤지컬 등 공연을 반복 관람하는 사람을 위한 **관극 기록·통계 트래커**입니다. 서버 없는 정적 웹앱으로, 내 좌석·티켓·메모 같은 개인 데이터는 이 기기의 브라우저(localStorage)에만 저장됩니다(선택적으로 Dropbox에 백업 가능). 여러 공연(쇼)을 지원하며, 접속한 주소(URL)에 따라 어떤 공연 데이터를 보여줄지 결정됩니다.

## 화면 구성 — 6개 탭

| 탭 | 기본 표시 | 하는 일 |
|---|---|---|
| **Schedule**(스케줄) | 항상 표시 | 전체 회차 표. 좌석·티켓 입력, 캐스팅 열람, 막공 표시 등 핵심 기능이 모여 있는 메인 화면 |
| **Tickets**(예매 티켓) | Settings에서 켜야 보임 | 보유 티켓을 취소 데드라인·예매일·공연일 순으로 정리해 보여주는 대시보드 + 취소료 계산기 |
| **Statistics**(통계) | 항상 표시 | 배역별·배역 페어별 출연 횟수, 대결 통계, 취소·변경 통계, 월별/요일별/티켓별 통계 등 |
| **Seat map**(좌석 지도) | 항상 표시 | 실제 좌석 도면 위에 관극 횟수 히트맵을 겹쳐 보여줌 |
| **Finale** | Settings에서 켜야 보임 | 막공(마지막 공연) 관극 인증용 캐스트보드 이미지 생성·다운로드 |
| **Settings**(설정) | 항상 표시(⚙ 아이콘일 수 있음) | 테마, 컬럼 표시, 개발 중 기능, 데이터 백업/내보내기 등 |

탭이 6개 다 켜지면 좁은 화면에서는 자동으로 Settings가 톱니바퀴(⚙) 아이콘으로 줄어듭니다.

## 매뉴얼 목차

1. [스케줄](01-스케줄.md) — 표 구조, 캐스트 보기, 좌석·티켓 입력, 막공 표시, 필터
2. [티켓(Tickets 탭)](02-티켓.md) — 티켓 목록, 취소료 계산기, 다중 티켓 관리, 예매처
3. [통계](03-통계.md) — 배역별/페어/대결/취소변경/기타 통계
4. [좌석 지도](04-좌석지도.md) — 관극 히트맵, 필터, 확대/미니맵
5. [Finale](05-피날레.md) — 캐스트보드 인증 이미지
6. [설정](06-설정.md) — 테마, 컬럼 표시, 데이터 백업/내보내기, 공연 전환

## 알아두면 좋은 공통 개념

- **좌석 = 티켓의 기준**: 스케줄의 좌석 입력칸에 좌석번호(`층-열-번`, 예 `1-7-23`)를 넣으면 그 회차의 "내 티켓"이 됩니다. 좌석이 있어야 가격·통계·좌석 지도·막공 계산 등 대부분 기능이 그 회차를 인식합니다.
- **숨김(0064·0078)**: 티켓 정보는 남긴 채 "이제 이 회차는 안 볼 것"이라고 표시하는 기능. Tickets 탭에는 배지로만 표시되고, 통계·좌석 지도·내 막공에서는 완전히 제외됩니다. 자세한 내용은 [스케줄](01-스케줄.md#숨김취소양도이력) 참고.
- **개발 중인 기능**: Settings에서 두 개의 독립된 스위치로 켜고 끕니다 — "Finale 보기"(Finale 탭)와 "취소 수수료 · Tickets 탭"(Tickets 탭 + 취소료 계산기). 둘 다 기본 OFF.
- **데이터 저장 위치**: 좌석·티켓·메모·설정은 이 브라우저의 localStorage에 공연 id별로 저장됩니다. 기기를 바꾸거나 브라우저를 초기화하면 사라지므로, Settings의 내보내기 또는 Dropbox 백업을 권장합니다.
- **여러 공연**: 주소의 슬러그(`/공연이름/`)나 `?show=` 파라미터로 공연을 구분합니다. 등록된 공연이 2개 이상이면 Settings에 공연 전환 UI가 나타납니다.

## 이 매뉴얼과 함께 보면 좋은 문서

- `requests/README.md`, 각 `requests/NNNN.md` — 기능이 어떤 요청으로 만들어졌는지 이력
- `HISTORY.md` — 완료된 작업 한 줄 요약(최신이 위)
- `KNOWN_ISSUES.md` — 조사 중 발견한 불일치·버그 후보 포함, 알려진 이슈 목록
- `CLAUDE.md` — 저장소 구조와 코딩 규칙(개발자용)
