# 막올림 (makollim)

뮤지컬을 자주 보는 사람을 위한 **개인 관극 기록 트래커**입니다. *막올림* — 막이 오르는 그 순간들을 모아 기록합니다. 회차별 캐스팅을 한눈에 보고,
내가 어떤 회차를 어떤 좌석에서 봤는지 기록하고, 좌석맵 위에서 관극 횟수를 시각적으로 확인하고,
배우/배역별 출연 통계와 티켓 가격까지 관리할 수 있습니다. 외부 서버 없이 브라우저에서만 동작하며,
입력한 기록은 브라우저(localStorage)에 자동 저장됩니다.

**한 서버(도메인)에서 여러 공연을 지원합니다.** 공연 데이터는 `shows/<공연id>/` 폴더 단위로 분리되어 있고,
공연마다 `https://도메인/<slug>/` 형태의 짧은 주소가 생깁니다(루트 `/` 접속은 기본 공연으로 이동).
예제 데이터는 뮤지컬 **빌리 엘리어트**(블루스퀘어 우리은행홀, slug `billy`)입니다.
새 공연 추가 방법은 아래 [새 공연 추가하기](#새-공연-추가하기)를 참고하세요.

## 주요 기능

- **Schedule** — 회차별 캐스팅 표. 배역/티켓/가격 컬럼을 숨기거나, 배역별 배우로 필터링, 회차별 좌석·메모·티켓 입력.
- **Statistics** — 배역별·조합별 출연 횟수 통계 (최초/공연시작/변경 반영/비중 반영 모드 지원).
- **Seat map** — 실제 극장 좌석 배치 위에 내가 관극한 좌석을 횟수에 따라 강조 표시. 층 토글·확대/축소·미니맵.
- **Settings** — 설정/좌석 기록 JSON 내보내기·가져오기, 저장 초기화.

## 실행 방법

데이터를 `shows/` 폴더에서 `fetch`로 불러오기 때문에 **파일을 더블클릭(`file://`)하면 동작하지 않습니다.**
반드시 **로컬 웹서버**로 열어야 합니다. 로컬에서는 루트(`/`)가 기본 공연을 바로 렌더하며(slug 리다이렉트 없음),
`/?show=<공연id 또는 slug>` 로 다른 공연을 볼 수 있습니다.

가장 간단한 방법:

- **VS Code** — [Live Server](https://marketplace.visualstudio.com/items?itemName=ritwickdey.LiveServer) 확장 설치 후 `index.html`에서 우클릭 → *Open with Live Server*
- **Python** — 폴더에서 `python -m http.server` 실행 후 브라우저로 `http://localhost:8000/` 접속(루트로 들어가면 `index.html`이 자동으로 열립니다)
- **Node** — `npx serve` 후 안내된 주소로 접속

## 프로젝트 구조

```
tracker/
├─ index.html   # 화면 마크업 (CSS/JS 링크만) — 배포 시 공연별 /<slug>/index.html 스텁의 원본
├─ css/styles.css                   # 스타일
├─ js/app.js                        # 전체 로직 (공연 해석 → 데이터 로드 → 렌더)
├─ shows/                           # ← 공연 데이터 (공연 하나 = 정의 파일 + 폴더)
│  ├─ index.json                    #   레지스트리: { "default": <기본 공연id>, "shows": [id…] }
│  ├─ betm-skorea-2026.json         #   공연 정의: 제목·기간·극장 연결 + slug(URL용 짧은 이름)
│  └─ betm-skorea-2026/             #   공연 데이터 폴더 (id와 같은 이름)
│     ├─ casts.json                 #     배역 · 배우 명단
│     ├─ schedule.json              #     회차 스케줄 + 회차별 캐스팅
│     ├─ grades.json                #     좌석 등급 · 가격
│     ├─ matches.json               #     대결(승부) 정의 (선택)
│     ├─ finale-boards.json         #     Finale 보드 레지스트리 (+ finale-boards/ 정의)
│     └─ images/                    #     배우 사진 · Finale 보드 배경 SVG
├─ data/holidays.json               # ← 공용: 공휴일(날짜→이름) — 스케줄 날짜 색상용
├─ theatres/                        # ← 공용: 극장(좌석 배치) 파일들 — 여러 공연장을 한 곳에서 관리
│  └─ bluesquare_wooribank_hall.json
├─ README.md
└─ LICENSE
```

### 접속 경로 (여러 공연)

- **`/<slug>/`** (예: `/billy/`) — 공연별 고정 주소. GitHub Pages 배포 시 GitHub Action이
  `index.html`을 복사해 `<slug>/index.html` 스텁을 만들고 공연 id를 주입합니다(저장소에는 커밋되지 않음).
- **`/`** (루트) — `shows/index.json`의 `default` 공연으로 이동합니다(배포 도메인에서는 그 공연의
  `/<slug>/`로 리다이렉트, 로컬 개발에서는 리다이렉트 없이 바로 렌더).
- `slug`는 URL용 짧은 이름일 뿐이라 언제든 바꿔도 되지만, **공연 `id`는 localStorage 키에 쓰이므로
  바꾸면 사용자 기록이 분리됩니다 — 한 번 정하면 바꾸지 마세요.**
- 공연 정의·보드 파일 안의 **상대경로는 그 공연 폴더(`shows/<id>/`) 기준**이고,
  `/`로 시작하면 사이트 루트 기준입니다(공용 자원 `/theatres/…`, `/fonts/…`).

> **극장은 공연과 분리되어 있습니다.** 어떤 극장을 쓸지는 공연 정의(`shows/<id>.json`)의 `theatre` 항목이 가리키는 파일로 결정됩니다. 그래서 `theatres/`에 여러 공연장 파일을 두고, 공연마다 `theatre`만 바꿔 연결하면 한 레포에서 여러 공연장을 관리할 수 있습니다.

> 입력한 좌석·메모·티켓·필터 등 **사용자 기록은 데이터 파일이 아니라 브라우저 localStorage**(키 `makollim:state:v1:<공연id>`)에 저장됩니다. `shows/`·`theatres/`는 "공연의 기본 정의"만 담습니다.

## 새 공연 추가하기

새 공연 하나 = **공연 정의 파일(`shows/<id>.json`) + 데이터 폴더(`shows/<id>/`) + 레지스트리 한 줄**입니다.
모든 파일은 **UTF-8 JSON**입니다. 아래 예시는 가상의 공연 "오페라의 유령"(id `potm-skorea-2026`)을 추가하는 경우입니다.

먼저 `shows/index.json`(레지스트리)에 공연 id를 등록합니다:

```json
{
  "version": 1,
  "default": "betm-skorea-2026",
  "shows": ["betm-skorea-2026", "potm-skorea-2026"]
}
```

- `default` — 루트(`/`) 접속 시 이동할 기본 공연의 **id**.

### 1. `shows/<id>.json` — 공연 정의 (제목 + slug + 극장 연결 + 공연 기간)

```json
{
  "id": "potm-skorea-2026",
  "slug": "phantom",
  "title": "오페라의 유령",
  "theatre": "/theatres/charlotte_theater.json",
  "startDate": "2025-04-12",
  "endDate": "2026-07-26",
  "runningtime": 160
}
```

- `id` — 공연 식별자(영소문자·숫자·`_`·`-`). **파일명·데이터 폴더명과 같아야 하며**, localStorage
  키(`makollim:state:v1:<id>`)와 Dropbox 백업 파일명에 쓰이므로 **한 번 정하면 바꾸지 않습니다.**
- `slug` — URL용 짧은 이름(`https://도메인/<slug>/`). 주소만 바뀌므로 언제든 변경 가능합니다.
- `title` — 공연 제목. 화면 상단 제목·브라우저 탭에는 **`막올림 · {title}`** 형태로 표시됩니다.
- `theatre` — 이 공연에서 사용할 **극장 파일의 경로**(`/`로 시작, 사이트 루트 기준). 아래 **5번 항목**에서 만든 극장 파일을 가리킵니다. `theatres/`에 여러 극장 파일을 두고 이 값만 바꾸면 다른 공연장으로 전환됩니다.
- `startDate` / `endDate` — 공연 시작일·종료일(`YYYY-MM-DD`). 헤더 우측에 **극장 이름과 함께 `시작일 ~ 종료일`** 로 표시됩니다(예: `블루스퀘어 우리은행홀 · 2025.04.12 ~ 2026.07.26`). 두 값이 모두 있어야 표시됩니다.
- `runningtime` — 공연 러닝타임(**분**). 공연 시작 시각 + 러닝타임이 지나면 **종료된 공연**으로 간주됩니다(스케줄 과거 표시, 통계의 관극/예매, 좌석맵 관극/예매에 사용). 없으면 기본 180분.
- `stats` — (선택) 통계 화면의 공연별 고정 설정. 없으면 모든 배역 통계가 삭제 가능하고 프리셋 조합도 없습니다.
  - `pinnedRoles` — 배역 통계에서 **삭제 버튼을 숨길**(항상 표시할) 배역명 배열. 예: `["빌리", "마이클"]`
  - `presetCombos` — 조합 통계에 **항상 표시**(삭제 불가, 순서 변경만 가능)할 프리셋 목록. 항목 필드:
    - `id` — 영문 식별자(저장된 순서·접힘 상태의 키로 쓰이므로 한 번 정하면 유지).
    - `roles` — 조합할 배역명 배열.
    - `label` — (선택) 블록 제목. 없으면 `"배역 × 배역"`.
    - `pointRoles` — (선택) `{ "배역명": 비중지점 }`. 지정한 배역은 출연 배우 전원이 아니라 **그 회차에서 해당 비중 지점(1~10)을 담당한 배우 1명**으로 좁혀 짝을 셉니다. 예: 드림 발레 페어 = `{ "id": "dreamballet", "label": "드림 발레 페어 (빌리 × 성인빌리)", "roles": ["빌리", "성인빌리"], "pointRoles": { "빌리": 7 } }` — 교체 공연에서 씬7(비중 7 지점)을 춘 빌리와 성인빌리의 짝 통계.

### 2. `shows/<id>/casts.json` — 배역과 배우 명단

배역(`role`)마다 출연 가능한 배우 목록을 적습니다. 배열 순서가 표/통계의 기본 순서가 됩니다.

```json
[
  {
    "role": "팬텀",
    "id": "phantom",
    "actors": [
      { "name": "조승우", "role": "cast" },
      { "name": "김주택", "role": "cast" }
    ]
  },
  {
    "role": "크리스틴",
    "id": "christine",
    "actors": [
      { "name": "손지수", "role": "cast" },
      { "name": "송은혜", "role": "alternative" }
    ]
  }
]
```

- `role`(배역명) — 스케줄/통계에서 컬럼·항목 이름으로 쓰이며, `schedule.json`의 캐스팅 키와 **반드시 일치**해야 합니다.
- `id` — (선택) **소문자·언더스코어(lowercase_snake)** 영문 식별자(예: `팬텀`→`"phantom"`, `Mrs. 윌킨슨`→`"mrs_wilkinson"`). Finale 정산판이 이 id로 배경 SVG 슬롯을 찾습니다(한글 `role`과 별개인 안정적 키). Finale 보드를 쓰지 않으면 없어도 됩니다.
- 배우의 `role`(역할 종류)에 따라 표에서 색/스타일이 달라집니다:
  `cast`(주연·기본), `alternative`, `swing`, `cover`, `standby`.
  `cast`가 아닌 배우는 실제 출연 기록이 있는 회차가 있을 때만 필터 목록에 노출됩니다.
- `fullName` — (선택) 배역의 실제 캐릭터 풀네임(예: `크리스틴` → `"크리스틴 다에"`, `라울` → `"라울 드 샤니"`). 표시에 필수는 아니고 참고·검증용입니다. 성이나 이름만 알면 아는 만큼만 적습니다.
- `shortName` — (선택) **스케줄 표 컬럼 헤더에서만** 대신 보여줄 짧은 이름(예: `role`이 `"오페라의 유령"`이어도 헤더엔 `"유령"`). 통계·좌석맵 등 다른 화면은 `role` 원래 이름을 씁니다. `role`은 `schedule.json`·`casting_by_date.json`의 키와 일치해야 하므로, 표에서만 줄여 보이고 싶을 때 사용합니다.

#### 그룹 역할 (`group: true`)

발레 앙상블·합창 앙상블처럼 **여러 사람이 한 덩어리로 묶이는 역할**은 `group: true`로 표시하고 구성원을 `members`에 적습니다.

- `group: true` — 이 역할의 `actors`가 개별 배우가 아니라 **팀/조**임을(또는 참조 전용 묶음임을) 나타냅니다.
- `actors[]` — 캐스팅에서 **고르는 단위**. 팀이 있으면 팀 목록(예: `A조`/`B조`), 팀이 없으면 생략합니다. **`actors`가 없는(비어 있는) 역할은 참조 전용**이라 스케줄 컬럼·통계·좌석맵에 표시되지 않습니다. (Finale 보드에서 팀별 섹션을 쓰면 팀 `actors`에도 `id`(lowercase_snake)를 답니다.)
- `members[]` — 그룹 내부 개인 구성원. 항목 형태는 그룹 성격에 따라 둘 중 하나:
  - **포지션형** — `role`(세부 포지션) + (선택)`fullName` + 배우: 팀 무관 고정이면 `name`, 팀별로 다르면 `byTeam`(`{ "A조": "김지연", "B조": "이서영" }`)
  - **기능태그형** — `name`(사람) + `roles`(기능 태그 배열, 예: `["앙상블"]`·`["스윙"]`, 겸직 시 `["스윙","댄스캡틴"]`)

```json
{
  "role": "발레 앙상블",
  "id": "ballet_ensemble",
  "group": true,
  "actors": [
    { "name": "A조", "id": "team_a", "role": "cast" },
    { "name": "B조", "id": "team_b", "role": "cast" }
  ],
  "members": [
    { "role": "메그", "fullName": "메그 지리", "name": "홍수민" },
    { "role": "제1무용수", "byTeam": { "A조": "김지연", "B조": "이서영" } }
  ]
},
{
  "role": "앙상블",
  "group": true,
  "members": [
    { "name": "박준호", "roles": ["앙상블"] },
    { "name": "최민지", "roles": ["스윙", "댄스캡틴"] }
  ]
}
```

> `fullName`·`shortName`·`group`·`members`는 모두 **선택 필드**라, 없던 공연 데이터도 그대로 동작합니다. `members`는 현재 화면 표시에는 쓰이지 않고 로스터 참조용 데이터입니다.

### 3. `shows/<id>/schedule.json` — 회차 스케줄과 캐스팅

공연 회차를 **시간 순서대로** 나열합니다. 각 회차의 `cast`는 `배역 → 배우 이름 배열` 형태입니다.

```json
[
  {
    "date": "2026-09-01",
    "time": "19:30",
    "seat": "",
    "note": "",
    "cast": { "팬텀": ["조승우"], "크리스틴": ["손지수"] }
  },
  {
    "date": "2026-09-02",
    "time": "14:00",
    "seat": "",
    "note": "",
    "cast": { "팬텀": ["김주택"], "크리스틴": ["송은혜"] }
  }
]
```

- `date`는 `YYYY-MM-DD`, `time`은 `HH:MM`. 공연 시작 3시간 뒤를 "종료"로 간주해 통계에 반영합니다.
- `seat`, `note`는 사용자가 앱에서 입력하는 값이라 보통 `""`로 비워 둡니다.
- `cast`의 키(배역명)는 `casts.json`의 `role`과 같아야 합니다.
- **공연 중 배역 교체(비중)** 를 표현하려면 이름 대신 `[이름, 비중]`을 쓸 수 있습니다(비중 합 10 기준).
  예) `"팬텀": [["조승우", 7], ["김주택", 3]]` → 7:3 비중으로 통계에 반영. 비중 `0`은 출연하지 않은 것으로 처리됩니다.

### 4. `shows/<id>/grades.json` — 좌석 등급과 가격

등급마다 속한 좌석 목록(`seatIds`)과 티켓 종류별 가격(`prices`)을 적습니다.

```json
[
  {
    "name": "VIP",
    "seatIds": ["1-1-8", "1-1-9", "1-1-10"],
    "prices": [
      { "name": "정가", "price": 190000, "discount": 0 },
      { "name": "청소년", "price": 150000, "discount": 21.05 }
    ]
  },
  {
    "name": "R",
    "seatIds": ["1-5-3", "1-5-4"],
    "prices": [
      { "name": "정가", "price": 160000, "discount": 0 }
    ]
  }
]
```

- `name`(등급명) — `VIP`, `R`, `S`, `A`, `B`는 좌석맵에서 색상 테마가 자동 적용됩니다. `보류`(회색)·`휠체어`(보라)도 전용 색이 있어 판매하지 않는 좌석·휠체어석을 표현할 수 있습니다(`prices: []`로 두면 티켓 선택지 없음). 그 밖의 이름도 동작하며 기본 회색으로 표시됩니다.
- `seatIds` — 이 등급에 속한 좌석 id 목록. 극장 파일(`theatres/…json`)의 좌석 id와 일치해야 합니다.
- `prices[]` — 한 등급에서 고를 수 있는 티켓 종류 목록. 각 항목은 세 필드로 구성됩니다:
  - `name` — 티켓 이름(예: `정가`, `청소년`). 첫 글자가 스케줄표 티켓 배지에 표시됩니다.
  - `price` — 실제 결제 금액(원). `가격` 컬럼에 `이 값 (+ 수수료 2,000원 옵션)`이 원화로 표시됩니다.
  - `discount` — 할인율(%). **정가는 `0`** 으로 둡니다. 티켓 배지와 선택 팝업에 표시되는 안내용 값으로, `price`와 독립적으로 적습니다.
  - `sids` — (선택) 이 티켓이 적용되는 **공연 ID 목록**. 없으면 모든 공연에 적용되고, 있으면 그 공연(예: 프리뷰 `["s2","s3",…,"s8"]`)에서만 티켓 선택지에 나타납니다. 공연 ID는 시간순 `s1, s2, …`입니다(좌석 JSON 내보내기의 키와 동일).
  - `times` — (선택) 이 티켓이 적용되는 **공연 시간 목록**(`HH:MM`). 없으면 모든 시간, 있으면 그 시간 공연에서만 선택지에 나타납니다(예: 마티네 `["14:30"]`). `sids`와 함께 쓰면 둘 다 만족해야 합니다.
  - `excludeSids` — (선택) 이 티켓이 **적용되지 않는 공연 ID 목록**. 여기에 든 공연에서는 선택지에서 빠집니다(예: 마지막 회차 `["s120","s121","s122","s123"]`에는 정가·초대권만 적용). `sids`(포함)와 반대 개념입니다.
  - `weekdayOnly` — (선택) `true`면 **주말(토·일)·공휴일 공연에는 적용되지 않습니다**(평일 전용, 예: 타임세일). 공휴일은 `data/holidays.json` 기준입니다.
  - `dateFrom` / `dateTo` — (선택) 적용 **기간**(`YYYY-MM-DD`, 포함). 이 기간의 공연에만 적용됩니다(예: KT멤버쉽 `dateFrom:"2026-05-01"`, `dateTo:"2026-06-05"`). 복합 제외(특정 요일·시간 등)는 `excludeSids`로 해당 공연을 빼서 표현합니다.
- 스케줄표에서 좌석을 입력하면 티켓 칸에 **등급 첫 글자 + 티켓 아이콘**이 뜨고, 이를 눌러 그 등급의 티켓 목록·수수료를 고른 뒤 저장하면 **등급 첫 글자 · 티켓 이름 첫 글자 · 할인율**이 표시됩니다.
  (수수료 금액은 `js/app.js`의 `TICKET_FEE` 상수입니다.)

### 5. `theatres/…` 극장 파일 — 좌석 배치 (가장 손이 많이 가는 파일)

극장의 모든 좌석을 좌표와 함께 나열합니다. 파일 이름은 자유이며(`theatres/charlotte_theater.json` 등),
공연 정의(`shows/<id>.json`)의 `theatre`가 이 경로(`/theatres/…`)를 가리키게 하면 됩니다. 한 레포에 여러 극장 파일을 두고 공연마다 골라 쓸 수 있습니다.

```json
{
  "theater": "샤롯데씨어터",
  "floorMeta": {
    "1": { "centerOriginX": -8.5, "centerWidth": 17, "centerMaxRow": 22 }
  },
  "seats": [
    { "id": "1-1-8", "floor": 1, "row": 1, "column": 8, "svgX": -17.6, "svgY": 0, "svgRot": 0 },
    { "id": "1-1-9", "floor": 1, "row": 1, "column": 9, "svgX": -16.6, "svgY": 0, "svgRot": 0 }
  ]
}
```

- `theater` — 화면 우상단에 표시되는 극장 이름.
- `seats[]` — 좌석 하나당 객체 하나.
  - `id` — `"층-행-번호"` 형식 문자열(예: `"1-7-23"`). `grades.json`의 `seatIds`와 `schedule.json`의 `seat`이 이 id로 좌석을 참조합니다.
  - `floor` / `row` / `column` — 정수. `floor`는 층 토글(1·2·3층)과 연결됩니다.
  - `svgX` / `svgY` — 좌석맵 평면 좌표(추상 단위, 대략 **1단위 ≈ 좌석 1칸**). `svgX`가 작을수록 왼쪽, `svgY`가 작을수록 위쪽입니다.
  - `svgRot` — 좌석 회전 각도(도). 곡선형 객석을 표현할 때 사용하며, 직선 배치면 `0`.
  - 좌석은 고정 크기 사각형으로 그려지므로 `svgX`/`svgY`/`svgRot`만 있으면 됩니다.
    (예전 데이터에 있는 `corners`/`svg` 필드는 현재 렌더링에 사용되지 않으므로 넣지 않아도 됩니다.)
- `floorMeta[층]` — 그 층 **중앙 블록**의 정보로, 행 번호 라벨 위치와 1층 초기 확대 배율 계산에 쓰입니다.
  - `centerOriginX` — 중앙 블록의 왼쪽 x 좌표.
  - `centerWidth` — 중앙 블록의 가로 폭(단위).
  - `centerMaxRow` — 행 번호 라벨을 몇 번 행까지 표시할지.

> **팁:** seatmap은 한 층씩 작성하는 것이 편합니다. 같은 행은 `svgY`를 같게, 열이 늘어날수록 `svgX`를 일정 간격으로 늘려 격자를 만든 뒤, 실제 객석 곡률에 맞춰 미세 조정하세요. 좌석 수가 많으면 스프레드시트로 좌표를 계산해 JSON으로 내보내는 방식을 권장합니다.

### 6. `data/holidays.json` — 공휴일 (선택, 전 공연 공용)

`날짜 → 공휴일 이름` 형태입니다. 스케줄 표에서 **일요일·공휴일은 빨강**, **토요일은 파랑**으로 날짜·시간이 표시됩니다(요일도 `26/04/12 (일)`처럼 함께 표기). 없어도 앱은 정상 동작하며, 이 경우 토/일만 색이 적용됩니다.

```json
{
  "2026-01-01": "신정",
  "2026-02-17": "설날",
  "2026-09-25": "추석"
}
```

### 7. `shows/<id>/matches.json` — 대결 (선택)

두 배역이 회차마다 승부를 겨루는 이벤트(예: 빌리 엘리어트의 "슈즈" 승부)를 정의합니다.
정의하면 스케줄 표에 대결 결과 컬럼(필터·숨기기 지원)과 통계 탭의 **대결 통계** 섹션이 생깁니다.
없거나 빈 배열이면 관련 UI가 나타나지 않습니다.

```json
[
  { "name": "슈즈", "point": 4, "roles": ["빌리", "마이클"], "defaultWinner": "빌리" }
]
```

- `name` — 대결 이름. 스케줄 컬럼 제목·대결 통계 제목으로 표시됩니다.
- `roles` — 대결하는 두 배역명(`casts.json`의 `role`과 일치).
- `point` — 대결이 벌어지는 **비중 지점**(1~10). 공연 중 배역이 교체된 회차에서 누적 비중으로 이 지점을 담당한 배우를 그 회차의 대결 참가자로 계산합니다(예: `4` = 극 중반 이전).
- `defaultWinner` — (선택) 결과가 기록되지 않은 종료 공연을 이 배역의 승리로 간주하는 통계 옵션("미기록=기본 승자" 체크 시)의 기본 승자.
- **회차별 결과는 `schedule.json`의 각 회차 `match` 필드에 기록**합니다: `"match": { "슈즈": { "winner": "빌리" } }` (무승부는 `"무승부"`, `note`로 메모 가능). 결과가 없으면 "모름"으로 집계됩니다.

### 마무리

파일을 바꿨다면 로컬 서버로 다시 열어 확인하세요. JSON 문법 오류가 있으면 데이터가 로드되지
않고 안내 창이 뜹니다. 브라우저 개발자 도구(F12)의 콘솔에서 어떤 파일이 문제인지 확인할 수 있습니다.
