# 요청 #0086 — Finale 코어: 부제 월 0패딩 옵션 + 시트맵 타깃을 SVG placeholder(id)로 지정

상태: 완료

---

## 요청 (작성: Spinouetter)

<!-- 대화로 요청됨 -->

- (1) Finale 보드 부제의 공연 기간에서 **월을 0패딩 없이**(예: `2026.04.12` → `2026.4.12`) 표기할 수 있게 해줘. (코어 수정)
- (2) 시트맵 위치·크기를 JSON 좌표(`target.rect`) 대신 **SVG 안에 안 보이는 상자를 넣고 그 id로 지정**하는 방식으로 바꿔줘. (코어 수정)

---

## 정리 (작성: Claude)

**(1) 부제 월 0패딩 옵션** — 부제(`subtitle`)의 공연 기간 문자열은 `periodStr()`가 `startDate`/`endDate`의 `-`를 `.`로 치환해 만들었다. ISO 날짜(`2026-04-12`)를 그대로 쓰므로 월이 항상 0패딩(`04`)으로 나왔다. `startDate`/`endDate`는 `isShowFinished()`·`fmtHeaderDate()` 등에서도 ISO 형태로 쓰이므로 **원본은 건드리지 않고**, 표기 단계에서만 0패딩을 제거하도록 매니페스트 옵션(`subtitle.padMonth:false`)을 추가했다.

**(2) 시트맵 타깃을 SVG placeholder(id)로** — 기존엔 시트맵 위치·크기를 매니페스트 좌표(`seatmap.target.rect`)로 지정했다. 이를 **SVG 안의 안 보이는 `<rect id=…>`** 를 두고 매니페스트에선 그 id만(`target.svgId`) 지정하면 되도록 바꿨다. 좌표가 도면(SVG)에 있으니 벡터 에디터에서 상자만 옮겨 위치·크기를 조정할 수 있다. 보드는 숨긴 컨테이너에서 렌더되어 `getBBox()`가 0을 주므로, **rect의 x/y/width/height 속성을 직접 읽어** 타깃으로 쓴다(레이아웃 비의존). placeholder는 transform 없는 페이지 레벨에 두어야 좌표=board 좌표.

## 할 일 (작성: Claude)
- [x] `periodStr(opts)`: `opts.padMonth === false` 면 월을 `parseInt`로 0패딩 제거(`04→4`, `07→7`). 기본(`padMonth` 미지정/true)은 기존과 동일하게 0패딩 유지. ISO(`YYYY-MM-DD`) 매칭 실패 시 기존 폴백(`-`→`.`).
- [x] `periodVenueVal(sub)`: 인자를 구분자(sep)에서 **subtitle 설정 객체**로 변경 — `sub.padMonth`·`sub.sep` 전달. 호출부는 `periodVenueVal(mani.subtitle)`.
- [x] (후속) 구판 공개 보드(billy-casting-board)에도 `subtitle.padMonth:false` 적용 — 2026 보드만 적용돼 있던 것을 통일. FIN_VER 22·finale.js v79
- [x] `injectSeatmap`: `cfg.target.svgId` 지원 — 그 id의 요소에서 x/y/width/height 속성을 읽어 타깃 rect로 쓰고, 읽은 뒤 DOM에서 제거. 속성이 없으면 `getBBox()` 폴백, 그것도 없으면 `cfg.target.rect` → 기본값 폴백.
- [x] `index.html` finale.js 캐시 v52 → v54.

## 결정 사항 (작성: Claude)
- **무회귀**: `padMonth`를 지정하지 않은 기존 보드는 월 0패딩 유지. `target.rect`로 지정한 기존 보드도 그대로 동작(폴백 유지). 새 방식(`target.svgId`)을 쓰는 보드에서만 SVG 상자를 참조.
- `startDate`/`endDate` 원본(ISO)은 그대로 — 다른 로직에서 그대로 사용.
- **매니페스트/도면(공연 데이터)은 이 커밋에 포함하지 않음(코어 함수만)** — `subtitle.padMonth:false`, `seatmap.target.svgId`, SVG의 `<rect id="fn-seatbox">` placeholder는 빌리 2026 보드 데이터라 별도 커밋으로 관리.
