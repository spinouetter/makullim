## 요청 (작성: Spinouetter)

<!-- 대화로 전달된 요청을 Claude가 옮겨 적음 -->

- 티켓 선택창에 이미 등록된 버튼들 위에, 삭제 버튼 위에 **숨김 / 취소 / 양도하기 / 이력** 기능을 넣어줘.
  - **숨김**: 현재 티켓 창 설정에는 남겨줘. 다만 자번 업데이트 안 함, 좌석 번호 취소선.
    숨김 누르면 버튼이 "보이기"로 바뀌고 보이기 누르면 돌아옴.
  - **취소**: 기존 정보는 그대로 유지하고 "취소 사유"(=내용), "취소 수수료"(=비용) 입력 창 띄운 뒤 관리.
  - **양도하기**: 기존 정보는 그대로 유지하고 "양수인"(=내용), "자체 할인"(=비용) 입력 창 띄운 뒤 관리.
  - **이력 (숫자)**: 취소 또는 양도 티켓이 있는 경우 숫자 표시, 별도 창으로 이력 표시.
    - 좌석번호 등급 / [취소/양도 아이콘] / 비용 / 내용 / 삭제
- 히스토리밖에 안 남은 공연의 경우 좌석 번호는 지워주고 티켓 창에 **이력 (x)** 로 표기.

## 정리 (작성: Claude)

- 스케줄 행의 티켓 선택 팝오버에 라이프사이클 버튼 행을 **삭제/취소/저장 줄 바로 위**에 추가
  (다중 티켓 관리 창 안의 팝오버에는 노출하지 않음 — `opts.lifecycle`로 분기).
- **숨김/보이기**: `p.ticketHidden` 토글. 티켓 정보는 유지하되 순번 컬럼의 **자번(자기 관극 순번)에서 제외**하고
  좌석 입력칸에 취소선(`.tk-hidden-seat`). 버튼 라벨이 숨김↔보이기로 바뀌며 팝오버는 열린 채 유지.
- **취소/양도하기**: 버튼을 누르면 같은 팝오버 안에 입력 폼이 뜬다.
  - 취소: "취소 사유"(내용) + "취소 수수료"(비용)
  - 양도: "양수인"(내용) + "자체 할인"(비용)
  - 확인하면 현재 맨 위 티켓의 {종류, 좌석, 등급, 비용, 내용}을 `p.ticketHistory`에 push하고,
    맨 위 티켓을 비운다(다중 티켓이 있으면 다음 티켓을 맨 위로 승격).
- **이력 (N)**: `p.ticketHistory` 길이. 팝오버 버튼(0건이면 비활성)·이력만 남은 회차의 티켓 셀 칩에서
  이력 창을 연다. 창의 각 행: `좌석번호 등급칩 / [취소|양도] 배지 / 비용 / 내용 / 삭제`.
  삭제는 그 기록만 지운다.
- **이력만 남은 회차**: 좌석이 비고 등급을 알 수 없어 기존엔 `—`였던 티켓 셀에,
  이력이 있으면 **"이력 (N)"** 칩을 띄워 이력 창으로 진입.
- 저장: localStorage 스냅샷(`ticketHidden`·`ticketHistory`) + 파일/Dropbox 내보내기(`ticketLife`, sid 기준).

## 결정 사항 (작성: Claude)

- **라이프사이클은 스케줄 팝오버 전용** — 다중 티켓 관리 창 안의 티켓 편집 팝오버에는
  넣지 않는다(그 창은 좌석·순서 관리 목적). `opts.lifecycle` 플래그로 구분.
- **취소/양도 = "맨 위 티켓 소진"** — 확인 시 그 티켓을 이력으로 넘기고 좌석을 비운다.
  다중 티켓이 있으면 기존 규칙대로 다음 티켓이 맨 위로 승격되어, 그 회차를 계속 관리할 수 있다.
- **비용의 의미는 종류별로 다름** — 취소=취소 수수료(내가 낸 손실), 양도=자체 할인(양도 시 깎아준 값).
  둘 다 이력에 `cost`로 저장하되 라벨만 다르게 보여준다(현재는 기록·표시용, 통계 반영은 범위 밖).
- **이력은 회차별 배열** — 한 회차를 취소했다가 다시 잡아 또 양도하는 식으로 여러 번 생길 수 있어
  누적 배열로 관리. 이력 창에서 개별 삭제 가능.
