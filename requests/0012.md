## 요청 (작성: Spinouetter)

테이블의 조정

- 테이블 헤더의 정렬을 가운데 정렬로 통일
- 스케쥴 테이블의 조정
  - 텍스트와 텍스트 입력, 캐스트 간의 정렬이 맞지 않음으로 모두 정렬되어 보이게 수정
    - 다만 배역이 다수 있을 때에는 맞지 않는게 맞음
  - 좌석 입력창이 커서 줄이 높아짐. 위 아래 여백을 줄여서 줄의 높이를 줄임
  - 한 페이지에 더 많은 배역이 보이도록 모든 컬럼의 좌우 여백 조절
- 통계 창
  - 전체, 종료, 관극, 예매의 아래쪽 여백 줄임. 지금보다 상자가 작게
  - 지금까지 쓴 금액, 앞으로 쓸 금액, 전체 (총액으로 변경)가 한 줄에 들어오는 방법 고민
- 시트 맵
  - 좌석 정보 창에서 날짜와 시간의 vertical-align이 맞지 않는 것 수정
  - 메모가 있는 경우 메모를 볼 수 있는 아이콘 추가해줘
- 세팅
  - 스케줄 보기 옵션에서 하이라이트된 텍스트가 주위의 텍스트와 vertical-align이 맞지 않음.
  - "오버레이 로 표시"라고 되어있는데, 이미 하이라이트가 되어 있기 때문에 띄어쓰기 필요없음.
  - 버튼 이름을 강조하는 것이 때문에 블록 요소는 필요없을 것 같은데, 들어간 것 같음.

## 정리 (작성: Claude)

테이블·통계·시트맵·세팅 다듬기.

**테이블 공통**: 헤더 텍스트 가운데 정렬로 통일.
**스케줄 테이블**: ① 셀 세로 정렬 통일(텍스트/입력/캐스트가 한 줄에 맞게 → `vertical-align:middle`, 캐스트 다수 줄은 예외 OK) ② 좌석 입력창 상하 패딩 축소로 행 높이↓ ③ 모든 컬럼 좌우 여백 축소로 더 많은 배역 표시.
**통계**: ④ 상단 카드(전체/종료/관극/예매) 더 작게+아래 여백↓ ⑤ 티켓 금액 3카드 한 줄에(모바일에서도), "전체"→"총액".
**시트맵 좌석 정보**: ⑥ 날짜·시간 vertical-align 수정 ⑦ 메모 있으면 메모 보기 아이콘 추가.
**세팅(스케줄 보기 옵션)**: ⑧ 라벨이 `display:flex; gap`이라 텍스트·`<b>`가 각각 플렉스 아이템으로 쪼개져 정렬 어긋남·불필요 간격("오버레이 로")·블록처럼 보임 → **텍스트를 `<span>`으로 감싸 한 인라인 런으로** 만들어 한 번에 해결.

## 추가 조사·수정 (Spinouetter: 날짜·시간 정렬 어긋남, `?v=35`)
- 조사(헤드리스 computed): 날짜·시간은 **동일 글꼴(Pretendard)·동일 13px**, 세로 위치도 동일 — 차이는 굵기뿐(날짜700/시간400). 표 내 실제 불일치 발견:
  - **좌석 입력칸이 Arial** (폼 입력이 글꼴 미상속) → `select,input` 에 `font-family:inherit` 추가.
  - **대결(슈즈) 셀 12px** → 13px로 통일.
  - **캐스트만 line-height 1.4** → tbody td에 `line-height:1.5` 통일(캐스트도 1.5).
- 재측정: 모든 셀 Pretendard·13px·lh 19.5px·텍스트 top 동일(완전 정렬). 굵기만 의도적 차이 유지.

## 헤더 vs 본문 정렬 조사·수정 (Spinouetter: 슈즈·배역 헤더와 배우 데이터 정렬 불일치, `?v=36`)
- 조사(헤드리스): **헤더는 전부 center**(0012-1에서 통일). 본문은 대부분 `start`(왼쪽), 가격만 `right`.
  → 슈즈(match)·배역(cast) 컬럼: 헤더 center인데 데이터 left라 불일치.
- 수정: `.match-cell`, `.cast-cell`에 `text-align:center` 부여 → 헤더와 본문 모두 가운데로 일치. (cast의 `.name`은 text-align 상속으로 가운데)
- 참고(미변경): 날짜·시간·좌석·티켓·메모는 헤더 center / 본문 left, 가격은 본문 right 유지. 필요 시 통일 가능.

## 티켓 팝업 화면 초과 수정 (Spinouetter, `?v=37`)
- 조사: 짧은 화면(390×560, 가시417)에서 팝업 bottom 453 > 417로 넘침(버튼 잘림). 직접 입력 칸(39px)이 주원인은 아니고, **팝업에 전체 높이 상한이 없어** 고정부(날짜+제목+수수료줄+버튼 ≈141px)+옵션이 합쳐져 초과.
- 수정: `.ticket-popover`를 `display:flex; flex-direction:column; max-height:calc(--app-height - 180px)`로, `.ticket-options`는 `flex:1 1 auto; min-height:0`(+1/3 cap) → 짧은 화면에선 옵션이 줄며 팝업이 화면을 안 넘김. 직접 입력 칸 패딩도 축소(39→32px).
- 검증: 560/620/700 모두 popBottom ≤ 가시높이, 버튼 항상 표시. 스크린샷 확인.

- 메모 아이콘 클릭 → `showToast(note)`로 내용 표시(좁은 시트맵에 적합), `title`도 부여.
