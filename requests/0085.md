- PDF 내보내기에서도 TOTAL 숫자가 웹/PNG처럼 지정 글꼴(Handlee)로 나오게 해줘 (코어 수정).

## 정리 (작성: Claude)

기존 PDF 내보내기(`exportPDF`+`bakeStyles`)는 텍스트 글꼴을 **Anton 아니면 IBM Plex Sans KR**로만 강제했다. 그래서 매니페스트에서 TOTAL 숫자를 Handlee로 지정해도 PDF에선 IBM Plex로 폴백됐다. → **매니페스트의 ttf 있는 폰트를 모두 PDF에 등록**하고, bake 시 **한글이 있는 텍스트는 한글 폰트(IBM Plex Sans KR)로, 그 외(라틴 숫자 등)는 계산된 글꼴이 등록돼 있으면 그대로** 쓰도록 일반화.

## 할 일 (작성: Claude)
- [x] `exportPDF`: KR·Anton 하드코딩 대신 `mani.fonts` 중 `ttf` 있는 폰트를 모두 `addFont`로 등록(등록 집합 `registered` 구성)
- [x] `bakeStyles(root, registered)` + `bakedFamily(cs,text,registered)`: 한글 포함 텍스트→`IBM Plex Sans KR Medm`, 아니면 계산 글꼴이 registered면 유지(부분매칭 폴백 포함), 아니면 KR
- [x] `fonts/Handlee-400.ttf` 번들, 매니페스트 Handlee 항목에 `ttf` 추가
- [x] `index.html` finale.js 캐시 v51 → v52
- [x] `bakedFamily` 단위테스트(분자/분모 Handlee, 한글명·태그라인 KR, 헤딩 Anton, 라틴 미등록→KR) 전부 통과

## 결정 사항 (작성: Claude)
- **무회귀**: registered에 Anton·KR이 그대로 포함되고, 한글은 여전히 KR로 강제 → 기존 보드 PDF 동일. 새로 등록되는 건 ttf가 있는 폰트뿐(구 보드 Handlee는 woff2만이라 미등록, 기존과 동일).
- 한글 판정은 텍스트 내용(`[가-힣…]`)으로. 라틴 전용 텍스트(숫자·TOTAL 라벨 등)만 자기 글꼴 허용.
- Handlee는 400 하나뿐이라 normal·bold 모두 같은 ttf로 등록.
- PNG/JPG 내보내기는 @font-face(woff2)로 렌더돼 원래도 Handlee 정상. 이번 변경은 벡터 PDF용.
