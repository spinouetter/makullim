## 요청 (작성: Spinouetter)

티켓 양도

티켓을 양도 받은 경우, 나의 예매 내역이 아닌 양도 받은 사람이 보내준 이미지를 보고 티켓을 찾아야 한다.
양도 받은 것을 쉽게 찾을 수 있도록 정보를 기록한다.

입력 UI

티켓 화면에서 수수료 +2000원 지우고 거기에 "양도받음" 이라는 선택 버튼을 추가한다.
기타 금액의 UI에서 위아래 화살표를 지운다.

표시 UI

스케쥴 표에서 할인율 오른쪽 위에 작은 점을 하나 찍어서 양도 티켓임을 나타낸다.

json

양도 받았다는 기록을 남기는 필드를 추가한다. 필드가 없으면 내가 구매한 티켓

---

## 정리 (작성: Claude)
- 양도받은 티켓을 기록·표시. 티켓 팝오버에서 수수료 토글 제거하고 "양도받음" 토글 추가, 기타 금액 입력의 스피너(화살표) 제거, 스케줄 셀 할인율 오른쪽 위에 점 표시. json에 ticketTransferred 필드(없으면 구매 티켓).

## 할 일 (작성: Claude)
- [x] 데이터 필드 `ticketTransferred` 추가(기본 false=내가 구매) + 초기화
- [x] 팝오버: `수수료 +2000` 라벨 → `수수료`로 변경(수수료 토글·ticketFee 그대로 유지), 별도로 `양도받음` 토글 추가
- [x] 기타 금액 입력 `type=number`→`type=text inputmode=numeric` (위아래 화살표 제거)
- [x] 저장/삭제 핸들러에 ticketTransferred 반영
- [x] 스케줄 셀: 양도 티켓이면 할인율(tk-disc) 오른쪽 위 작은 점(파랑)
- [x] 영속화: buildStateSnapshot/applyState + 좌석 JSON 내보내기/가져오기(6번째 요소=양도)
- [x] 헤드리스 확인(양도받음 토글·화살표 제거·점·localStorage 라운드트립)

## 결정 사항 (작성: Claude)
- "수수료 +2000원"은 라벨만 "수수료"로 변경(요청 정정). 수수료 토글·ticketFee·가격계산(+2000) 모두 그대로 유지. 양도받음은 별도 토글로 추가.
- 양도받음은 가격에 영향 없는 표식. 좌석 JSON 형식 확장: `[좌석,티켓,수수료,메모,(할인율 or null),(양도 true)]` — 양도일 때만 5·6번째 포함.
- 점 색은 선택 하이라이트와 구분되도록 파랑(#5b9bff), 미선택 상태면 아이콘 우상단에 표시.
