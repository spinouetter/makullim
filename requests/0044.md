## 요청 (작성: Spinouetter)

## 정리 (작성: Claude)

0043에서 상단 카드에만 넣었던 **남음(= 전체 − 종료)** 열을, 같은 `전체·종료·관극·예매`
구조를 쓰는 나머지 통계 표에도 종료와 관극 사이에 추가해 상단 카드와 순서를 맞춤.

## 할 일 (작성: Claude)

- [x] app.js 배역별 출연 통계 — 헤더·행에 `남음`(`total − ended`) 열 추가
- [x] app.js 기타 통계(월별·요일별) — `cells` 헬퍼·헤더에 `남음` 추가, 월별 빈 행 colspan 5→6
- [x] app.js 배역 조합 통계 — `buildComboBody` 헤더·행에 `남음` 추가
- [x] index.html — app.js 캐시버스터 v108 → v109
- [x] 검증 — `node -c`로 문법 확인

## 결정 사항 (작성: Claude)

- 대결(match) 통계는 `전체·승·패·무·승률` 구조라 대상 아님(요청대로 제외).
- 기타 통계의 **티켓별** 표는 `관극·예매·총액`만 있어 전체/종료 개념이 없으므로 제외.
- "남음"은 상단 카드와 동일하게 좌석 유무 무관한 **회차 기준**(전체 − 종료).
  가중 모드에서 분수값이 나오는 배역별·조합 표는 `fmtStatValue(total − ended)`로 표기.
