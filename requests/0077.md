## 요청 (작성: Spinouetter, 대화로 요청)

막공 기능을 다시 정리해줘. 설정 메뉴를 토글 4~5개로 단순화하고, 위 두 개와 아래 두 개 사이에 절취선을 둬:

- **배우 막공** — 배우가 마지막으로 공연할 때 이름을 강조.
- **페어막** — 배역을 1개 고르면 그 배역을 중심으로 한 주역 페어막(이름에 테두리), 2개 이상 고르면 조합 페어막을 테두리로 묶음. (기존 "페어막"과 "주역 페어막"을 하나로 합침.)
- ──(절취선)──
- **내 막공** — 전체 막공과 함께 내 막공도 표시(내 막공은 진하게).
- **미정 배역 포함**(기본 OFF) — 총막이 미정인 배역도 막공/페어막에 포함. 끄면 막공/페어막 계산과 배역 선택 버튼 양쪽에서 제외.
- **막공만 표시**("페어막만 표시"에서 이름 변경) — 막공/페어막 배우가 있는 공연만 남기고 접기.

세부:
- 배우 막공(전체·내 모두)도 "미정 배역 포함"을 따른다.
- 내 막공의 미정 판정 기준은 "총막(전체 마지막 회차)"이 아니라 **내가 마지막으로 티켓팅한 회차**로 본다. 그 회차에서 배역이 확정돼 있으면 총막이 미정이라도 포함.
- 내/전체 구분 표시 규칙(아래 표대로), 페어링 주역이 배우막공이면 알약이 우선(테두리 생략).

## 정리 (작성: Claude)

요청 0046/0051/0058/0076에 걸쳐 쌓인 막공 옵션(배역별 막공·페어막·주역 페어막·주역 미정 제외·내 막공)을 **토글 5개 + 절취선** 구조로 재설계한다.

- **페어막 통합**: 기존 별도 옵션이던 "페어막(조합)"과 "주역 페어막"을 하나의 "페어막" 토글로 합친다. 배역 선택 개수로 모드가 갈린다 — **1개=주역 페어막(lead 중심, 이름 테두리+주역 자신 굵게)**, **2개 이상=조합 페어막(선택 배역 묶음)**.
- **미정 배역 포함**(신규, 기본 OFF): 총막(최신 회차) 캐스팅이 미정인 배역을 막공/페어막 계산과 배역 선택 버튼 양쪽에서 넣을지/뺄지 하나로 제어. 기존 "주역 미정 제외"(0075)를 대체·일반화.
- **배우 막공**(구 "배역별 막공") 라벨·설명 정리. 전체·내 모두 "미정 배역 포함"을 따른다.
- **내 막공 미정 기준 변경**: 전체 막공은 "총막 미정 배역 제외"지만, 내 막공은 **내 마지막 티켓 회차**에서의 캐스팅을 기준으로 미정 여부를 판정(그 회차에 확정돼 있으면 포함).
- **막공만 표시**: 구 "페어막만 표시"의 이름 변경 + 대상 확장(막공/페어막 배우가 있는 회차 유지).
- **내/전체 시각 규칙**(확정된 표):

| 내 | 전체 | 배우막공(채운 알약) | 페어막(테두리) |
|----|------|--------------------|----------------|
| ✓ | ✓ (일치) | 골드 100% + 이름 양옆 점 | 실선 + 점 |
| ✓ | ✗ (내만) | 골드 80% + 이름 양옆 점 | 파선(같은 두께) + 점 |
| ✗ | ✓ (전체만) | 골드 100%, 점 없음 | 실선, 점 없음 |

  - 점은 이름 양옆(양쪽) 세로 중앙. 채운 알약의 점=배경색(var(--panel)), 테두리의 점=골드.
  - 내 막공 OFF: 내/전체 구분 없음 — 전부 실선/채움, 점 없음.
  - **우선순위**: 배우막공(내·전체 어느 쪽이든)이 있으면 페어막 테두리는 생략(알약 우선). 주역 자신도 배우막공이면 lead-self 테두리 대신 알약으로 굵게.

## 결정 사항 (작성: Claude)
- 페어막 통합에서 "1개=주역 / 2개+=조합" 분기는 별도 토글 없이 **선택 개수로만** 판정 — 옵션 수를 줄이자는 요청 취지에 맞춤.
- 내 막공 미정 기준을 "내 마지막 티켓 회차"로 잡은 이유: 총막이 미정이어도 내가 실제로 본 마지막 회차엔 캐스팅이 확정돼 있을 수 있으므로, 그 회차 기준이 "내" 막공 의미에 맞다.
- 배우막공 우선(테두리 생략): 알약(막공)이 페어막보다 강한 신호이며, 주역이 배우막공일 때 알약+테두리가 겹쳐 지저분해지는 문제를 없앤다.
- 내만 배우막공 채움 농도는 100%↔80% 사이를 오갔고 **최종 80%**로 확정(일치=100%와 점 유무·농도로 구분).
- 0076에서 도입한 "전체 막공=옅은 점선" 시각 스킴은 이 재설계로 **채운 알약(농도·점 구분)** 으로 대체됨 → 0076은 완료 처리.
- 표시 굵기는 CSS 지정(700)대로지만 실제 렌더는 글꼴에 좌우됨 → 본문 글꼴을 실제 Bold 글리프가 있는 IBM Plex Sans KR로 교체(같은 커밋의 별도 변경, 요청 문서 없음·git history 관리).
