## 요청 (작성: Spinouetter)

티켓 이력 목록 창과 다중 티켓 목록 창을 동일한 스타일로 통합해줘

---

## 정리 (작성: Claude)

- 두 창(**다중 티켓 관리** `tmBody` · **티켓 이력** `thBody`)의 행이 서로 다른 마크업/스타일이라 통일감이 없었음.
  같은 "티켓 한 줄" 레이아웃을 공유하는 공통 행 스타일로 묶는다.
- 이력 창을 2줄(좌석·알약·배지·비용 / 메모·삭제)에서 **관리 창과 같은 1줄**로 정리.
- 좌석 칸 폭·눈 버튼·티켓 알약 폭·우측 금액 정렬·행 높이/여백을 두 창이 픽셀 단위로 일치하게.

## 할 일 (작성: Claude)
- [x] 공통 행 클래스 신설: `.tk-lrow`(행)·`.tk-lseat`(좌석 68px)·`.tk-leye`(눈)·`.tk-lpill`(알약 80px)·`.tk-lnote`(메모)·`.tk-linfo`(우측 금액).
- [x] 관리 창 행: `.tm-row`에 `.tk-lrow` 병용, 눈 버튼 인라인 스타일 → `.tk-leye`, 가격 → `.tk-linfo`, 티켓 트리거 폭 80px로 알약과 정렬.
- [x] 이력 창 행: 2줄 → 1줄(`좌석·눈·알약·취소/양도 배지·메모(말줄임)·비용·✕`). 삭제 버튼은 텍스트 "삭제"→`✕`.
- [x] `ticketPillHtml` 알약 클래스 `th-pill`→공통 `tk-lpill`. 사라진 `.th-*`(l1/l2/seatwrap/seat/eye/pill/note2) CSS 정리.

- [x] 헤드리스 검증: 두 창 스크린샷으로 열 정렬·행 높이·알약/눈/금액 위치 일치 확인.

## 후속 (작성: Spinouetter)
- 관리 창에도 **티켓 정보가 채워진 상태**로 확인(스크린샷).
- 이력에서 **메모는 다음 줄**로 내려서 표시.
- 이력 가격 표기: **<취소선>원래가격</취소선> + 취소수수료(취소) / 자체할인(양도)**.

## 후속 반영 (작성: Claude)
- 공통 뼈대 `.tk-lrow`는 **flex 레이아웃만** 담당하도록 축소하고, 여백·경계선은 각 창 래퍼(`.tm-row`/`.th-row`)로 이동(이력에 메모 2번째 줄이 생겨 분리 필요).
- 이력 행 = 1줄(좌석·눈·알약·취소/양도 배지·가격·✕) + **메모 2번째 줄**(`.th-sub`, 알약 아래 정렬, 말줄임+title). 메모 없으면 1줄만.
- 가격 = `histOrigPrice(h)`로 계산한 **원래가격(취소선, `.th-orig`)** + **비용(`.th-fee`, 골드)**. 원래가격 계산 불가(무명·할인권 등)면 비용만. 임의 할인권은 정가×(1−할인율)로 폴백.
- 캐시버스터 app.js v189·styles.css v134. 헤드리스 재검증(관리 창 가격·알약 표시 / 이력 취소선+수수료·자할·메모 2줄).

## 후속2 (작성: Spinouetter)
- **좌석 번호 입력창을 "티켓 입력(팝오버)" 안에 넣기** — 티켓 이력 편집과 동일하게.
- **"+ 티켓 추가"를 누르면 티켓 입력 팝오버가 뜨게**.

## 후속2 반영 (작성: Claude)
- `buildTicketPopover`에 `seatField` 옵션 추가 → 이력 편집(`historyEdit`)과 같은 좌석 입력 줄(`.tk-hist-seat`)을 팝오버 상단에 표시. 관리 창 팝오버는 `{bookingField, seatField:true}`로 연다.
- 관리 창 행의 인라인 좌석 입력(`.tm-seat-input`) 제거 → **읽기 전용 좌석 버튼**(`.tm-seat-btn`, 좌석 없으면 "좌석 입력"). 좌석 버튼·티켓 트리거를 누르면 팝오버가 열림.
- 빈 티켓도 기본 등급(grades[0])으로 팝오버를 열어 **팝오버 안에서 좌석부터 입력**. 좌석을 바꾸면 등급·티켓 목록·가격이 따라 갱신(현재 팝오버 선택은 유지). 좌석 중복은 알림 후 되돌림.
- **"+ 티켓 추가"** → 빈 티켓 추가 후 곧바로 팝오버 오픈(좌석 입력에 포커스). 인라인 편집용 롱프레스 포커스 로직 제거(드래그 순서변경은 유지).
- 캐시버스터 app.js v190·styles.css v135. 헤드리스 검증(관리 팝오버 좌석칸·추가 시 팝오버·좌석 입력→등급 갱신[2-6-3→S석]).

- 좌석 유효성 실시간 반영: `input` 시 유효하면 눈 활성, 잘못되면 `.tk-hist-seat.invalid-seat`(빨간 물결 밑줄) + 눈 `disabled`(기존 `.seat-input.invalid-seat` 규칙에 `.tk-hist-seat`도 포함).
- 캐시버스터 app.js v191·styles.css v136. 헤드리스 검증(유효=눈 활성 / `9-9-9`=빨간 밑줄+눈 비활성).
- **간격 조정(팝오버)**: 좌석 입력이 `flex:1`(전체 폭)이라 텍스트는 왼쪽에, 눈은 입력창 오른쪽 끝(먼 거리)에 있어 사이가 벌어져 보였음 → 입력을 고정폭(100px, 내용에 맞춤)으로 줄여 눈이 좌석 번호 바로 옆에 붙게. styles.css v137.
- **간격 조정(관리·이력 목록)**: 목록 행의 좌석 칸(`.tk-lseat`)도 같은 이유로 폭 68px(줄맞춤용) 안에서 좌석번호가 짧으면 눈까지 빈 공간이 벌어져 보임 → 실제 데이터의 최장 좌석번호(7자, 예 "1-10-10") 기준으로 68px→50px로 줄여 간격을 좁히되 행마다 줄맞춤은 유지. styles.css v138.
- **취소선 원가 오른쪽 정렬**: `.th-orig`(취소선 원가)가 `.th-fee`(취소수수료/자체할인) 바로 앞에 붙어 있어, 그룹 전체(margin-left:auto)만 행 오른쪽에 붙고 원가 자체는 뒤따르는 금액의 자릿수에 따라 좌우로 흔들렸음(예: "4,000원"과 "120,000원"이 서로 다른 폭이라 앞의 원가 위치가 행마다 달라짐) → `.th-orig`·`.th-fee` 각각 고정폭(56px) 오른쪽 정렬 열로 분리해 두 값 모두 행마다 같은 자리에 정렬. styles.css v139.

## 후속4 반영 (작성: Claude)
- 좌석 입력줄(`.tk-hist-seat-row`)에 `opts.bookingField`가 있으면 예매일 필드를 같은 줄에 이어 붙이도록 통합(기존에는 가격 요약 아래 별도 줄이었음). 좌석 필드가 없는 경우를 위한 단독 `.tk-booking-row` 렌더는 폴백으로 유지.
- 좌석 입력 폭 100px→68px(관리·이력 목록의 `.tk-lseat`와 동일하게 최장 좌석번호 7자 기준)로 줄여 예매일이 들어갈 자리 확보.
- 예매일 라벨을 텍스트("예매일")에서 **달력 아이콘**(eye 아이콘과 같은 크기·스타일, title 툴팁)으로 바꿔 좁은 줄에서 날짜 입력(MM/DD/YYYY) 표시 폭을 넉넉히 확보(아이콘 전환 전엔 날짜 끝자리가 달력 피커 아이콘과 겹쳐 보였음).
- `.tk-add-multi` 버튼 라벨 "＋ 좌석 추가" → "**중복 티켓 관리**"(title도 "현재 티켓 저장 후 중복 티켓 관리로 이동"으로 갱신). 폭 고정 없이 `white-space:nowrap`이라 라벨 길이 변경에 자동 대응.
- 캐시버스터 app.js v193·styles.css v141. 헤드리스 검증(관리·이력 팝오버에서 좌석+눈+달력아이콘+예매일 전체값 표시 확인, 기본 팝오버 버튼 라벨 확인).
- **버그 수정**: 커스텀 달력 아이콘 외에 `<input type="date">`의 브라우저 기본 달력 아이콘이 같은 줄에 또 나타나 아이콘이 2개로 보임 → `::-webkit-calendar-picker-indicator`를 숨겨 커스텀 아이콘 하나만 남김(입력칸은 그대로 클릭해서 날짜 선택기 열림, 값 입력도 정상). styles.css v142.
- **간격·정렬 조정**: 눈 아이콘과 달력 아이콘이 바로 붙어 있어 좁아 보임 + 날짜 입력이 `flex:1`로 남은 공간을 다 채워 불필요하게 넓었음 → 날짜 입력을 고정폭(104px)으로 줄이고, 달력 아이콘에 `margin-left:auto`를 줘 [달력 아이콘+날짜 입력]을 한 덩어리로 행 오른쪽 끝에 정렬(눈-달력 사이에 자연스러운 간격 생김). styles.css v143.

## 결정 사항 (작성: Claude)
- **좌석 입력 위치 통일**: 관리·이력 모두 좌석을 **팝오버 안**에서 입력(목록 행은 읽기 전용 표시). 인라인 편집을 없애 두 창의 편집 흐름을 동일하게.
- 빈 티켓의 팝오버는 기본 등급으로 열되, 좌석 입력 즉시 실제 등급으로 재렌더(이력 편집과 동일한 폴백 방식).
- **이력 삭제**는 행 오른쪽 `✕`로 유지(관리 창은 좌석 비우기·순서 드래그가 고유 기능). 각 창 고유 요소(드래그 핸들 / 취소·양도 배지·메모·삭제)는 공통 뼈대 위에 얹는다.
- 취소/양도 의미는 좌측 **배지(취소·양도)**가 이미 구분하므로 가격 셀엔 별도 라벨 없이 금액만(툴팁으로 보강).
