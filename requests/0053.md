## 정리 (작성: Claude)

- 좌석맵에서 좌석을 선택하면 **무대에서 실제 거리가 같은 좌석**을 강조하는 **숨김(실험) 기능**.
- **켜는 법**: 주소에 쿼리 `?equidist`(예: `/billy/?equidist=1`)를 붙였을 때만 동작. 평소엔 완전히 비활성(기존 동작 그대로).
- **거리 계산**: 무대 위 **두 초점까지 거리의 합**이 같은 좌석(=같은 타원 위). 단, 좌석맵은 **세로(무대→객석 깊이) 축이 실제의 일부 비율로 눌려** 그려져 있어(블루스퀘어 = 실제의 42%) 그 축을 복원한 뒤 계산한다. 이 비율은 **공연장 속성**(`theatre.heightScale`).
- **초점 모드(후속 요청 — URL 값으로 선택)**:
  - `?equidist`(또는 `=1`/`=center`) → **무대 중앙 한 점** 초점(동심원). 두 초점이 같은 점인 특수 경우.
  - `?equidist=ends` → **무대 양 끝** 두 점을 초점으로 하는 타원.
  - `?equidist=thirds` → **무대 1/3·2/3** 두 점을 초점으로 하는 타원.
- **표시**: 같은 층에서 (거리합)/2 오차 ±허용치 안의 좌석을 청록 테두리로 강조 + 안내용 타원(세로 축소 반영)을 좌석 뒤에 그린다.

### 데이터/공식
- 공연장 JSON에 `heightScale`(0~1, 예: `0.42`) 추가 — 세로 축이 **실제의 그 비율로** 눌려 그려짐을 뜻함.
- 복원계수 `vScale = heightScale`(0.42). 실제 깊이 = `svgY / vScale`.
- 초점 `F1,F2`(무대 y=cy 위 두 x). 좌석 지표 `M = (dist(P,F1)+dist(P,F2)) / 2`, `dist = √( dx² + (dy/vScale)² )`. center 모드는 `F1=F2` 라 단일 거리와 동일.
- 같은 타원 판정: `|M_i − M_선택| ≤ EQUIDIST_TOL`(현재 0.7).
- 안내 타원(그린 좌표): 중심=두 초점 중점, `rx=M`(가로 반장축), `ry=√(M²−c²)·vScale`(세로 반단축), `c=|F1x−F2x|/2`. 초점이 가로로 놓여 축정렬 타원(회전 없음).

## 결정 사항 (작성: Claude)

- **숨김 = URL 게이트만**: 설정 UI를 두지 않고 `?equidist`가 있을 때만 활성화(요청대로 명시적으로만). 슬러그 주소(`/billy/?equidist=1`)에 붙이면 되고, 루트(`/?...`)는 슬러그로 리다이렉트되며 파라미터가 빠질 수 있으니 슬러그 주소에 붙인다.
- **세로 축소는 공연장 속성**: `heightScale`로 공연장 JSON에 두어 공연장마다 다르게 설정 가능. 값이 없으면 보정 없이 정원으로 계산(안전 기본값). 의미는 "좌석맵 세로가 **실제의 heightScale 배**"(블루스퀘어 0.42 = 실제의 42%로 눌림) — 복원은 `svgY / heightScale`. (초기엔 '42%만큼 축소=÷0.58'로 오해했으나, 사용자 확인으로 '실제의 42%=÷0.42'로 정정.)
- **무대가 정의된 층에서만**: 거리 원점은 `floorMeta[floor].stage`. 현재 무대 좌표는 1층에만 있어 1층에서만 강조되고, 2·3층 좌석 선택 시엔 조용히 아무것도 하지 않는다(층별 무대 좌표가 생기면 자동 확장).
- **강조는 같은 층 안에서만**: 층마다 좌표계가 독립적이라 거리 비교는 선택 좌석과 같은 층으로 한정.
- 허용오차 `EQUIDIST_TOL=0.7`은 아치가 한 좌석 두께로 보이도록 잡은 기본값. 필요하면 조정.
- **초점 모드도 URL로만 선택**(`?equidist=ends|thirds`, 기본 center). 지표를 '두 초점 거리의 평균'으로 잡아 center(단일 초점)와 자연스럽게 통일 — 같은 허용오차·같은 강조 로직을 세 모드가 공유한다. 초점이 무대 위 가로로 놓여 안내 타원은 회전 없는 축정렬로 그린다.
