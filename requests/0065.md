## 요청 (작성: Spinouetter)

<!-- 대화로 전달된 요청을 Claude가 옮겨 적음 -->

- 통계에 양도/양수 티켓 수·금액, 취소한 티켓 수·취소 수수료 전액합 표시해줘.
  이 때 (금액의) 기본은 (티켓 가격+수수료).
- 그리고 티켓 기타 금액에 `-`도 입력 가능하게(양도받았을 때 자체 할인).
- (후속) 가격을 두 개의 컬럼으로 나눠줘. 앞쪽은 총금액, 뒤쪽은 자체할인 or 취소수수료.
- (후속) 메인 화면의 이력 (숫자)는 "양도 (x) / 취소 (y)" 둘 중 하나만 있으면 하나만 표시.
- (후속·정정) 양도/취소 칩은 그냥 "이력 (x)". 자체할인·수수료 컬럼은 이력에서
  sum(취소 수수료) + sum(자체 할인)을 포함.
- (후속·재정정) 현재 티켓의 모든 비용은 가격 컬럼에. "자체할인·수수료" 컬럼 →
  "기타 비용"(이력 비용 전부 합, 전부 내가 지불한 비용). 기타 비용은 Settings의
  "기타 비용 표시하기"를 켰을 때만, 헤더에 숨기기 + 자체할인/취소수수료 개별 선택 옵션.
- (후속) 숨김 있는 필드는 Settings에 표시/숨김 global 선택지를 만들고, 그 필드는
  표시 드롭박스(숨김 바)에 안 보이게.
- (후속) "스케줄 컬럼 표시" 설명 제거, 체크박스 대신 공통 토글 버튼(floor-toggle-btn)으로.

## 정리 (작성: Claude)

- **기타 통계**에 "티켓 취소·양도 통계" 블록 추가(데이터가 있을 때만 표시). 3행:
  - **양수 (양도받음)**: 현재 맨 위 티켓 중 `ticketTransferred`인 것. 수 + 금액(기본=티켓가+수수료).
  - **양도**: 이력(`ticketHistory`)의 `transfer`. 수 + 금액(기본, 이력에 저장된 `amount`).
  - **취소**: 이력의 `cancel`. 수 + 금액(=취소 수수료 `cost` 합).
  - "기본 금액"은 **티켓 가격 + 수수료**만(기타·임의할인 조정 제외). 0064의 취소·양도 확정 시
    그 시점의 기본 금액을 이력 항목 `amount`로 저장해 둔다.
- **기타 금액 음수 허용**: 티켓 팝오버의 "기타 …원" 입력에 맨 앞 `-`를 부호로 인정.
  양도받을 때 자체 할인처럼 최종가를 깎는 데 쓴다(가격 = 티켓가 + 수수료 + 기타).
  기존엔 `>0`만 허용해 음수가 0으로 잘렸던 것을 저장·복원·내보내기·좌석 JSON 전 경로에서 허용.

## 결정 사항 (작성: Claude)

- **양수는 활성 티켓, 양도·취소는 이력** — 양도받음(`ticketTransferred`)은 지금 들고 있는 티켓의
  성격이라 현재 맨 위 티켓에서 집계하고, 양도·취소는 티켓을 소진한 결과이므로 이력에서 집계.
- **양도·취소 금액의 의미가 다름** — 양도/양수는 "기본 티켓 가치(가격+수수료)", 취소는 "취소 수수료 합".
  한 표의 금액 열이지만 취소 행에만 작은 "수수료" 주석을 붙여 구분.
- **이력에 기본 금액을 저장** — 이력 항목엔 티켓 종류·수수료가 없어 나중에 재계산이 불가하므로,
  취소·양도 확정 시점의 기본 금액을 `amount`로 박아 둔다(0064 이전 이력은 `amount` 0으로 보정).
- **맨 위 티켓만 집계** — 다중 티켓 규칙(CLAUDE.md)대로 양수 집계도 맨 위 티켓 기준.
- 통계 반영은 표시(수·합계)까지이며, 총액 카드·티켓별 표의 가격 계산엔 기존대로 기타(음수 포함)가 반영된다.
