---

## 요청 (작성: Spinouetter)

스케줄의 옵션으로 지금 버튼을 추가함

이 버튼은 화면을 조금이라도 스크롤하면 활성화되고, 누르면 새로고침했을 때로 스크롤이 초기화됨
초기화 이후에는 다시 비활성화

---

## 정리 (작성: Claude)

스케줄 툴바에 **"지금" 버튼** 추가.
- **활성 조건:** 스케줄 스크롤 영역(`.table-scroll-wrap`)이 "홈 위치"에서 벗어나면(세로/가로 어느 쪽이든) 활성화.
- **홈 위치:** 새로고침 직후와 동일 = `scrollToCurrentPerf()`가 만든 위치(현재/다음 공연이 위에서 4번째 줄, scrollLeft 0). 현재 공연이 없으면 최상단.
- **클릭:** 홈 위치로 스크롤 초기화 → 다시 비활성화.

## 할 일 (작성: Claude)

- [x] 툴바에 `#nowBtn` "지금" 버튼 추가(기본 비활성, margin-left:auto로 우측 정렬 시작) (`?v=24`)
- [x] `goToNow()`: scrollLeft=0 + scrollTop=0 후 `scrollToCurrentPerf()`, 홈 scrollTop 기록
- [x] `updateNowBtn()`: 홈과 2px 초과 차이(세로) 또는 scrollLeft>2면 활성(.active)·아니면 disabled
- [x] wrap scroll 리스너에 updateNowBtn 연결, 최초 자동스크롤 후 홈 기록·버튼 갱신
- [x] 헤드리스 검증: 홈=비활성 / 세로·가로 스크롤=활성 / 클릭=홈(3642)복귀·비활성. 스크린샷에 활성(금색) 버튼 확인

## 결정 사항 (작성: Claude)

- 버튼은 항상 보이고 disabled로 활성/비활성 표현(`하이라이트 해제` 버튼과 동일 패턴, `.floor-toggle-btn:disabled` 스타일 재사용).
- 홈 scrollTop은 `scrollToCurrentPerf` 호출 직후 값을 `scheduleHomeScrollTop`에 기록해, 비교로 활성 여부 판단(허용 오차 2px).
- 가로 스크롤(scrollLeft>2)도 "스크롤됨"으로 간주.
