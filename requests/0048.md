# 요청 #0048 — 헤더 제목 모바일 한 줄 자동 맞춤(fit-to-width)

상태: 완료 <!-- 대기 / 진행 중 / 완료 / 보류 -->

---

## 요청 (작성: Spinouetter)

<!-- 대화로 전달된 요청을 Claude가 옮겨 적음 -->

모바일에서 헤더 제목("막올림 · 빌리 엘리어트 더 뮤지컬 (2021)")이 길어서 오토랩(줄바꿈)이 되는데,
이럴 때 글자 크기가 자동으로 줄어서 한 줄에 나오게 할 수 있어?

---

## 정리 (작성: Claude)

- 헤더 `h1`("막올림 · <공연 제목>")은 고정 `font-size:22px`라, 화면이 좁으면(특히 모바일)
  제목이 두 줄로 줄바꿈된다.
- 공연마다 제목 길이가 다르므로, **가용 폭을 실측해 한 줄에 들어갈 때까지 글자 크기를 줄이는**
  fit-to-width 방식으로 처리한다(특정 공연 하드코딩 없이 모든 공연에 동작).
- `fitHeaderTitle()`: `h1`을 `white-space:nowrap`으로 두고 폰트 크기를 기본값(22px)에서 시작해,
  `h1.scrollWidth`가 헤더 콘텐츠 폭(패딩 제외)을 넘으면 0.5px씩 줄인다. 가독 하한 13px에서 멈추며,
  그 아래로는 다시 줄바꿈을 허용(폭이 극단적으로 좁을 때의 안전장치).
- 최초 로드(제목 확정 후)·창 리사이즈·화면 회전·웹폰트 로드 완료 시점에 다시 맞춘다.

## 할 일 (작성: Claude)

- [x] `updateLayoutVars()`에 `fitHeaderTitle()` 합류(리사이즈·회전 시 재계산) — 제목 크기 변경이

- [x] `init()` 제목 확정 직후 1회 호출, `document.fonts.ready` 후에도 재호출(폰트 폭 변동 대응)
- [x] 캐시버스터 — app.js v113 → v114
- [x] 검증 — 헤드리스 320/360/390/768px 폭에서 제목이 모두 1줄(각각 14.5/17/18/22px), SEVERE 없음

## 결정 사항 (작성: Claude)

- **CSS 미디어쿼리 대신 JS 실측**: 제목 길이가 공연마다 달라 고정 `clamp()`로는 짧은 제목이
  불필요하게 작아지거나 긴 제목이 여전히 넘칠 수 있다. 콘텐츠 폭을 재는 방식이 모든 제목에 안전.
- **폭 측정 기준은 `header` 콘텐츠 박스**: 제목 블록(`.title-block`)은 `width:fit-content`라
  제목을 감싸며 커지므로 가용 폭 기준이 될 수 없다. 헤더 폭에서 좌우 패딩을 빼 실제 가용 폭을 얻는다.
- **하한 13px**: 그 아래는 가독성이 떨어져, 더 좁으면 한 줄 고집 대신 줄바꿈을 허용한다.
- **인라인 스타일로 처리**(styles.css 무변경): `white-space`·`font-size`만 JS로 주입해 로직을
  한곳(app.js)에 모으고 CSS 버전 변동을 피함.
