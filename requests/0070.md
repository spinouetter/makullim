## 요청 (작성: Spinouetter)

예매 히스토리 탭 (History) 하나 만들어줘.

그리고 여기에는 현재 남아있는 취소 수수료가 남아있는 티켓 부터 보여줘서 내가 언제까지 최소의 취소 수수료로 취소할 수 있는지 한눈에 볼 수 있게

## 정리 (작성: Claude)

- 새 탭 **History**(Schedule 다음). 보유 중인 **예정 티켓**을 모아, 각 티켓을 **지금 취소하면 드는 최소 수수료**와
  **그 금액으로 취소할 수 있는 마감일(날짜 + n일 남음)** 로 보여준다. 취소료 엔진은 0069(취소료 계산기) 재사용.
- 정렬: **취소 데드라인 임박순**(현재 구간 마감일이 가까운 것부터) → 언제까지 싸게 취소할 수 있는지 한눈에.

## 후속 (작성: Spinouetter)
- 히스토리를 **Tickets**로 이름 변경.
- 정렬: **예매날짜순 · 공연날짜순 · 취소 데드라인 임박순**. 데드라인순은 마지막 예정 공연 뒤에 **이미 끝난 공연을 날짜순**으로.
- **예매 날짜가 같으면 공연 빠른 순.**
- **취소 공연 보기 / 양도 공연 보기** 옵션.

## 후속 반영 (작성: Claude)
- 탭·페이지·함수 rename: History→**Tickets**(`data-page/id=tickets`, `renderTickets`, `ticketsBody`).
- 상단 컨트롤 바(sticky): **정렬 세그먼트**(데드라인순/예매순/공연순) + **취소·양도 보기 체크토글**. 상태는 저장(state)에 영속.
- 아이템 = 보유 티켓 + (토글 시)취소·양도 이력. 정렬:
  - **데드라인순**: ①데드라인 있는 예정(임박순, 동률이면 공연 빠른 순) → ②예정·예매일 미입력 → ③지난(종료 보유+취소+양도, 공연일 **내림차순**). ③ 앞에 "지난 공연" 구분선.
  - **예매순**: 예매일 오름차순, **같으면 공연 빠른 순**, 예매일 없으면 맨 뒤.
  - **공연순**: 공연일시 오름차순.
- 취소/양도 이력이 예매순에 뜨도록 `ticketHistory`에 **bookingDate 보존**(normTicketHist + 저장 시). 지난 카드는 2줄(취소/양도 뱃지·수수료·사유 / 관람 완료).
- 카드 클릭: 보유=스케줄 팝오버(예매일 없으면 계산 패널), 이력=해당 행으로 이동. 캐시버스터 app.js v179·styles.css v124.
- 검증 — 헤드리스: 데드라인순(지난 공연 구분·양도/취소/관람완료 뱃지), 예매순(예매일 오름차순·미입력 맨 뒤), 공연순(토글 OFF=이력 숨김) 스크린샷.

## 후속2 반영 (작성: Claude)
- 상태 `devFeatOn`(기본 **OFF**, 저장됨). Settings > 기능 표시에 체크박스 `optDevFeat`.
- 게이팅: Tickets 탭 표시(`applyTicketsVisibility`) / 티켓 팝오버 시계 아이콘 · 취소료 계산 패널(0069) / 취소 폼의 취소료 자동 프리필 — 모두 `devFeatOn`일 때만.
  끄면 탭 숨김(열려 있으면 스케줄로), 아이콘·패널 미표시, 프리필 없음(수동 입력은 기존대로 유지).
- Settings 섹션 제목 **"기능 표시" → "개발 중인 기능 보기"**(Finale도 개발 중이라 같은 섹션에 묶음). 토글 라벨의 중복 "(개발 중)" 접두어 제거.
- 검증 — 헤드리스: OFF=탭 숨김·시계 없음 / ON=탭·시계 표시 / OFF 복귀 확인. 캐시버스터 app.js v180·styles.css v125.

## 결정 사항 (작성: Claude)
- **보유(예정) 티켓 기준**: 맨 위 티켓 좌석 있음 + 종료 전(`!isEnded`) + 숨김/전관/스케줄취소 아님. 다중 티켓의 추가 좌석은
  예매일이 맨 위 티켓에만 있어 v1은 맨 위 티켓만 다룬다.
- **"최소 수수료 마감일"** = 현재 구간의 마감일(그 금액으로 취소 가능한 마지막 날). 취소료는 공연에 다가갈수록 오르기만 하므로
  현재 구간 금액이 곧 지금 가능한 최소액. 카드에 다음 구간 금액도 병기해 "이후 얼마"를 보여준다.
- **예매일 미입력** 티켓은 계산 불가라 하단 그룹으로 분리하고, 카드 클릭 시 계산 패널을 바로 연다.
- 별도의 취소·양도 '기록'은 기존 티켓 이력 창에 있으므로, 이 탭은 앞으로의 취소 데드라인에 집중한다.
