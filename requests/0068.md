## 요청 (작성: Spinouetter)

<!-- 대화로 전달된 요청을 Claude가 옮겨 적음 -->

- 양도하기/취소하기 기능 개선(모바일 증상)
  - 버튼 누르면 팝오버 가로·세로 길이가 다 변함. 가로는 텍스트 입력창 때문 —

  - 양도/취소하기 누르면 아래 버튼 줄 두 줄을 모두 지우고 돌아가기/확인만 남김.
- 양도/취소 이력 UI 개선
  - 두 줄 리스트: 좌석번호 ( [등급] 티켓이름 할인율 ) 원래가격 [양도/취소] 취소수수료 [삭제] / 양도 메모
  - UI 너비는 다른 팝오버와 동일하게.

## 정리 (작성: Claude)

- **팝오버 고정 폭**: `.ticket-popover`를 `width:max-content` → **고정 288px**(내용 자연폭)로. 폼이 열려도
  가로 길이가 변하지 않는다(입력창은 `width:100%`라 폭을 밀지 않음). 티켓 이름은 넘치면 말줄임.
- **취소·양도 폼 = 컴팩트 팝오버**: 폼이 열리면 `buildTicketPopover`가 **폼만 있는 팝오버**로
  대체한다(날짜 + 제목 + 양수인/사유 입력 + 취소수수료/자체할인 + 돌아가기·확인).
  티켓 목록·가격요약·수수료 줄·티켓 관리 줄·삭제/닫기/저장 줄을 **모두 제거** → 세로가 짧아져
  배경이 잘려 겹치던 증상 해소(원인=내용이 max-height 초과 시 오버플로).
- **이력 2줄 UI**: 1줄 `[좌석 (등급칩 티켓이름 할인율) 원래가격 배지 비용]`, 2줄 `[메모 … 삭제]`.
  이력 항목에 `ticketType`·`discount`를 새로 저장(옛 기록은 없으면 생략). 창 너비는 340px로
  좁혀 팝오버에 가깝게(기존 480px). 삭제 버튼은 폭 제약상 2줄 오른쪽으로 이동.

## 결정 사항 (작성: Claude)

- **폼은 별도 컴팩트 팝오버** — 기존엔 폼이 전체 팝오버 하단에 끼어 높이가 커지고
  삭제/닫기/저장 줄이 남았다. 폼 전용 화면으로 바꿔 요청(두 줄 제거·돌아가기/확인만)과
  배경 겹침(오버플로)을 한 번에 해결.
- **삭제 버튼은 이력 2줄로 이동** — 팝오버 폭(≈266~340)에 1줄로 모든 필드+삭제를 넣으면
  티켓 이름이 1글자로 잘려, 삭제를 메모와 같은 2번째 줄 오른쪽에 배치. 이름 가독성 우선.
- **이력 창 340px** — "다른 팝오버와 동일"에 가깝게 좁히되, 요청한 필드(이름·할인율·두 금액)가
  읽히도록 266이 아닌 340으로 절충(기존 480 대비 대폭 축소).
- 옛 이력 항목은 `ticketType`/`discount`가 없어 티켓 이름 없이 등급칩만 표시(하위 호환).
