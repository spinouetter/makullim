## 요청 (작성: Spinouetter)

모바일에서의 전반적인 UI 개선 작업 요청

- 스케줄, 통계, 시트 맵 할 것 없이 화면 아래에 여백이 없음. 그래서 마지막 셀이 바로 브라우저 UI의 컨트롤에 붙으므로 여백 부여가 필요함
- 모바일 환경에서 필요한 사항
  - 탭 버튼의 패딩 또는 마진 감소
  - 전체 화면의 좌, 우 마진 감소
  - 스케쥴 셀 내부에서의 좌, 우 마진 감소
  - 시트 맵 옵션 조정 (아이콘이 여러줄에 표시됨)
    - 1층, 2층, 3층을 1F, 2F, 3F로 조정
    - 좌석번호를 번호로 짧게 표시
- 스케줄에서 좌석, 메모를 입력할 때 시각화 부족
  - 팝업 메뉴가 뜰 때 팝업이 스크린 밖으로 나가는 경우 이를 스크린 안쪽에 생기로록 조정
- 스케줄을 스크롤할 때 배우화면에서는 날짜 확인 어려움. 아래의 세 방법은 옵션에서 설정 가능하게 해줘
  - 방법1
    - 셀을 드래그로 왼쪽으로 스크롤 시 날짜 시간 좌석을 collapse하여 플로팅으로 계속 보여줌
      - 예 (06/04 19:30 1-2-3)
  - 방법2
    - 날짜 셀을 클릭하면 셀이 하이라이팅 되어서 스크롤해도 빠른 구분이 가능
  - 방법3
    - 셀을 드래그로 가로로 스크롤 하면 세로 스크롤을 막음

## 정리 (작성: Claude)

모바일에서의 전반적인 UI 개선. 크게 5개 묶음으로 이해했습니다.

**A. 하단 여백 (전 페이지 공통)**
- 스케줄·통계·좌석맵 모두 스크롤 끝에 하단 여백이 없어 마지막 셀/콘텐츠가 모바일 브라우저 하단 UI(주소창·내비게이션 바)에 붙음.
- 원인: 레이아웃이 `100vh` 기반( [styles.css:74](css/styles.css#L74) )이라 모바일 동적 툴바 높이를 고려 못 하고, 실제 스크롤 컨테이너(`.table-scroll-wrap`, `.stats-scroll-area`, 좌석맵 `.page`)에 하단 패딩이 부족함.

**B. 모바일 간격 축소 (좁은 화면용 미디어 쿼리 신설)**
- 탭 버튼([nav button, styles.css:129](css/styles.css#L129)) 패딩/마진 축소.
- 화면 좌우 마진 축소 — `main`의 `padding:18px 20px`([styles.css:145](css/styles.css#L145)) / `nav`,`header` 좌우 패딩 20px.
- 스케줄 셀 내부 좌우 패딩 축소 — `thead th` 10px / `tbody td` 10px([styles.css:229,238](css/styles.css#L229)).
- 현재 모바일 미디어 쿼리는 `@media (max-width:520px)` 하나(티켓 카드)뿐 → 모바일 전용 블록을 새로 만들어 위 값들을 한 곳에서 조정.

**C. 좌석맵 옵션 줄바꿈 (`.seatmap-right-group`이 좁은 화면에서 여러 줄)**
- `1층/2층/3층` → `1F/2F/3F`, `좌석번호` → `번호`로 라벨 단축 (텍스트는 [index.html:76-83](index.html#L76-L83)에 하드코딩).
- 라벨 단축 + 모바일에서 버튼 패딩 축소로 한 줄에 들어가게.

**D. 스케줄 좌석/메모 입력 팝오버가 화면 밖으로 나감**
- [adjustPopoverToTable, app.js:619](js/app.js#L619)는 **세로(상하) 넘침만** 보정하고 **가로(좌우)는 보정하지 않음**. 팝오버는 `left:0`/`align-right`([styles.css:151,168,306](css/styles.css#L151))로만 배치 → 좁은 화면 우측 셀에서 가로로 잘림.
- 해당 함수에 **가로 보정(translateX)** 추가: 팝오버 우/좌가 스크롤 영역(또는 viewport)을 넘으면 안쪽으로 끌어옴.

**E. 스케줄 가로 스크롤 시 날짜 식별 (설정에서 켜고 끄는 3가지 방법)**
> 요청: 셋 다 "옵션에서 설정 가능"하게. → Settings에 토글 UI 추가 + 상태 저장([buildStateSnapshot, app.js:1972](js/app.js#L1972) / [applyState, app.js:2006](js/app.js#L2006)).
- **방법1 (플로팅 요약):** 가로로 왼쪽 스크롤하면 날짜·시간·좌석을 collapse한 플로팅 라벨로 계속 표시 (예: `06/04 19:30 1-2-3`).
- **방법2 (행 하이라이트):** 날짜 셀 클릭 시 그 행을 하이라이트 → 스크롤해도 구분 쉬움.
- **방법3 (가로 스크롤 잠금):** 가로 드래그 스크롤 중에는 세로 스크롤을 막음.

### 진행 현황

**A. 하단 여백 — 두 번 빗나간 뒤 처음부터 재구성 (이전 시도 전부 revert 후 재작성)**

- 빗나간 시도(기록용):
  - 1차: 스크롤 컨테이너 `padding-bottom` → 자식이 `<table>`/WebKit에서 스크롤 끝에 반영 안 됨.
  - 2차: `::after` 스페이서 + `env()` 사용했으나 폴백이 없어 calc 무효(별도 버그). `100dvh`도 추가했으나 기기에서 무효였던 것으로 보임.
- **실제 원인 (Spinouetter 확인: 실제 모바일 + 여백 전혀 안 생김):**
  `html, body { height:100% }`로 본문 높이가 **큰(툴바 숨김 기준) 뷰포트**로 고정 + `overflow:hidden` → 스케줄 스크롤 영역의 **바닥이 브라우저 하단 툴바 뒤에 영구히 숨겨짐.** 그래서 마지막 행도, 추가한 여백도 전부 안 보였음. `100dvh`는 해당 기기 브라우저가 미지원/오작동 시 무효.
- **해결 (재구성, `?v=4`):**
  - **JS로 실제 보이는 높이 측정** → `--app-height`에 주입 ([app.js `updateAppHeight`](js/app.js)). `window.visualViewport.height`(없으면 `innerHeight`)를 `resize`·`orientationchange`·`visualViewport.resize`마다 갱신. 모든 모바일 브라우저에서 동작하는 고전적 100vh 해법.
  - CSS: `html, body { height: var(--app-height, 100%) }`, `main { height: calc(var(--app-height, 100vh) - var(--header-h)) }` → 스크롤 영역 바닥이 **항상 보이는 화면 안**에 위치.
  - 스크롤 끝 여백: `#page-schedule .table-scroll-wrap::after`(실제 스페이서 블록, `height: calc(40px + env(safe-area-inset-bottom, 0px))`) → 끝까지 스크롤하면 마지막 행 아래 여백이 **동적으로** 나타남(스크롤 중엔 안 보임 = 요청한 동작).
  - `main` 하단 패딩 40px→0 (여백 출처를 스크롤 끝 스페이서로 일원화).
- **A 추가 수정 (Spinouetter 확인: 반영됨 + 2건 요청):** (`?v=5`)
  - 마지막 행 아래 회색 구분선 제거: `#scheduleTable tbody tr:last-child > td { border-bottom:none; }`
  - 스크롤 끝 여백 2배: `::after` 높이 `40px → 80px`.
- (보류) 통계·좌석맵·설정 페이지의 하단 여백은 A 최종 확인 후 같은 방식으로 적용 예정.

- **추가 조치(Spinouetter 요청, `?v=10`):** 배지를 버튼 **오른쪽 위→오른쪽 아래**로 이동 (`top:-7px`→`bottom:-7px`). 버튼이 위에 붙는 구조상 상단 배지는 nav 밑으로 스크롤될 때 잘릴 수밖에 없음. 아래로 옮기니 버튼 상단이 nav에 들어가도 배지는 nav 아래에 그대로 보임.
- 헤드리스 확인: 정지/스크롤(20px) 모두 배지 "1"이 잘리지 않고 표시됨(배지 top 117→137, nav와 간격 8→28px).

**D. 스케줄 팝오버 가로 화면밖 보정 — 완료** (`?v=11`, js)
- [adjustPopoverToTable](js/app.js#L619)에 가로 보정 추가: 세로(기존)+가로를 한 `translate()`로 합침. 팝오버 오른쪽이 화면(`clientWidth`)을 넘으면 왼쪽으로 끌어오고, 그 결과 왼쪽을 넘으면 안쪽으로 클램프(화면보다 넓으면 왼쪽 가장자리 8px에 맞춤). 대상: `.ticket-popover`, `.memo-popover`.
- 헤드리스 검증(390px, 셀을 오른쪽 끝에 두고 열기):
  - 메모 팝오버: width 212 → `translateX(-170)` → left170/right382 (vw390), 화면 안 ✓
  - 티켓 팝오버: width 344 → `translateX(-207)` → left38/right382, 화면 안 ✓
  - 넘치지 않는 경우엔 가로 보정 없음(transform X=0).
- (선택) 컬럼 헤더의 배역 필터 드롭다운(`.role-dropdown`)은 자체 `align-right` 로직 보유. 같은 넘침 보이면 추가 가능.

**D 후속 (Spinouetter 피드백·추가 요청) — 완료** (`?v=12`)
- **버그 원인 규명:** 티켓을 바로 누르면 저장 버튼이 안드로이드 컨트롤에 가리고, 메모를 먼저 열었다 닫고 누르면 컨트롤 위로 뜨는 차이 → 동적 하단 툴바 + `--app-height` 타이밍 문제. 메모 입력칸 포커스가 `visualViewport` 리사이즈를 유발해 `--app-height`가 실제 높이로 보정되기 때문.
