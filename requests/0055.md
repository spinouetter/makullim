# 요청 #0055 — 티켓 팝오버에서 '좌석 추가하기'로 다중 티켓 진입

상태: 진행 중 <!-- 대기 / 진행 중 / 완료 / 보류 -->

---

## 요청 (작성: Spinouetter)

다중 티켓 관리 모드를 조금 바꿀래. 티켓 입력창을 띄우면 날짜 적힌 곳 오른쪽에 오른쪽 정렬로 티켓 추가하기 버튼을 누르면 현재 티켓 내용 저장하고 다중티켓 모드로 들어가줘.

(보충) 지금의 다중 티켓 모드를 켜면 여전히 `+` 버튼이 스케줄 표에 표시된다. 다중 티켓 모드를 끄면 티켓 입력창에서 추가할 수 있게.

---

## 정리 (작성: Claude)

- 티켓 선택 팝오버(`buildTicketPopover`) 상단에 **날짜(왼쪽) + '좌석 추가하기' 버튼(오른쪽 정렬)** 행을 둔다. (버튼 문구는 후속 지시로 '티켓 추가하기'→'좌석 추가하기'로 변경.)
- 버튼을 누르면 **현재 팝오버의 티켓 내용을 맨 위 티켓으로 저장**한 뒤 **다중 티켓 관리 창(`openTicketManager`)을 연다**(거기서 2번째 티켓을 추가).
- **표시 조건**: 다중 티켓 모드(`multiTicketMode`)가 **꺼졌을 때만** 이 버튼을 노출한다. 켜져 있으면 기존처럼 스케줄 표 좌석칸의 `+` 버튼으로 관리한다(둘이 겹치지 않게).

## 할 일 (작성: Claude)

- [x] 스케줄 호출부에서 `showAddTicket: !multiTicketMode` 전달(관리창 내부 편집 팝오버는 버튼 없음)

## 결정 사항 (작성: Claude)

- **'다중티켓 모드로 들어가'** = 전역 `multiTicketMode` 설정을 켜는 게 아니라 **티켓 관리 창(overlay)을 여는 것**으로 해석. 사용자가 굳이 설정을 켜지 않고도 한 번에 2번째 티켓을 추가할 수 있게 하는 지름길. (관리창을 닫으면 티켓이 2장 이상이면 좌석칸에 개수 버튼이 남아 이후에도 관리 가능.)
- **버튼 노출은 모드 OFF일 때만**(보충 지시 반영): 모드 ON이면 좌석칸 `+`가 이미 있으므로 팝오버 버튼은 숨겨 중복을 피한다.
- 저장 동작은 기존 `저장`(tk-save)과 동일하게 맨 위 티켓 필드(ticketType/Discount/Fee/Extra/Transferred)를 기록 후, 스케줄·통계·좌석맵 갱신.
- 관리창 내부에서 티켓을 편집할 때 뜨는 팝오버(`tmEditTi`)에는 이 버튼을 넣지 않는다(이미 관리 모드 안이므로).
- 관련 원 기능: 다중 티켓 관리(요청 0025).
