# 요청 #0087 — 재관카드(Stamp) 기능: 빌리 Billy's Diary 도장판

상태: 완료

---

## 요청 (작성: Spinouetter)

<!-- 대화로 요청됨 -->

Stamp 기능을 만들거야. 개발하는 동안에는 앵커 또는 쿼리 `stamp`로 접근 가능.

- 빌리 엘리어트는 공연을 볼 때마다 스탬프를 찍어주고, 스탬프가 일정 갯수 이상이 되면 선물을 준다. 도장판은 **10칸**, 선물은 **3·7·10회** 때 각각 체크박스가 있다.
- 이 페이지에는 **닫힌 도장판과 펼쳐진 도장판**이 나열된다. 10개 다 찍힌 건 닫힘(표지), 다 안 찍힌 건 펼침.
- **자동** 버튼: 지금까지 내 공연 중 도장 안 찍은 공연을 찾아 순서대로 도장을 찍고, 10개가 다 차면 새 도장판을 만든다.
- 도장판 이름은 **Billy's Diary**, 새 도장판은 **Billy's Diary #n**(나중에 바꿀 수 있음).
- 도장판은 닫고 열 수 있다. 도장판에는 스탬프 칸과 날짜 칸이 있다. 스탬프는 **EXCELLENT / HOME COMING** 두 개. 날짜는 `7/14`, 같은 날 공연이 둘이면 `7/14-1`, `7/14-2`.
- **HOME COMING** 스탬프는 6/16–6/21 공연. **5/12–5/17**은 도장 두 개 찍어주는 날(하나가 마지막 칸이면 하나 더 만들어서 찍기).
- 열기 후 도장/날짜를 누르면 **지울지·바꿀지** 물어보는 선택지가 있다.
- **다른 도장 찍기**로 임의의 날짜로 찍을 수 있고, 이 때는 **메모 삽입** 가능.
- 도장판별 **비우기**, **전체 지우기** 버튼과 **도장판 삭제**도 만들 것.
- 기능(페이지) 이름은 **재관카드**. 표지·도장판은 **제공한 이미지를 그대로** 사용.

## 정리 (작성: Claude)

**재관카드(Stamp)** 는 공연을 볼 때마다 도장을 찍는 다이어리형 트래커다. `finale.js`처럼 **app.js 이후 로드되는 독립 모듈 `js/stamp.js`(IIFE)** 로 구현했고, 설정은 **공연 데이터(`shows/<id>/stamp.json`)** 에서 읽어(데이터 주도) 코드에는 빌리 전용 값이 없다. 사용자 상태(찍힌 도장·이름·펼침·선물체크)는 자체 localStorage 키(`makollim:stamp:v1:<id>`)에 저장한다.

- **표지/도장판은 제공 이미지 그대로** — 표지(`diary-cover.jpg`)·도장판(`diary-board.jpg`)을 `shows/<id>/images/`에 축소 저장하고, 도장판 이미지 **위에 도장·날짜·선물체크를 절대좌표(이미지 비율)로 오버레이**한다. 좌표(그리드·선물 □ 위치)는 `stamp.json`에 두어 이미지가 바뀌면 값만 조정하면 된다.
- **닫힘/펼침** — 10칸이 다 차면 자동으로 닫혀 표지로 보이고, 표지 클릭 또는 `열기`로 펼친다. 펼치면 도장판 표가 나온다.
- **자동** — `performanceData.performances` 중 `isEnded && hasSeat`(관극)인 회차를 시간순으로 훑어, 아직 안 찍은 회차(sid 기준)를 순서대로 찍는다. 마지막 도장판이 차면 새 도장판을 만든다.
- **도장 종류·개수·날짜** — 규칙은 `stamp.json`. HOME COMING은 날짜 범위(6/16–6/21) 규칙, 그 외 EXCELLENT. 더블데이(5/12–5/17)는 한 회차에 2개(칸이 넘치면 다음 도장판으로 이어짐). 날짜는 `M/D`, 같은 날 관극이 둘 이상이면 `-1/-2` 접미사.
- **편집** — 펼친 도장판에서 도장/날짜 칸을 누르면 팝오버로 **바꾸기(도장종류·날짜·메모)·지우기**. 빈 칸을 누르면 그 칸에 직접 찍기.
- **다른 도장 찍기** — 임의 날짜 + 메모로 다음 빈 칸에 찍기.
- **비우기/전체 지우기/삭제** — 도장판별 비우기(이름 유지), 전체 지우기(초기화), 도장판 삭제.
- **개발용 게이트** — 상시 탭 없이 `#stamp` 또는 `?stamp`로만 접근(정식 승격 전). app.js는 손대지 않고 stamp.js가 페이지를 직접 활성화한다.

## 결정 사항 (작성: Claude)
- **데이터 주도** — 빌리 전용 값(도장 규칙·이미지·좌표·선물 위치)은 전부 `stamp.json`. 다른 공연은 자기 `stamp.json`을 두면 코드 수정 없이 동작(없으면 기본값으로 빈 도장판만).
- **app.js 무수정** — `performanceData`가 `stamp` 키를 싣지 않으므로 설정은 `stamp.json`을 stamp.js가 직접 fetch(`window.showUrl`). 페이지 활성화도 stamp.js가 클래스 토글로 처리(숨긴 탭은 `activateTab`이 무시하기 때문). 다른 탭 클릭 시엔 기존 `activateTab`이 `#page-stamp`의 active를 정리.
- **상태는 자체 저장소** — 개발 중 기능이라 앱의 `buildStateSnapshot`/내보내기에는 아직 포함하지 않음(회귀 위험 최소화). 정식 승격 시 통합 검토.
- **선물 위치** — 이미지의 □ 위치는 행마다 달라(특히 7번은 2줄) 행별 좌표(`giftPos`)를 이미지에서 측정해 저장. 날짜는 선물 반대쪽에 배치.
- **더블데이/막칸** — 도장은 순차 배치라 더블데이 2개가 도장판 경계를 넘으면 자연히 다음 판으로 이어진다(“하나가 마지막이면 하나 더 만들어 찍기” 충족).
- **관극 판정** — 앱 공통 기준 `isEnded(p) && hasSeat(p)` 재사용. 자동은 sid로 중복 방지.
- **표시 순서** — 새로 만든 도장판이 위로 오도록 렌더는 역순(최신 먼저). 번호/이름은 생성 순서 인덱스를 유지(#2·#3 그대로).
- **자동 대상 선택 · 자동 채움 on/off** — 도장판이 여러 개일 때 자동이 채울 **기본 도장판(`autoTargetId`)** 을 도장판별 "자동 대상" 버튼으로 지정(다시 누르면 해제=최신 도장판 기본). 도장판별 **"자동 채움: 켬/끔"(`autoFill`)** 으로 자동에서 제외 가능. 대상 지정 시 자동 채움을 강제 on, 대상 도장판의 채움을 끄면 대상 지정 해제. **자동 대상이 꽉 차면 다른(자동 제외 포함) 도장판으로 넘기지 않고 새 도장판을 만들어 이어 찍는다**(새 판이 대상 승계). 지정이 없으면 자동채움 on이면서 안 찬 **최신** 도장판을 기본 대상으로. 배지(자동 대상/자동 제외)·툴바 정보에 현재 대상 표시. stamp.js ?v=3.
- **빈칸 클릭 = 공연 선택 팝오버** — 빈칸을 누르면 **안 찍은 회차(오래된 순)** 를 단추로 보여주고(가로 스크롤), 맨 왼쪽 **"임의로 넣기"**(SID 관리 안 함, 날짜+메모 자유 입력)를 sticky로 고정. 공연 단추를 누르면 그 회차(sid·날짜라벨·규칙 스탬프)를 그 칸에 찍는다. **SID로 연결된 칸**을 누르면 **지우기/바꾸기** 선택 → 바꾸기는 공연 선택 팝오버를 다시 연다(다른 회차/임의로 교체, 이전 sid는 다시 미도장으로). 임의 칸은 바로 임의 입력창(지우기 포함). 팝오버 바깥 클릭 리스너를 closePop에서 정리(팝오버 전환 시 조기 닫힘 버그 방지).
- **꽉 찬 도장판** — 자동 관련 메뉴(자동 대상·자동 채움 버튼/배지)를 숨긴다(대상이 될 수 없으므로). **닫힌(표지) 카드에는 찍힌 스탬프 개수(n/10)** 를 우상단에 표시.
- **색상** — 도장은 EXCELLENT·HOME COMING 모두 **붉은색**(테두리 포함, HOME COMING은 원형으로만 구분). 날짜는 **검은색에 가까운 볼펜색**(#1a1c22). stamp.js ?v=4.
- **공연 선택 표시 시점** — 공연 선택 팝오버는 **공연 시작 2시간 전부터** 그 회차를 보여준다(`isPickable`: 좌석 있음 + `now ≥ 시작−2h`). 자동/‘안 찍은 회차’ 집계는 기존대로 **종료된 관극**(`isEnded && hasSeat`) 기준 유지. 회차 접미사(-1/-2)는 종료 여부와 무관하게 **그 날 좌석이 있는 회차 수**로 계산해 라벨이 안정적. stamp.js ?v=5.
- **표지 이름 표시** — 캡션(이미지 아래 이름)을 없애고 표지 이미지 위에 표시. 제목이 **'Billy's Diary'** 그대로면 표기 없음(이미지에 이미 있음), **'Billy's Diary #숫자'** 형태(자동 #n이든 직접 그렇게 개명하든)면 'Billy's Diary' 옆에 **#숫자만**, 그 외 직접 지은 이름이면 스케치북 **오른쪽 아래**에 전체 이름. **찍힌 개수(n/10)** 는 스케치북 **오른쪽 위**. 카드 폭에 비례(컨테이너 쿼리 `cqi`).
- **표지 표현값은 stamp.json `cover`에서 설정(JS 하드코딩 금지)** — `cover.num`(위치 x/y·크기·기울기 rotate·색·italic)·`cover.name`(right/bottom·maxWidth·크기·기울기·색·italic)·`cover.baseTitle`(#n 패턴 기준). 숫자는 표지 스크립트 색(#a43233)·스크립트 기울기에 맞춰 기울임, 커스텀 이름은 **THE MUSICAL 색(#1b1b19)** 으로 흰 도화지 오른쪽 아래. 코드는 값을 인라인 스타일로 적용만 한다.
- **이름 변경 프리필** — 기본 이름(#n) 도장판은 rename 프롬프트를 **빈 칸**으로 열어, 그냥 확인만 눌러도 타이틀로 굳지 않는다(빈 값=기본 이름 유지).
- **다크모드 색 반전 방지** — 다이어리 카드(`.stamp-card`)에 `color-scheme:only light`. 밝은 테마가 `color-scheme:light dark`라 OS 다크모드가 고정 일러스트/도장 색을 반전(도장 핑크·날짜 흰색)시키던 것을 강제 라이트로 막는다. stamp.js ?v=8, styles.css v152.
- **이름 글꼴 · 번호 시작값** — 표지 이름/번호(`#n`) 오버레이 글꼴을 앱 본문 글꼴(**IBM Plex Sans KR**, `font-family:inherit`)로. 도장판 번호는 **첫 판부터 #1**. stamp.js ?v=10, styles.css v153.
- **명시적 이름 체계(기본 이름 유지 폐지)** — 이름을 동적으로 만들지 않고 **항상 저장**한다. 새 도장판은 `newBoard`가 **"기본제목 #n"** 을 명시 부여(`n` = 현재 `기본제목 #n`들의 **최대 숫자 +1**, `nextBoardNumber`). 옛 빈 이름 도장판은 로드 시 `migrateNames`로 명시 이름 부여. 이름 변경은 **빈 값 불가**(현재 이름 유지). 표시는 그대로 `기본제목 #숫자` 패턴이면 `#숫자`만, 아니면 커스텀 이름. stamp.js ?v=11.
