## 요청 (작성: Spinouetter)

티켓 선택 창의 UI 변경

- 선택 해제를 삭제로 변경
  - 선택 해제와 저장 사이에 취소 추가
- 티켓은 할인율이 높은 순, 가나다 순으로 정렬하여 표시함
  - 다만 아래 순서는 별도로 관리함 (JSON에 정렬에 관한 별도의 key 부여)
    - 위에서 부터 - 정가, 조기예매, 재관할 (0, 1, 2)
    - 가장 마지막 - 초대권 (-1)
- 기타 비용을 입력할 수 있는 칸 추가 (예를 들자면 취소 수수료)
  - 최종 비용은 티켓 가격 + 수수료 + 취소 수수료

## 정리 (작성: Claude)

티켓 팝오버([buildTicketPopover, app.js:149](js/app.js#L149)) 4가지 변경:

1. **버튼: [선택 해제][저장] → [삭제][취소][저장]**
   - 삭제 = 기존 '선택 해제'(티켓·수수료·할인·기타비용 초기화).
   - 취소 = 변경 없이 팝오버만 닫음(신규).
   - 저장 = 기존과 동일.
2. **티켓 정렬** — 적용 가능한 티켓 목록을 정렬:
   - 위 고정: `sort` 키 ≥0 오름차순 (정가0·조기예매1·재관람2). ※요청의 "재관할"은 데이터의 **재관람** 오타로 간주.
   - 가운데: `sort` 키 없는 항목 = 할인율 내림차순, 같으면 이름 가나다.
   - 맨 아래: `sort` 키 <0 (초대권 -1).
   - grades.json prices에 `sort` 키 추가(정가0·조기예매1·재관람2·초대권-1), 코드에서 읽어 정렬.
3. **기타 비용 칸 추가** — 공연별 `ticketExtra`(원). 팝오버에 입력칸, 최종 가격 = 티켓가 + 수수료 + 기타비용. `ticketPriceOf`에 반영(가격 컬럼·통계 합계 모두).

## 결정 사항 (작성: Claude)

- "재관할" → **재관람**(데이터 실제 이름)으로 처리.
- `ticketExtra`는 메인 설정 저장(localStorage·설정 내보내기)에는 포함하되, **좌석 JSON 내보내기/가져오기 포맷에는 미포함**(문서화된 포맷 유지). 좌석 JSON 가져오기 시 기타비용은 0으로 초기화됨.
- 삭제 버튼은 파괴적이므로 빨강 계열 색으로 구분.
- 기타 비용은 티켓 종류가 선택돼야 최종가에 반영(ticketPriceOf는 type 없으면 null).
