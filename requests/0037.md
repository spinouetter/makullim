## 요청 (작성: Spinouetter)

- 시트맵의 색(등급색·관극 횟수 히트맵)을 사용자가 직접 지정하는 기능.
- **마스터 토글**로 켜고 끔 — 끄면 현재 **테마의 기본 색** 사용.
- 토글을 꺼도 설정한 값은 **그대로 유지**(초기화 개념 없음).
- **이름 붙인 프리셋**으로 저장. 프리셋을 수정하면 이름이 `이름 (수정됨)`으로 바뀌고 **덮어쓰기 저장** 또는 **새 이름으로 저장** 가능.
- **히트맵**도 추가. 히트맵 **max값(최대 10)** 설정. 숫자를 바꾸면 max를 넘는 단계는 **숨김(설정값은 보관)**. max보다 크게 관극한 자리는 **max색으로 칠함**(기존 규칙과 동일).
- **프리뷰**를 팝업 레이아웃으로 표시. **내자리**·**랜덤 생성** 버튼 둘.

### 결정 사항(확정)
1. 테마를 바꿔도 커스텀 색 **유지**.
2. 등급색 프리셋과 히트맵 프리셋은 **분리**된 독립 목록.
3. 전역 "설정 초기화"는 프리셋을 **보존**(사용자 데이터) — 토글만 끔.
4. "새 이름으로 저장"은 `prompt()`로 이름 입력.

## 정리 (작성: Claude)

### 개념 모델
- **작업 세트(working)** — 지금 화면에 적용 중인 편집 상태. 토글 ON이면 시트맵에 반영.
- **프리셋** — 이름 붙은 저장본. 등급/히트맵 **각각 독립 목록**. 작업 세트는 특정 프리셋에서 파생(`baseId`), 편집하면 `dirty=true`.
- 마스터 토글 하나로 등급·히트맵 커스텀을 동시에 켜고 끔.

### 색이 수렴하는 지점
- 등급색: CSS 변수 `--g-vip/--g-r/--g-s/--g-a/--g-none`(테마별 정의). 커스텀 ON = `documentElement`에 인라인 주입(테마 캐스케이드보다 우선), OFF = 인라인 제거.
- 히트맵: `countHeatColor(count)`가 작업 세트의 `heat[]`·`heatMax` 참조(ON/프리뷰), 아니면 표준 상수 `COUNT_HEAT`.

### 데이터 모델 (state)
```
seatColorEnabled         : bool
seatGradeWork  : { baseId, dirty, grades:{vip,r,s,a,none} }
seatGradePresets : [ { id, name, grades } ]
seatHeatWork   : { baseId, dirty, heat:[…10], heatMax }
seatHeatPresets  : [ { id, name, heat:[…10], heatMax } ]
```
- `buildStateSnapshot`/`applyState`에 추가 → localStorage·내보내기/가져오기 자동 포함.
- 히트색은 항상 10개 보관, `heatMax`만큼만 표시/사용.

### UI (설정 "컬러 테마" 아래)
- 마스터 토글 + 등급 프리셋(선택·저장·새이름) + 등급 색 5개.
- 히트맵 프리셋(선택·저장·새이름) + max 선택 + 히트 색(1..max 표시).
- **프리뷰 열기** → 팝업(시트맵 렌더 + 내자리/랜덤 생성).

### 프리뷰
- 팝업에서 작업 세트 색을 강제 적용해 렌더(토글 OFF여도 미리보기 가능).
- 내자리 = 실제 관극수(`seatMapCount`), 랜덤 생성 = 시드 랜덤 관극수로 히트맵 전 구간 확인.

## 결정 사항 (작성: Claude)
- **단일 마스터 토글** — 원 요구("이 기능은 토글로") 유지. 등급/히트맵 프리셋만 분리.
- **초기화 없음** — 토글 OFF는 값 보존. 전역 설정 초기화도 프리셋 유지.
- **히트 색 10개 상시 보관** — max는 표시/클램프 상한만 조절(넘는 단계 숨김·보관).
