## 요청 (작성: Spinouetter)

DROPBOX 설정 백업 및 불러오기

- Github Actions를 사용하여 Secret에 저장된 DROPBOX_APP_KEY를 사용한다.
  - 이를 위한 action 스크립트를 생성한다.
  - 혹시 위 키가 없으면 백업 기능을 끈다.
  - Github actions를 사용하지 않는 로컬 개발 환경에서 키를 전달 하는 방법을 만든다.

- 설정 내보내기 / 가져오기 아래에 Dropbox에 스케쥴 백업하기 가져오기를 추가한다.
  - json으로 내보내기와 동일한 내용을 dropbox에 저장한다.
    - 파일명은 <meta.json의id>-YYMMDD-HHMM.json이다.
  - 서버에서 파일목록 가져오기 버튼을 만들고 이를 누르면 파일 목록이 드랍다운에 표시된다.
    - 날짜와 시간 (경과시간) 이 목록에 뜬다.
    - 경과시간은 1시간 이내는 분, 하루 이내는 시간, 한 달 이내는 일, 1년 이내는 월, 1년 이후는 년이 뜬다.
    - 설정 덮어쓰기라는 체크박스가 있어서 (기본 미체크) 체크 되어 있을 경우 설정을 업데이트하고, 그 이외에는 스케줄을 덮어쓴다. 덮어쓴다는 의미는 기존 스케줄을 지우고 새로운 스케줄로 덮어쓴다는 것이다.
    - 이름에 (개발중) 이라는 내용을 덮어쓴다. 테스트 이후 문제 없으면 지울 것이다.

- 백업 파일 충돌을 막기 위해 아래 두가지 작업을 추가한다.
  - 로컬 스토리지에 최근 1000개의 변경사항을 트래킹한다.
    - timestamp와 동작, 변경된 컨텐츠를 기록한다.
  - 이와 별도로 마지막으로 스케줄을 업데이트한 timestamp를 기록한다.

- 백업 파일을 가져올 시 현재 세션의 last_updated와 백업의 timestamp를 비교하여 가져올지 물어보는 팝업을 표시한다.
- 현재 세션의 히스토리와 백업 파일의 히스토리를 비교하여, 가장 최근에 일치하는 변경사항을 찾는다.
  - 현재 새션의 마지막 변경이 백업에 존재하면 바로 가져온다.
  - 현재 새션과 백업이 마지막 변경 이후에 차이가 있다면 이를 읽어올지 물어보는 창을 표시한다.
    - 첫 번째 줄에는 공통으로 존재하는 변경사항을 표시한다.
    - 이후 네 개의 컬럼으로 된 테이블이 나온다.
    - 왼쪽에는 현재 세션의 변경사항이 표시된다 (시간, 내용)
    - 오른쪽에는 백업 파일을 변경사항이 표시된다.

## 정리 (작성: Claude)
- **이번 범위 = 단계 1(백업/복원 핵심)** (사용자 확정). 변경이력 1000개 추적·last_updated·히스토리 충돌 비교(4컬럼 표)는 **다음 단계로 분리**.
- 연동: **백엔드 없는 정적 사이트 → 브라우저 OAuth2 PKCE**(앱 시크릿 불필요, App key만). App key는 빌드시 주입(공개값). 키 없으면 백업 기능 비활성.
- '데이터 내보내기/가져오기' 아래에 **Dropbox 백업 (개발중)** 섹션: 연결 → 백업하기 / 파일목록 가져오기(날짜+경과시간) / 가져오기 + '설정 덮어쓰기' 체크박스.
- 백업 내용 = 데이터 내보내기와 동일(`buildExportPayload`). 파일명 `<meta.id>-YYMMDD-HHMM.json`. 경과시간: 1시간내 분 / 하루내 시간 / 한달내 일 / 1년내 월 / 이후 년.
- '설정 덮어쓰기' 체크 → 화면 설정까지 덮어씀, 미체크(기본) → 좌석/티켓/메모(스케줄)만 덮어씀.

## 결정 사항 (작성: Claude)
- App key(공개)만 사용하는 PKCE → 정적 사이트에서 안전. 토큰은 브라우저 localStorage 보관. `token_access_type=offline`로 refresh token 발급(만료 자동 갱신).
- Dropbox 앱은 **App folder 권한** 가정(목록 `path:""`=앱 폴더 루트, 업로드 `/<파일>`). Full Dropbox 권한이면 경로 조정 필요.
- 워크플로 heredoc 들여쓰기 문제 회피 위해 `sed` 치환 사용(App key는 영숫자라 안전).

### 실 환경 설정 안내 (사용자 작업)
1. **Dropbox 앱 콘솔**: OAuth2 Redirect URI에 배포 주소와 로컬 주소를 등록 — `https://makollim.com/` , `http://localhost:8000/` (앱 페이지 경로와 정확히 일치해야 함). 권한(Permissions)에서 `files.content.write`, `files.content.read` 켜기.
2. **GitHub 저장소 Secret**: `DROPBOX_APP_KEY` = Dropbox 앱의 App key(공개값).
3. **GitHub Pages 소스**: Settings → Pages → Source = **GitHub Actions** (현재 브랜치 배포 → Actions 배포로 전환). CNAME(makollim.com) 유지됨.
4. **로컬 개발**: 콘솔에서 `localStorage.setItem('makollim:dropboxAppKey','<APP_KEY>')` 후 새로고침하면 '연결' 버튼이 보임.
