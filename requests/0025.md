## 요청 (작성: Spinouetter)

다중 티켓 관리 모드

- 세팅에 "다중 티켓 관리 모드"을 추가하거나 끌 수 있는 기능 추가. 기본 값 끄기.

모드가 켜졌을 때 동작

- 입력 창과 눈알 사이에 + 버튼 추가
  - + 버튼은 티켓을 하나도 입력하지 않을 때는 비활성화
  - 티켓을 하나 입력하면 + 버튼이 나와서 티켓을 관리하는 UI가 출연함

- 티켓 관리 UI 동작 방식
  - 타이틀은 날짜 
  - 하나의 열에는 좌석, 눈, 티켓, 가격 (스케쥴 화면과 동일) 표시
  - 티켓은 드래그로 순서 변경 가능
    - 스케쥴 표에 적용되는 것은 가장 위의 티켓
  - 마지막 열에는 + 버튼 추가

- + 버튼 눌렀을 때의 동작
  - 티켓 입력 팝오버가 떠서 새로운 티켓 입력가능.
    - 겹치는 위치는 에러로 입력 불가
    - 삭제는 티켓을 다시 눌러서 나오는 팝오버에서 삭제 버튼 클릭하여 가능

- 티켓이 증가했을 때 표현 방식
  - 티켓이 2장 이상 있을 때, + 버튼 위에 티켓 숫자가 표현됨
  - 스케쥴 표의 가격은 가장 위의 티켓만 반영됨

- 다수의 티켓이 있을 때 다중 티켓 관리 모드를 끌 때의 동작
  - 다수의 티켓이 있을 때에는 경고 메시지 후 끔
  - 데이터를 바꾸진 않음, 다만 UI만 바뀜
  - 두 장 이상의 티켓이 있는 경우 좌석입력 텍스트 입력의 오른쪽 위에 점 추가
  - 만약 좌석을 삭제했을 때 두 개 이상의 티켓이 있었을 경우 가장 위의 티켓이 스케쥴 표에 표시됨

## 정리 (작성: Claude)
- 설정에 "다중 티켓 관리 모드" 토글(기본 끔).
- 모드 ON: 좌석 입력칸과 눈 아이콘 사이에 `+` 버튼. 티켓 0개면 비활성, 1개 이상이면 활성 → 누르면 티켓 관리 UI.
- 관리 UI: 제목=날짜, 각 행 = 좌석·눈·티켓·가격(스케줄과 동일), 드래그로 순서변경(맨 위가 스케줄에 적용), 마지막 행에 `+`.
- `+` → 티켓 입력 팝오버(새 티켓). 같은 좌석 중복이면 에러. 삭제는 티켓 클릭→팝오버 삭제버튼.
- 티켓 2장 이상이면 `+` 위에 개수 배지. 스케줄 가격은 맨 위 티켓만.
- 모드 OFF 전환: 티켓 여러 장이면 경고 후 끔(데이터 유지, UI만). 2장 이상이면 좌석 입력칸 오른쪽 위에 점. 좌석 삭제 시 맨 위 티켓이 스케줄에 표시.

### 구현 결정(확인용)
- 데이터: 공연마다 `tickets`(배열). 각 티켓 = {seat, ticketType, ticketFee, ticketDiscount, ticketExtra, ticketTransferred}. **맨 위(tickets[0])가 기존 평면 필드(p.seat 등)에 미러링**되어 스케줄·시트맵·통계는 그대로 맨 위 티켓을 사용(리플 최소화). 메모(note)는 공연 단위 유지.
- 레거시(배열 없음): 기존 좌석/티켓이 있으면 tickets 1개로 변환.
