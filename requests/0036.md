## 요청 (작성: Spinouetter)

- Finale 정산판을 **스크립트와 데이터 완전 분리**로 구조화. 코드에 박힌 "배경/데이터/매핑"을 JSON으로.
- **여러 관극 기록판**을 등록·전환할 수 있게: 레지스트리 JSON + 보드 정의 JSON. 개발중·버그 보드는 숨김.
- 슬롯(배우 이름·관극수·사진)·그룹(발레·앙상블)·타운합계를 **casts.json에서 자동 파생**(중복 선언 제거). casts에 id 필드 추가(소문자 snake).
- 스탯은 정산 통계표(app.js)와 **같은 계산을 공유**.
- 유니크 요소(seatmap·total·subtitle·headings)는 **명시적**으로 존재. bindings처럼 "어디 숨었는지 모르게" 하지 말 것.
- 정산판에서 특정 배역·커버를 **제외(exclude)**, 배치 못한 배우는 **경고/무시 옵션**.

## 정리 (작성: Claude)

### 파일 구조
- `json/finale-boards.json` — 레지스트리: `default`·보드 목록(`id`·`name`·`def`·`hidden`).
- `json/finale-boards/<id>.json` — 보드 정의(순수 렌더 스펙, id/name 없음). `?board=<id>`로 선택(숨김 보드도 직접 오픈).

### 보드 정의(매니페스트) 섹션
- `background`(svg/image·viewBox 크롭), `fonts`(woff2+ttf 임베드), `styles`(camelCase=SVG 속성명), `photos`(경로 pattern·placeholder).
- `slotTemplate.fields` — 슬롯 필드 테이블(text/count/photo), `{slot}`·`{i}` 치환. `groupCount` — 그룹 actor 총계 `fn-group-{actor}`(구조적).
- `exclude`(타입별: role/cover/actor) + `onUnplaced`(warn/ignore).
- **명시적 싱글톤**: `totalCount`·`subtitle`·`headings`·`seatmap`(값은 암묵 — 엔진이 grandTotal·periodVenue 등 계산). **`bindings` 배열 없음.**

### 엔진(finale.js)
- `loadRegistry`/`resolveEntry`/`loadManifest` — 레지스트리→활성 보드.
- `castSlotGroups(casts)` — casts 순회로 슬롯 자동 파생: 주연=id, 발레=팀 actor별(byTeam)+어른(byTeam 없는 members), 앙상블=members.
- 슬롯 통계는 **app.js `computeRoleActorStats(role, mode)` 재사용**(단일 소스). 그룹 멤버는 개별 통계 없음(관극수 슬롯 자체가 없음; 그룹 값=전 공연).
- 빈/제외 슬롯 = 숨김(흰색). 역할/커버 exclude 반영.

### casts.json
- 각 역할·발레팀에 lowercase_snake `id`(billy·mrs_wilkinson·ballet_girls·ashington …). SVG 슬롯 토큰과 일치 → 주연·그룹 자동 파생.

### app.js
- `renderStats()`의 인라인 배우 집계를 `computeRoleActorStats(role, mode)`로 추출 → 통계표·Finale 공유.

## 결정 사항 (작성: Claude)
- **id = lowercase_snake, SVG 슬롯 토큰과 동일** — 주연은 casts에서 매핑 없이 파생. 공백 있는 역할(Mrs. 윌킨슨)은 슬러그 불가피해 id로 해결.
- **값 계산은 코드, 매핑은 JSON** — 통계모드·byTeam 판정 등은 코드; "무슨 값을 어디에"만 JSON.
- **그룹 멤버는 개별 통계 없음** — 발레/앙상블 슬롯에 관극수 요소가 없음. 그룹 actors(팀)=타운 합계만, 그룹 값=전 공연=total.
- **발레 섹션 배정 = casts byTeam(데이터)** — 기존 공연기반 휴리스틱 제거(문서화된 의도와 일치).
- **빈/제외 슬롯 = 숨김(흰색)** — 사진 없는 실제 배우는 로고 유지(이름 있음).
- **bindings 배열 폐기** — 구조적으로 처리 가능(그룹총계)은 slotTemplate, 유니크 싱글톤은 이름 있는 top-level 키로 명시.
- **앙상블 순서 = casts 선언 순서**(구 관극수 정렬 대체 — 그룹 스탯 제거로).
- 리팩토링 전 구간 **렌더 픽셀 동일** 유지(앙상블 순서만 의도적 변경).
- 파킹(훗날): 이름→id 전환(byTeam 키·casting_by_date 등), `roster`→`cast` 용어.
