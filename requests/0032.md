## 요청 (작성: Spinouetter)

- 지난 시즌 캐스트보드 SVG(`temp/billy(1).svg`)를 분석해 Finale 페이지를 그 디자인(빌리 보드)으로 바꾼다.
- 배역 슬롯에 현재 casts.json 캐스트를 채우되 **슬롯>캐스트면 `NAME` 플레이스홀더, 슬롯<캐스트면 위에서부터**.
- 배역 영문↔한글 매핑은 지정한 표대로. 관극수는 현재 캐스트 기준으로 통계 계산.
- 사진은 원본 `<image>` 위치 그대로 두고 **플레이스홀더**로 교체(절대좌표 재계산 금지, 계층 요소는 살리고 내용만 변경).
- 이미지 핀치/터치 줌, '숫자 초기화' 버튼 제거, 대신 통계 4옵션(최초/공연시작/배역변경/비중) 선택에 따라 숫자 변경.
- 좌석표: 기존 STAGE·작은 박스·범례 제거하고 실제 좌석 다이어그램을 올린다. VIP=기존 좌석색, R·S·A는 회색 램프. 관극 횟수는 시트맵과 같은 1~10 색, 범례는 하단·숫자 아래.

## 정리 (작성: Claude)
- **템플릿**: `temp/billy(1).svg`(Visio, ~343MB, PNG 61장)에서 배우명·관극수 `<text>`에 `fn-name-{SLUG}-{i}`·`fn-cnt-{SLUG}-{i}` id를 부여하고, 원본 `<image>`는 위치 유지·href만 `photo-placeholder.svg`로 교체해 `images/finale-board.svg`(~465KB) 생성. 각 배역은 서로 다른 depth-2 그룹 → 그룹+x순서로 슬롯 매핑(절대좌표 비사용).
- **finale.js 전면 개편**: 보드 fetch → 슬롯 id `textContent`만 변경으로 현재 캐스트 채움(NAME/위에서부터 규칙), 통계 4모드(`getCastContributions`)로 분모·분자 계산, 발레/앙상블은 casting_by_date 집계. 좌석 영역은 `#de6363` 패널로 덮고 실제 좌석 다이어그램(등급 회색 램프·관극 COUNT_HEAT 1~10·하단 범례·STAGE·테두리) 주입. 핀치/휠/드래그 줌. SVG/PNG/JPG/PDF 저장 유지.
- 산출물: `images/logo.svg`(벡터, 날짜·장소 편집 가능), `images/photo-placeholder.svg`(4:5), `images/finale-board.svg`, `images/finale-board.meta.json`.
