# 요청 #0033 — Finale 배우 사진 적용(이름 매칭·축소 크롭·전체 표시·상단 정렬)

## 요청 (작성: Spinouetter)

- images 디렉토리에 배우 사진을 넣었으니 Finale 만들 때 이 사진들을 가져다 쓴다.
- 사진 비율이 틀과 정확히 안 맞으니 **가득 채우고(축소 후 크롭)** 넘치는 부분은 잘라 표시.
  - 세로가 남으면 가로를 틀에 맞춰 축소→상하 크롭 / 가로가 남으면 세로를 틀에 맞춰 축소→좌우 크롭 (= cover).
- 사진 없으면 `플레이스홀더.jpeg` 사용.
- 사진 용량·크기를 먼저 줄인다.
- 원본 SVG가 큰 이미지에서 일부만 잘라 보여주던 구조라면 그 잔재를 제거하고 **사진 전체**를 표시.
- 가운데 정렬이 아니라 **위(상단) 정렬**.

---

## 정리 (작성: Claude)
- 사진은 **배우 이름 파일명**(`images/<이름>.jpeg`). finale.js가 각 슬롯을 채울 때 그 배우 사진을 지정, 로드 실패(없는 배우)면 `플레이스홀더.jpeg`로 폴백(onerror).
- 보드 `<image>` 63개에 `fn-photo-{배역}-{i}` id 부여(배역 그룹+x순서로 fn-name과 1:1) + `preserveAspectRatio="xMidYMin slice"`(cover·상단 정렬).
- **잔재 제거**: 원본은 이미지를 clip 창(4:5, 65×81/55×69)보다 ~2.1배 크게 배치해 얼굴 일부만 노출하던 구조 → 각 이미지 박스를 자기 **clip 창 크기·위치에 맞춰** 사진 전체가 프레임에 cover로 들어가게 함.
- 사진 축소: 최대 변 680px·EXIF 회전 보정·품질 82 → **302MB→1.0MB**(장당 24~43KB, ~440×680). 원본은 별도 백업.

## 결정 사항 (작성: Claude)
- cover = SVG `preserveAspectRatio="…slice"` + 기존 `clip-path`로 프레임 밖 확실히 크롭.
- 상단 정렬 = `xMidYMin`(가로 가운데·세로 위) → 인물 머리 유지·아래 크롭.
- 사진 존재 여부는 onerror 폴백으로 처리(정적 사이트라 파일 목록을 못 읽음). 콘솔 404는 폴백 트리거(무해). 필요 시 manifest로 제거 가능.
- 사진 원본(수 MB~20MB)은 저장소에 부적합 → 표시 크기에 맞춰 축소본만 커밋.
