## 정리 (작성: Claude)
- 기존 "공연 좌석·티켓·메모 JSON 내보내기/가져오기" 섹션을 **구 버전(레거시) 방식**으로 재정의.
- **내보내기는 좌석만**(`{sid: "좌석"}`) — 티켓·메모·할인·양도·기타비용은 안 내보냄. **가져오기는 그대로**(기존 배열 형식·문자열 모두 인식, 엑셀 따옴표 처리).
- 섹션 제목 → "구 버전 방식의 JSON 내보내기 / 가져오기".
- 설명 교체: 엑셀 정산판처럼 좌석만 저장/읽기, 전체 기능은 '설정 내보내기/가져오기' 사용, 엑셀 복사 시 불필요한 따옴표도 처리. (복잡한 서식 설명 삭제)

## 할 일 (작성: Claude)

- [x] 섹션 제목·설명 텍스트 교체, 복잡한 형식 안내 삭제, 붙여넣기 placeholder를 좌석 전용 예시로
- [x] 가져오기 따옴표 견고화: 좌석 값 앞뒤 잔여 따옴표 제거(parseLooseJSON의 텍스트 단위 처리에 더해 값 단위도)
- [x] 헤드리스 확인(좌석만 내보냄·따옴표 섞인 텍스트 가져오기 정상, JS 에러 없음)

## 결정 사항 (작성: Claude)
- `buildSeatExportJSON`(티켓·메모 포함)은 **설정 내보내기 전용으로 유지**하고, 레거시 좌석 내보내기는 별도 `buildSeatOnlyJSON` 사용 → 두 기능이 서로 영향 없음.
- 가져오기 코드는 기존 배열 형식도 계속 인식(하위 호환). 새 내보내기 파일은 `{sid:"좌석"}` 단순 형식.
