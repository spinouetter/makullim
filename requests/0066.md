## 요청 (작성: Spinouetter)

<!-- 대화로 전달된 요청을 Claude가 옮겨 적음 -->

- (아티팩트 시안 검토 후) 새 디자인 시안을 실제 앱에 반영해줘.
- 시안 요구: 컴팩트하게, 새 할인권 입력 유지, 기타 입력에 "할인 −" 가이드,
  티켓 목록 스크롤, 가격 구성 요약, 선택 티켓 강조, 티켓 관리 그룹 분리, 액션 위계.

---

## 정리 (작성: Claude)

- 티켓 선택 팝오버(`buildTicketPopover`)를 시안대로 재구성. 기능·핸들러는 유지하고 구성/스타일만 정리.
  - **제목**: 등급칩(등급 첫 글자·등급색) + "○○석 티켓 선택" + 예매처(오른쪽).
  - **선택 강조**: 현재 선택된 티켓 옵션에 골드 배경·테두리(`.ticket-option.sel`).
  - **가격 구성 요약(신규)**: 티켓가 · 수수료 · 자체할인(음수 초록) · **총금액**. 티켓/수수료/기타/직접입력을
    바꾸면 **즉시 갱신**(`ticketCostSummaryHtml` + `refreshTicketCostSummary`, 좌석 기준 계산).
  - **기타 입력 가이드**: 빈 칸 placeholder "할인 −"로 음수(자체할인) 입력 유도.
  - **티켓 관리 그룹**: 숨김·취소·양도하기·이력을 "티켓 관리" 라벨 아래로 분리.
  - **액션 위계**: 저장=골드 채움(주동작), 삭제=위험(빨강 외곽), 하단 취소→**닫기**(라이프사이클 취소와 구분).
  - 컴팩트: 옵션 행·여백 정리.
- 다중 티켓 관리 창의 팝오버(`opts.lifecycle` 없음)에도 가격 요약이 붙되, **그 티켓의 좌석 기준**으로
  계산·실시간 갱신(맨 위 티켓과 좌석이 다를 수 있음).

## 결정 사항 (작성: Claude)

- **핸들러·클래스 유지, 구성만 재배치** — 기존 저장/삭제/취소/양도/이력/직접입력 로직이 의존하는
  클래스·data 속성을 그대로 두어 회귀 위험 최소화.
- **가격 요약은 좌석 기준 계산** — `ticketPriceOf(seat, …)`. 관리창은 편집 중 티켓의 좌석을 우선 사용
  (`t.seat != null ? t.seat : perf.seat`).
- **하단 "취소" → "닫기"** — 라이프사이클의 "취소"(티켓 취소)와 팝오버 닫기를 구분해 혼동 제거.
- 시안의 "좌석 추가하기"는 "＋ 좌석 추가"로 축약(동작 동일).
