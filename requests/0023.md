# 요청 #0023 — 데스크탑 스케줄 표 마우스 드래그 스크롤

상태: 완료 <!-- 대기 / 진행 중 / 완료 / 보류 -->

---

## 요청 (작성: Spinouetter)

데스크탑 환경에서도 스케쥴 표를 마우스 드래그 앤 드롭으로 움직이도록 변경

---

## 정리 (작성: Claude)
- 데스크탑에서 스케줄 표(`.table-scroll-wrap`)를 마우스로 잡아끌어 가로·세로 스크롤.

## 할 일 (작성: Claude)
- [x] `setupScheduleScrollLock`에 마우스(pointerType=mouse) 드래그 스크롤 추가
- [x] 입력/버튼/링크/드롭다운/팝오버 위에서는 드래그 미시작(클릭·입력 보존)
- [x] 4px 이상 움직일 때만 드래그로 전환, 드래그 직후 클릭(하이라이트 등) 1회 무시
- [x] 커서 grab/grabbing + 드래그 중 user-select 차단
- [x] 헤드리스 확인(빈 셀 드래그 → 스크롤, seat-input에서는 드래그 안 됨)
- [x] (보완) 마우스 드래그도 '세로 스크롤 잠금' 옵션 반영 — 시작 방향으로 한 축만 이동(가로 우세 시 세로 잠금)

## 결정 사항 (작성: Claude)
- 터치(모바일)는 기존 핸들러 유지, 데스크탑은 pointer 이벤트로 별도 처리(중복 없음).
- pointerdown에서 즉시 캡처하지 않고 임계값(4px) 초과 시 캡처 → 짧은 클릭은 정상 동작.
