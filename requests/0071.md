# 요청 #0071 — 티켓 관리 창 예매일 설정 · 이력 기록 편집

상태: 진행 중 <!-- 대기 / 진행 중 / 완료 / 보류 -->

---

## 요청 (작성: Spinouetter)

두 가지 수정

* 티켓 관리 창에서 티켓을 선택했을 때 예매일 설정가능하게
* 이력 창에서 티켓 눌러서 선택했을 때 예매일, 좌석, 등급, 양도취소 여부, 메모 수정할 수 있게

---

## 정리 (작성: Claude)

- (1) **다중 티켓 관리 창**의 티켓 팝오버에서 **예매일**을 설정할 수 있게 → 추가 티켓도 예매일을 가짐(취소료·데드라인 계산 가능).
- (2) **티켓 이력 창**에서 기록의 티켓(알약)을 누르면 **예매일·좌석·등급·취소/양도 여부·메모**를 편집.

## 할 일 (작성: Claude)
- [x] 모델: 모든 티켓이 `bookingDate` 보유 — topTicketObj·setTickets·applyPerfState(extraTickets)·스냅샷.
- [x] buildTicketPopover에 `opts.bookingField` → 예매일 date 입력. parseTicketPopover가 있을 때만 bookingDate 반영.
- [x] 관리 창은 `buildTicketPopover(..., {bookingField: devFeatOn})` — 저장 시 tmCommit→setTickets로 각 티켓에 예매일 저장.
- [x] 이력 창: 알약 클릭 → 인라인 편집 폼(좌석+등급 select, 예매일+취소/양도 select, 메모, 삭제/취소/저장). 좌석이 등록 좌석이면 등급 자동.
- [x] CSS(.tk-booking-row·.th-edit 등), 캐시버스터 app.js v185·styles.css v129. 헤드리스 검증(관리창 예매일 저장·이력 편집 반영).

## 후속 (작성: Spinouetter/Claude)
- 이력 편집을 **관리 창과 동일한 전체 티켓 팝오버**로 띄우고 추가 정보까지 표시·수정.
- 반영: `buildTicketPopover`에 `historyEdit` 모드(좌석 입력 + 취소/양도·취소수수료·메모). 이력 창은 인라인 폼 대신
  `tm-pop-overlay` 오버레이로 이 팝오버를 띄움 → 좌석·티켓종류·수수료·기타·예매일 + 취소/양도·수수료·메모 편집.
  좌석/종류 변경 시 현재 값 커밋 후 재렌더(등급·라벨 갱신). app.js v186·styles.css v130.

## 결정 사항 (작성: Claude)
- 관리 창 예매일 필드는 **devFeatOn(개발 중 기능)일 때만** 노출(취소료 기능의 일부).
- 이력 편집은 일반 기능(이력 창 자체가 devFeat와 무관하게 접근 가능)이라 게이팅하지 않음.
- 이력 편집에서 좌석을 등록 좌석으로 바꾸면 등급은 좌석에서 자동(gradeOf), 미등록이면 등급 select 값 사용. 비용/티켓종류/금액은 이 폼에서 건드리지 않음.
