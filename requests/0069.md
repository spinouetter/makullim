## 요청 (작성: Spinouetter)

<!-- 대화로 전달된 요청을 Claude가 옮겨 적음 -->

- 취소료 계산기를 만든다.
  - 티켓 입력창 어딘가에 **시계 아이콘** 표시. 기본은 비활성 상태.
  - 아이콘을 누르면 **날짜 입력창 + 취소료 계산 창**이 뜬다.
  - 저장하고 닫으면 **예매한 날짜가 저장**되고 시계 아이콘이 활성화된다.
- 취소 수수료 계산(예매처마다 다름):
  - 당일 0원
  - 예매 후 10일 내 & 공연 10일+ 전 → 수수료
  - 예매 후 10일 후 & 공연 10일+ 전 → 수수료 + 4000
  - 공연 10일 내 → 수수료 + 티켓×0.10
  - 공연 7일 내 → 수수료 + 티켓×0.20
  - 공연 3일 내 → 수수료 + 티켓×0.30
  - 공연 당일 → 수수료 + 티켓×0.80
- 정정: **+4000은 예매처마다 다름(최소 수수료로 생각)**. 예) 10,000원 티켓을 공연 3일 내
  취소하면 3,000+2,000이 아니라 **4,000+2,000**을 낸다(위약금이 최소수수료보다 작으면 최소수수료).

### 확인된 결정(대화)
- 기본 수수료·최소 수수료는 **`resellers.json`에 예매처별로 저장**.
- 저장 시 **예매일을 저장**하고, 계산된 취소료를 **기존 '취소' 폼에 자동 반영**.

## 정리 (작성: Claude)

- **데이터 모델**: 예매처 정책을 두 값으로 표현(예매처마다 다름).
  - `cancelBaseFee`(기본 수수료): 예매 당일 취소를 제외하면 항상 부과.
  - `cancelMinFee`(최소 수수료): 취소 위약금의 **하한**.
  - 위약금 = `max(티켓정가 × 구간%, 최소수수료)`, 총 취소료 = `기본 + 위약금`.
- **계산 규칙**(`computeCancelFee`, 오늘=KST):
  - 예매 당일(예매일==오늘) → **0원**(전액 무료, 다른 조건보다 우선).
  - 청약철회 구간(예매 10일 내 & 공연 10일+ 전) → 위약금 0, **기본수수료만**.
  - 그 외 공연 근접도로 %: 당일 80% / 3일 내 30% / 7일 내 20% / 10일 내 10% /
    (공연 10일+ 전 & 예매 10일 지남) 0%. 각 구간 위약금에 최소수수료 하한 적용.
- **UI**: 티켓 팝오버(스케줄 행, `opts.lifecycle`) 헤드에 **시계 아이콘**(`.tk-cc-open`).
  예매일 미저장이면 흐릿한 비활성, 저장되면 골드 활성. 클릭 시 팝오버가 **계산 패널**로 대체
  (`tkLifeForm`과 같은 컴팩트 팝오버 방식, 상태 `tkCancelCalc`):
  예매일(`<input type=date>`) + 예매처·기본·최소 표시 + 오늘 기준 취소료(구간·총액·구성)를
  **실시간 재계산**(예매일 바꾸면 즉시). 돌아가기/저장.
- **저장·연동**: 저장 시 `perf.bookingDate` 기록 → 아이콘 활성화. '취소하기' 폼을 열 때
  예매일이 있으면 취소 수수료 칸을 계산값으로 **자동 프리필**. 티켓 삭제·취소 확정 시 예매일도 초기화.
- **영속성**: `bookingDate`를 `perfSnap`/`applyPerfState`에 추가(localStorage·Dropbox 스냅샷).

## 후속 반영 (작성: Claude)
- `cancelSchedule(perf, bookingDate, asOf)` 추가 — 각 구간의 **마감일**(그 금액으로 취소 가능한 마지막 날)과
  오늘까지 남은 일수·금액을 반환. 마감일 = 청약철회 `min(예매+10, 공연-11)` / 공연10일내 `공연-8` /
  7일내 `공연-4` / 3일내 `공연-1` / 당일 `공연일`.
- 계산 패널 결과를 **헤드라인(오늘 취소 시 구간·금액) + 데드라인 표**로 재구성(`cancelCalcResultHtml`).
  각 행 `날짜(MM/DD) · n일 남음 · 구간(%) · 금액`, **현재 구간 골드 강조**, 지난 마감은 숨김(전부 지났으면 마지막 1행).
  styles.css `.tk-cc-now`·`.tk-cc-dl-list`·`.tk-cc-dl(.cur)`. 캐시버스터 app.js v176 · styles.css v121.

## 후속4 — 실제 예매처 규정 반영, 엔진 전면 교체 (작성: Spinouetter/Claude)
- 놀티켓·신시컴퍼니 실제 취소 규정을 받아 모델을 **예매처별 데이터 정책(cancelPolicy)**으로 교체. 스키마 확정.
- **예매수수료(baseFee)는 예매처 설정으로 이전**(하드코딩 2000 제거). 놀·멜·NOL=2000, 신시=없음.
- 티켓 **첫 유료 선택 시** 예매처가 `feeDefault`면 '수수료' 자동 체크(기존 티켓 수정 시엔 유지).
  baseFee(금액)와 자동체크(feeDefault)는 **별개 속성** — 신시는 baseFee 2000이되 자동 체크는 안 함(놀·멜·NOL만 feeDefault). 예매처별 구성이 조금씩 다름.

### cancelPolicy 스키마(resellers.json, 예매처별)
`deadline{daysBefore,hour,holidayHour?}`(취소 마감·일/공휴일 시각) · `freeOnBookingDay` · `freeBookingDays` ·
`freeRequiresFar`+`farThresholdDays`(무료를 관람 N일전 이전에서만) · `imminentBooking{days,freeDays}`(임박 예매 무료 축소) ·
`tiers[{fromDaysBefore,toDaysBefore, pct | fixed+capPct}]`(관람 근접 위약금). 정책 없으면 기본=놀티켓형.

### 계산식
`d=공연−오늘, b=오늘−예매, base=티켓 '수수료' 체크?예매처 baseFee:0`.
①마감 지남→취소 불가 / ②예매당일(b≤0) or (b≤freeDays & (!freeRequiresFar or d≥far))→무료 /
③tier(from≥d≥to): pct면 티켓×pct%, fixed면 min(fixed, 티켓×capPct%). **총 취소료 = 위약금 + base**.

### 놀티켓/멜론/NOL(기본)
마감 전날17시 · 예매7일 무료(관람10일전 이전) · 예매8일후&관람10일전: min(4000,10%) · 9~7일전10% · 6~3일전20% · 2~1일전30% · 당일 마감.
### 신시컴퍼니
마감 전날17시(일/공휴일 11시) · 예매당일·예매7일 무료 · 그 후 정률 10% · **임박 예매(공연10일내) → 예매 익일부터 10%**.

- 코드: `resellerObjOf`(좌석→예매처 캐시)·`resellerBaseFee`·`resellerCancelPolicy`·`DEFAULT_CANCEL_POLICY`·`cancelDeadlineInfo`/`isPastCancelDeadline`.
  `ticketPriceOf`·가격요약·취소계산 모두 baseFee 사용. computeCancelFee/cancelSchedule 정책기반 재작성(마감=`closed`). 헤드리스로 두 예매처 전 구간·자동수수료 검증. app.js v182·styles.css v127.

## 결정 사항 (작성: Claude)

- **정책을 base/min 두 값으로 모델링** — 사용자 정정("+4000은 예매처별 최소 수수료")을 반영해
  위약금을 `max(티켓×%, 최소수수료)`로 계산. 원 규칙의 "수수료+4000"·"수수료+티켓%"이
  모두 하나의 공식으로 일관되게 표현됨(예매 10일 후=티켓 0%라 위약금=최소수수료).
- **티켓 % 기준가 = 선택된 티켓 종류의 가격**(수수료·기타 제외). 할인권을 선택했으면 그 가격 기준.
  → **확인 필요**: 정가 기준으로 할지 사용자 확인 대기.
- **예매처 수수료 값은 예시(기본 2,000 · 최소 4,000)로 넣음** — 실제 예매처별 값은 사용자 확인 후 조정.
  (놀티켓·신시컴퍼니·멜론티켓·NOL티켓)
- **계산 패널은 티켓 종류가 선택돼 있어야 계산** — 티켓가를 알아야 %계산이 가능. 미선택 시 안내 문구.
- **과거 공연**(오늘>공연일)은 편의상 '공연 당일(80%)'로 처리 — 정상 사용(예정 공연 취소)에는 영향 없음.
