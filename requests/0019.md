# 요청 #0019 — 막올림별 localStorage 네임스페이스 (meta.id)

---

## 요청 (작성: Spinouetter)

하나의 도메인에서 여러 막올림을 돌릴 수 있도록 로컬스토리지 변경 필요함
meta.json의 id로 구분 ID는 영소문자숫자_로 구성됨.

json 내보내기할 때 id도 같이 내보냄. 다만 이 경우 id가 다른데 불러올거냐는 질의하에 진행함

---

## 정리 (작성: Claude)
- 한 도메인에서 여러 막올림 운용 → `meta.json`의 `id`(영소문자·숫자·_)로 localStorage 네임스페이스 분리.
- 설정 내보내기에 `id` 포함, 가져오기 시 id가 다르면 확인(confirm) 후 진행.

## 할 일 (작성: Claude)
- [x] `meta.json`에 `id: "betm-skorea-2026"` 추가
- [x] localStorage 키를 `makollim:state:v1:<id>`로 네임스페이스 (`setupStorageNamespace`, 이름 musicalTracker→makollim)
- [x] id 허용 문자에 `-`(하이픈) 추가, 내보내기 파일명도 makollim-settings/seats.json으로
- (구버전 마이그레이션 기능은 미배포라 불필요 → 추가했다가 제거)
- [x] 설정 내보내기 JSON에 `id` 포함 (`{id, ...snapshot}`)
- [x] 가져오기 시 `state.id !== APP_ID`면 confirm 후 진행
- [x] id 정규화(`sanitizeAppId`: `[a-z0-9_-]`만, 없으면 `default`)

## 결정 사항 (작성: Claude)
- 좌석 JSON(`{sid:[...]}`)은 평면 스키마라 id를 넣으면 깨지므로 **설정 내보내기/가져오기**에만 id 적용.
- 기존 사용자 데이터 보존을 위해 네임스페이스 없던 레거시 키를 현재 id로 자동 이전(이전 후 레거시 삭제).
- localStorage는 키로 이미 막올림이 구분되므로 스냅샷 내부엔 id를 저장하지 않음(내보내기 시에만 부착).
