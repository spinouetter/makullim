# 요청 #0047 — 공연 취소·일정 변경 기록/표시

## 요청 (작성: Spinouetter)

공연에 취소 또는 날짜 변경 기능을 넣어줘.

* 이 정보는 json에 기록되고 스케줄에 표시될거야.
* 취소 공연은 스케줄 창에 날짜와 시간 좌석 번호 까지 흐린 글씨에 취소선을 넣어줘
* 스케줄이 변경된 공연에 대한 표시는 다만, 내부적으로 통합 관리하기 위해서 미리 넣어두는 거야. 스케줄은 여러번 바뀔 수 있으니 리스트에 저장해 두는게 좋을거야.
* 위 정보는 취소되거나 바뀐 공연에만 들어갈거야.

---

## 정리 (작성: Claude)

- 공연 정의 데이터(`shows/<id>/schedule.json`)의 **개별 공연 항목**에 취소·일정변경 정보를 선택적으로 기록하고, 스케줄 화면에 표시한다.
- **취소**: 해당 공연의 **날짜·시간·좌석번호**를 흐린 글씨 + 취소선으로 표시하고, 옆에 작은 `취소` 배지를 붙인다.
- **일정 변경(리스케줄)**: 스케줄은 여러 번 바뀔 수 있으므로 **이전 일정을 리스트로 보관**한다. 현재 일정은 항목의 `date`/`time`, 지나간 일정은 `reschedules` 리스트(오래된 순)에 쌓는다. 스케줄 화면에는 날짜 옆에 변경 아이콘(↻)을 붙이고, 마우스 툴팁으로 변경 이력을 보여준다(우선은 내부 통합 관리를 위한 최소 표시).
- 두 필드는 **취소·변경된 공연에만** 넣는다(일반 공연 항목엔 추가하지 않으며, 없으면 기존과 동일하게 렌더).

### 데이터 형식 (schedule.json 항목에 선택적으로 추가)
```jsonc
{
  "date": "2026-05-01", "time": "19:00", "seat": "", ...,
  "cancelled": true,                 // 취소된 공연
  "reschedules": [                   // 지난 일정 이력(오래된 순), 각 {date,time}
    { "date": "2026-04-20", "time": "19:00" },
    { "date": "2026-04-25", "time": "14:00" }
  ]
}
```

## 결정 사항 (작성: Claude)

- **데이터 주도**: 취소·변경은 공연 정의(schedule.json)의 필드로만 표현하며, 없으면 렌더가 기존과 동일하다. 코드에 특정 공연을 하드코딩하지 않는다.
- **일정 변경은 리스트로 보관**(`reschedules`): 여러 번 변경돼도 이력이 남도록. 현재 일정은 항목의 `date`/`time`가 항상 최신, 과거 일정만 리스트에 쌓는다.
- **취소선 범위**: 날짜 텍스트·시간·좌석 입력값에만 적용. `취소` 배지·리스케줄 아이콘·좌석 눈 버튼 등은 제외(가독성).
- **localStorage와 무관**: 취소·변경은 공용 공연 데이터라 매번 schedule.json에서 로드되며, 사용자 백업(좌석/티켓/메모)에는 포함되지 않는다.
- 리스케줄 표시는 요청대로 **우선 최소(아이콘+툴팁)**로 넣어 통합 관리 기반만 마련했다. 이후 표시를 더 키우는 것은 추가 요청 시 반영.
- **통계 제외 범위**: 취소 공연은 좌석 기반 집계(관극·예매·금액·배우별·조합별·좌석맵)에서만 제외한다. 스케줄 회차 카운트(전체·종료·남음, 배역별 표의 전체·종료 열)는 공표된 스케줄을 나타내므로 취소 공연도 포함해 그대로 센다.
- **(후속) 취소 행 캐스팅 취소선**: 취소 공연은 날짜·시간·좌석뿐 아니라 캐스팅 배우 이름에도 취소선+흐림을 적용한다(CSS `#scheduleTable tr.perf-cancelled .cast-cell .name`, 변경/미출연 배우 `zero-weight` 표기와 동일한 line-through+opacity).
