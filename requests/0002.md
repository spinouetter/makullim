# 요청 #0002 — 스케줄 가로 스크롤 시 날짜 식별 (3옵션)

상태: 구현 완료 (검증됨, 커밋 대기) <!-- 대기 / 진행 중 / 완료 / 보류 -->

> 0001의 묶음 E를 분리해 진행. 앞선 모바일 UI 개선(A~D)은 0001에서 완료, 브랜치 `mobile-ui-improvements`에 커밋됨.

---

## 요청 (작성: Spinouetter)

스케줄을 가로로 스크롤하면 배우 컬럼만 보여서 어떤 회차(날짜)인지 알기 어렵다.
아래 세 방법을 **옵션(설정)에서 켜고 끌 수 있게** 해줘.

- **방법1 (플로팅 요약)**: 셀을 왼쪽으로 가로 스크롤하면 날짜·시간·좌석을 collapse해
  플로팅으로 계속 보여줌. 예) `06/04 19:30 1-2-3`
- **방법2 (행 하이라이트)**: 날짜 셀을 클릭하면 그 행이 하이라이팅되어, 스크롤해도 빠르게 구분 가능.
- **방법3 (가로 스크롤 잠금)**: 셀을 가로로 드래그 스크롤하면 세로 스크롤을 막음.

---

## 정리 (작성: Claude)

세 방법 모두 **신규 기능**이며, 공통적으로 (1) 동작 구현 + (2) Settings에 on/off 토글 + (3) 상태 저장이 필요.

- **방법1 — 플로팅 날짜 요약**
  - 트리거: `#page-schedule .table-scroll-wrap`의 가로 스크롤(`scrollLeft > 0`)로 날짜/시간 컬럼이 가려질 때.
  - 표시: 스크롤 영역 좌상단(헤더 아래)에 고정 라벨로 `날짜 시간 좌석`을 collapse해 표시.
  - 어느 회차를 보여줄지: 현재 화면 상단에 보이는 행 기준(또는 첫 가시 행).
- **방법2 — 행 하이라이트(클릭 고정)**
  - 날짜 셀 클릭 → 해당 `<tr>`에 하이라이트 클래스 토글. 가로/세로 스크롤해도 유지.
  - Spinouetter 결정(0001): **"새로고침 후 유지" 여부도 Settings 옵션으로 추가**(켜면 저장, 끄면 세션 한정).
- **방법3 — 가로 드래그 중 세로 스크롤 잠금**
  - 포인터 드래그가 가로 우세(|dx|>|dy|)면 세로 스크롤 차단(`touch-action`/포인터 핸들링).

## 결정 사항 (작성: Claude)

- 세 방법은 **독립 토글**(상호 배타 아님), **기본값 모두 OFF**로 기존 동작 유지.
- 방법1의 "왼쪽으로 스크롤" = 가로 스크롤로 날짜 컬럼이 가려지는 상황으로 해석.
- 구현/검증은 헤드리스 브라우저(`scratchpad/shot.py` 등)로 확인 후 보고.

### 확인 완료 (Spinouetter 답변)

1. **한 번에 세 방법 다** 구현.
2. 방법1 플로팅은 **모든 줄에** 표시 → 왼쪽 고정(sticky) 컬럼으로 회차마다 `날짜 시간 · 좌석` 표시.
3. 방법2 하이라이트는 **여러 행 동시 가능** + 옵션에 **하이라이트 해제 버튼** 추가.
- (그리고: main에 머지하고 main에서 작업 — 완료.)

### 진행 현황 — 구현 완료 (`?v=13`, main)

- **방법1 (플로팅 고정 컬럼):** `floatDateOn` 켜면 스케줄 맨 왼쪽에 `position:sticky; left:0` 컬럼("일정") 추가. 회차별 `MM/DD HH:MM · 좌석` collapse 표시. 가로 스크롤해도 모든 줄에 고정. (헤드리스: 300px 스크롤에도 offset 0으로 고정, display table-cell 확인.)
- **방법2 (행 하이라이트):** `rowHighlightOn` 켜면 날짜/플로팅 셀 클릭으로 그 줄 토글. **여러 행 동시**. 색은 파랑 계열로 현재공연(금색)과 구분. `하이라이트 모두 해제` 버튼 + `새로고침 후 유지` 저장 옵션. (헤드리스: 클릭→`row-highlight` 적용, 2행 동시 캡처 확인.)
- **방법3 (가로 드래그 중 세로 잠금):** `lockVScrollOn` 켜면 `table-scroll-wrap`의 touch 제스처가 가로 우세(|dx|>|dy|)일 때 `preventDefault`로 세로 차단 + 가로 직접 이동. (터치 제스처라 헤드리스 자동검증 불가 — 실기기 확인 필요.)
- **Settings UI:** "스케줄 보기 옵션" 섹션 신설(체크박스 3 + 저장옵션 + 해제버튼). 헤드리스로 토글·렌더 확인.
- **상태 저장:** `buildStateSnapshot`/`applyState`에 4개 토글 + `highlightedRows`(저장옵션 ON일 때만 보존) 추가. 헤드리스로 localStorage 저장값 확인.

### 재설계 (Spinouetter 피드백 반영) — `?v=15`

**방법1 — 컬럼 → 오버레이로 재구현**
- 더 이상 새 컬럼을 추가하지 않음. `float-cell`을 **0폭 sticky 셀**로 두고 그 안의 `.float-label`을 **절대배치 오버레이**로 띄움.
- 가로로 **년도 폭(날짜셀 첫 3글자 "YY/" 실측, ~22px)**만큼 스크롤되면 `.h-scrolled` 클래스가 붙어 오버레이 표시, 계속 유지. 그 전(스크롤 0 근처)에는 숨김.
- 라벨 포맷: 년도 제거(MM/DD) + 분 제거(시만) + 좌석. 예 `06/28 14 · 1-2-3`. 요일 색상 유지.
- 헤드리스 검증: 스크롤 0 → display:none, 임계값+10 → display:flex·왼쪽 고정(offset 0), 텍스트 `04/12 19`.

**방법2 — 해제버튼 이동 + 다중선택 + 색 강화**
- "하이라이트 해제" 버튼을 **설정 → 스케줄 툴바('캐스트 변경 기록' 옆)**로 이동. 하이라이트 모드일 때만 표시. (설정에는 안내 문구만)
- 다중 선택: `highlightedRows`(Set) 기반 토글 — 헤드리스로 3행 동시 하이라이트 확인.
- 색 대비 강화: 배경 `rgba(74,128,210,0.45)` + 왼쪽 바 `#5b9bff`(date-cell에 직접 걸어 플로팅 켜져도 보임). 금색 현재공연과 명확히 구분.

**방법3** — Spinouetter 확인: 잘 동작함(변경 없음).

### 추가 피드백 반영 — `?v=16`
- **방법1 오버레이:** 좌석번호 제거. 대신 **티켓 아이콘(🎟)** 표시 — 티켓 선택됨(`ticketType` 있음)이면 진하게(opacity 1), 없으면 옅게(0.25). 라벨 예: `06/27 14 🎟`.
- **방법2 해제버튼:** 하이라이트 1개 이상이면 **활성**(`disabled` 해제 + `.active` 강조), 누르면 전체 해제. 0개면 **비활성**(`disabled`, 눌러도 반응 없음). 날짜 클릭/해제 시 즉시 상태 갱신.
- **설정 화면**: "해제 버튼이 나타난다"는 안내 문구 삭제. 방법1 설명도 '오버레이'로 수정.
- 헤드리스 검증: 버튼 0선택→disabled / 1선택→active+enabled / 해제→disabled. 아이콘 티켓행 opacity 1 vs 비티켓행 0.25. 라벨 `06/27 14🎟`.

### 추가 피드백 2 — `?v=17`
- **방법2: Spinouetter 확인 = 구현 완료.**
- **방법1 티켓 아이콘 항상 옅음 수정:** 판정 기준을 `ticketType`(종류 선택) → **`grade`(좌석이 등급에 매칭=티켓 있음)**로 변경. 보통 좌석만 입력하므로 이전 기준은 전부 옅게 나왔음. 이제 좌석 입력만으로 진하게.
- **아이콘 크기:** 티켓 열 `.tk-icon`과 동일하게 11→12px.
- 헤드리스 검증: 좌석만 입력→on(opacity 1), 미입력→0.25, 크기 12px==티켓열 12px.
