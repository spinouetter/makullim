## 요청 (작성: Spinouetter)

통계 페이지 및 시트 맵 업데이트 조건

아래의 경우에 업데이트가 되어야 해.
- 좌석을 입력했을 때
- 티켓을 변경하였을 때 (추가, 삭제, 변경 모두)
- 공연이 끝났을 때. - 공연 시간 + runnningtime

---

## 정리 (작성: Claude)

통계 페이지·시트맵이 다음 3가지 변경에 즉시 갱신되어야 함:
1. 좌석 입력 — **이미 처리됨** (seat-input change 핸들러가 renderStats·renderSeatMap 호출, [app.js:621-622](js/app.js#L621-L622)).
2. 티켓 변경(추가·삭제·변경) — **누락**. 티켓 저장([tk-save, app.js:523](js/app.js#L523))·해제([tk-clear, app.js:536](js/app.js#L536))가 `renderSchedule()`만 호출하고 통계·시트맵은 갱신 안 함.
3. 공연 종료(시작+runningtime 경과) — **이미 처리됨** (30초 interval이 isEnded 수 변화 감지 시 renderStats·renderSeatMap, [app.js:2562-2563 부근](js/app.js)).
- 좌석 JSON 일괄 적용(applySeatJSONData)·설정 가져오기도 이미 둘 다 호출함.

→ 실제 필요한 변경은 **티켓 저장/해제 핸들러에 renderStats()·renderSeatMap() 추가**.

## 할 일 (작성: Claude)

- [x] tk-save 핸들러에 renderStats()·renderSeatMap() 추가
- [x] tk-clear 핸들러에 renderStats()·renderSeatMap() 추가
- [x] 헤드리스로 티켓 변경 시 통계(티켓 금액) 갱신 검증 — 정가 선택·저장 시 통계 `전체 0원→170,000원` (페이지 이동 없이) 확인 (`?v=18`)

## 결정 사항 (작성: Claude)

- 좌석 입력 핸들러와 동일 패턴(`renderSchedule(); renderStats(); renderSeatMap(); saveState();`)으로 통일.
- 티켓 변경은 통계의 '티켓 금액'에 직접 영향. 시트맵은 직접적 영향은 적지만 요청대로 함께 갱신(무해, 일관성).
