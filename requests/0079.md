# 요청 #0079 — 배역 쇼트 네임(shortName) 적용 범위 정리

상태: 완료 <!-- 대기 / 진행 중 / 완료 / 보류 -->

---

## 요청 (작성: Spinouetter)

레이블이 쇼트 네임으로 쓰이는 것은 테이블의 컬럼명만. 그 이외의 영역에 쇼트 네임이 쓰이는 곳있는지 조사해줘

→ (조사 결과에 이어) 좌석지도 "이 좌석에서 본 공연"과 통계 "배역 페어 출연 횟수"도 쇼트 네임, 테이블 컬럼에 대한 토글 스위치(설정의 스케줄 컬럼 표시, 페어막 배우 선택 등)는 풀네임으로.

---

## 정리 (작성: Claude)

`shortName`(casts.json, 배역별 선택 필드)은 원래 `scheduleRoleLabel(role)` 한 함수를 통해서만 읽혔고, 그 함수는 스케줄 탭 자체의 표 `<th>`(1124행)와 Settings의 배역 컬럼 토글 버튼(6196행) 두 곳에서만 호출되고 있었다. 조사해보니 같은 "배역별 `<th>` 컬럼 헤더" 성격인데도 쇼트 네임을 안 쓰던 곳이 두 군데 더 있었다:
- 좌석 지도 → 좌석 상세 "이 좌석에서 본 공연" 표
- Statistics → 배역 페어 출연 횟수(커스텀 조합) 표

사용자 판단: **실제 표의 컬럼 헤더는 쇼트 네임, 배역을 고르는 토글/피커류는 풀네임**으로 원칙을 정리 — Settings 배역 컬럼 토글도 사실 "표 컬럼 헤더"가 아니라 "그 컬럼을 켜고 끄는 선택 버튼"이므로 풀네임으로 되돌림.

## 할 일 (작성: Claude)
- [x] 좌석 지도 좌석 상세 "이 좌석에서 본 공연" 표 헤더: `r` → `scheduleRoleLabel(r)`(+ escHtml 누락도 함께 보완).
- [x] Statistics 배역 페어 커스텀 조합 표 헤더: `escHtml(r)` → `escHtml(scheduleRoleLabel(r))`.
- [x] Settings "스케줄 컬럼 표시"의 배역 토글 버튼: `scheduleRoleLabel(role)` → `role`(풀네임으로 되돌림).
- [x] 캐시버스터 app.js?v=203.
- [x] 헤드리스로 4곳 모두 확인: 스케줄 표(쇼트 네임 유지) · Settings 토글(풀네임으로 변경 확인) · 좌석 상세 표(쇼트 네임 적용 확인) · 배역 페어 커스텀 표(쇼트 네임 적용 확인). "배역 통계 추가"·"페어막 배역 선택" 칩 피커는 원래부터 풀네임이라 변경 없음(일관성 확인).

## 결정 사항 (작성: Claude)
- 원칙: **실제 데이터가 나열되는 표의 컬럼 헤더**(스케줄 표, 좌석 상세 표, 배역 페어 통계 표)는 공간이 좁아 쇼트 네임을, **배역을 고르는 토글/칩 피커류**(Settings 컬럼 토글, 배역 통계 추가, 페어막 배역 선택)는 어떤 배역인지 명확히 알아야 하므로 풀네임을 쓴다.
