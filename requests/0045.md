## 정리 (작성: Claude)

Statistics 탭의 **배역 조합 출연 횟수** 섹션 명칭을 **배역 페어 출연 횟수**로 변경.
사용자에게 보이는 "조합" 문구만 "페어"로 교체하고, 내부 코드 식별자(`comboBlocks`,
`buildComboBody` 등)와 주석은 그대로 둠(동작 변경 없음, 순수 라벨 변경).

## 할 일 (작성: Claude)

- [x] index.html — 섹션 제목 `배역 조합 출연 횟수` → `배역 페어 출연 횟수`
- [x] index.html — 빈 상태 안내 `아직 생성된 조합이…` → `아직 생성된 페어가…`
- [x] index.html — 설정 자동저장 설명의 `배역 조합` → `배역 페어`
- [x] app.js — 중복 생성 토스트 `조합이 이미 존재합니다` → `페어가 이미 존재합니다`
- [x] index.html — app.js 캐시버스터 v109 → v110
- [x] 검증 — `node -c` 문법 확인, 사용자 노출 "조합" 잔여 0건(내부 주석만 남김)

## 결정 사항 (작성: Claude)

- 내부 식별자·주석의 `combo`/`조합`은 유지(리팩터링 아님, 화면 라벨만 변경).
- 참고: 이 기능은 2개 이상(3개+) 배역 조합도 지원하지만 요청대로 라벨을 "페어"로 통일.
  향후 2개로 제한하거나 내부 명칭까지 바꾸려면 별도 요청 필요.
