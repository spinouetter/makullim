# 요청 작업 방식 (requests/)

이 폴더는 **Spinouetter → Claude** 작업 지시를 파일로 주고받는 공간입니다.
대화창에 길게 적는 대신, 요청을 파일로 정리해 두면 Claude가 읽고 정리해서 진행합니다.

## 흐름

1. **요청 작성 (Spinouetter)** — `TEMPLATE.md`를 복사해 새 번호 파일(`0002.md`, `0003.md`, …)을 만들고
   맨 위 **"## 요청 (작성: Spinouetter)"** 칸에 하고 싶은 일을 자유롭게 적습니다. 간단해도 됩니다.
2. **정리 & 결정 (Claude)** — Claude가 그 파일을 읽고, 같은 파일의 아래 칸들을 채웁니다:
   - **정리** — 요청을 Claude가 이해한 대로 다시 정리
   - **할 일** — 실제로 필요한 작업 항목(체크리스트)
   - **결정 사항** — Claude가 내린 판단·선택과 이유 (진행하며 계속 업데이트)
   - **상태** — `대기 / 진행 중 / 완료 / 보류`
3. 추가 요청이나 피드백은 같은 파일 요청 칸에 덧붙이거나, 새 번호 파일로 만듭니다.
4. **완료 시 커밋 확인 (Claude)** — 하나의 작업(요청)이 완료되면 Claude는 **항상 커밋할지 Spinouetter에게 물어봅니다.**
   Spinouetter가 동의하면 해당 요청 문서와 함께 커밋합니다. (요청하지 않는 한 push는 하지 않습니다.)

## 규칙

- **요청 칸은 Spinouetter만, 나머지 칸은 Claude가** 채웁니다 (섞이지 않게).
- 파일명은 4자리 번호 순서대로: `0001.md`, `0002.md` …
- "이 요청 좀 봐줘" / "requests 확인해줘" 라고만 해도 Claude가 새 파일이나 변경된 파일을 찾아 처리합니다.
- 한 요청 = 한 파일을 권장하지만, 관련된 후속 작업은 같은 파일에 이어 적어도 됩니다.
- **작업 완료 시 Claude는 자동으로 커밋하지 않고, 먼저 "커밋할까요?"라고 묻습니다.**
