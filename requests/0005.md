# 요청 #0005 — 좌석번호 크기·관극횟수 구분·시트맵 뷰 저장

상태: 완료 <!-- 대기 / 진행 중 / 완료 / 보류 -->

---

## 요청 (작성: Spinouetter)

- 좌석 번호 크기 조절
  - 현재 시트의 크기를 변경하지 않은 상태에서 좌석 번호의 크기만 키움
  - 관극을 본 횟수가 표시되는데, 1~10사이의 관극은 구분될 수 있도록 필요함
    - 단순한 색상의 그라데이션은 이를 잘 표현하기 어려운 것 같음.
    - 1과 2의 구분이 9-10의 구분 보다 더 명확해야 함

- 시트 맵 위치와 배율 저장. (새로고침 시 이전값 유지)

---

## 정리 (작성: Claude)

세 가지 작업:
1. **좌석번호 글자 키우기** — 셀 크기(`SEAT_SIZE` 0.78)는 그대로, 번호 텍스트 `font-size:0.32`(SVG단위)만 키움 ([buildSeatSvgInner numberLabel, app.js:1150](js/app.js#L1150)).
2. **관극 횟수 1~10 구분(비선형)** — 현재 `seatVisualStyle`는 선형 opacity(`0.55 + count/10*0.45`). 낮은 횟수일수록 변화 폭이 크도록 **오목 곡선(sqrt)**으로 변경 → 1↔2 차이 > 9↔10 차이. (정확한 숫자는 코너 배지로 계속 표시.)
3. **시트맵 위치·배율 저장** — 줌/팬 상태인 `mainViewBox`를 상태에 저장하고 새로고침 시 복원. 저장 지점은 모든 변경의 단일 적용 함수 `applyMainViewBox()`(디바운스).

## 할 일 (작성: Claude)

- [x] 좌석번호 `font-size` 0.32 → 0.48 (weight 600→700) (`?v=19`)
- [x] ~~`seatVisualStyle` 비선형(sqrt)~~ → **재작업**: opacity 그라데이션으로는 1·2 구분 불가(Spinouetter 피드백). **관극 횟수를 좌석 채움 색(이산 히트맵)**으로 변경.
- [x] `mainViewBox` 저장/복원: snapshot/applyState에 `seatViewBox`, renderSeatMap 복원 분기(`pendingSeatViewBox`), applyMainViewBox에 디바운스(250ms) saveState
- [x] 헤드리스 검증: 폰트 0.48 / opacity Δ(1→2)0.072>Δ(9→10)0.028 / 줌·팬 후 새로고침 복원=저장값. 확대 스크린샷에 번호 가독 확인

## 결정 사항 (작성: Claude)

- ~~opacity 곡선~~ → ~~좌석 채움 히트맵+등급 테두리~~ → **최종(재재작업, `?v=21`)**: 좌석 채움은 **등급색만(테두리 없음)**, 관극 횟수 색은 **숫자가 적힌 배지 동그라미 '안'**에 칠함(`countHeatColor`, 1 파랑→…→10 자홍). 숫자는 흰 글자(외곽선으로 가독). 범례의 관극 횟수도 **숫자 동그라미**(`.cnt-dot`)로 표시.
  - 헤드리스 검증: 좌석 rectFill=등급색(모두 VIP 골드), 배지 circle fill=횟수 색(1 파랑/2 청록/5 노랑/10 자홍). 스크린샷에서 배지·범례 동그라미 확인.
  - **추가 미세조정(Spinouetter, `?v=22`):** ① 본 적 없는(흐린) 좌석은 번호 외곽선(halo) 제거(관극 좌석만 halo). ② 범례 관극횟수 원을 등급 사각형과 같은 12px로 축소. ③ 숫자를 원 정중앙 정렬. ④ 범례 원 간격 12→6px(절반). 스크린샷 확인.
  - **범례 레이아웃 미세조정(Spinouetter, `?v=23`):** 등급 레이블 위로(legend margin-top 14→6px), 등급↔관극횟수 줄 간격 좁힘(row-gap 8→3px), 관극횟수 원 1.2배(12→14px, font 8px).
- 뷰 저장은 `applyMainViewBox()`에 디바운스(250ms) saveState를 넣어 드래그·핀치·휠·버튼·미니맵 모든 경로를 한 곳에서 커버.
- 복원은 첫 렌더에서만(`pendingSeatViewBox`) 적용하고, 이후 preserveZoom/초기뷰 로직은 그대로.
