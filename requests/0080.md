# 요청 #0080 — 대결 컬럼 숨김도 Settings로 통합, 스케줄 탭 "숨김 바" 완전 삭제

## 요청 (작성: Spinouetter)

대결의 숨김 설정도 세팅으로 그래서 숨겨도 이제는 드롭박스 안나오게. 스케줄 탭 드롭박스 기능 완전삭제

---

## 정리 (작성: Claude)

요청 0066으로 순번·좌석·티켓·가격·기타비용·메모·배역 컬럼은 전부 Settings → "스케줄 컬럼 표시"에서 전역으로 켜고 끌 수 있게 됐지만, **대결(match) 컬럼만 예외**로 남아 있었다: 대결 컬럼을 숨기면 스케줄 표 상단에 별도의 "숨김 바"(select + "표시" 버튼)가 나타나야만 복원할 수 있었다(Settings에는 대결 토글이 아예 없었음).

이번 요청으로 대결 컬럼도 Settings에서 관리하도록 통합하고, 이제 쓸모없어진 "숨김 바" 기능(HTML 요소·JS 렌더 로직·전용 CSS) 자체를 완전히 삭제했다.

## 할 일 (작성: Claude)
- [x] Settings "스케줄 컬럼 표시"에 "대결" 하위 섹션 추가 — 공연에 대결이 정의된 경우에만 표시(`performanceData.matches` 기반, 배역 토글과 동일한 패턴).
- [x] 대결 컬럼 헤더의 "숨기기" 버튼도 `refreshColumnSettings()`를 호출해 Settings 토글과 즉시 동기화.
- [x] 스케줄 탭의 "숨김 바"(`#scheduleHiddenBar`) 완전 삭제: index.html 요소, js/app.js의 렌더/복원 로직, 전용 CSS(`.hidden-roles-bar` 등) 전부 제거.
- [x] 이제 어디서도 안 쓰는 `colLabel()`/`COL_LABELS` 죽은 코드 함께 제거.
- [x] 캐시버스터 app.js?v=204, styles.css?v=147.
- [x] 헤드리스로 확인: 대결 있는 공연(betm-skorea-2026)에서 Settings 토글로 숨김/복원이 스케줄 표에 정확히 반영, 대결 없는 공연(elisabeth_2026)에서는 "대결" 섹션 자체가 안 보임, `#scheduleHiddenBar` 요소가 DOM에서 완전히 사라짐.
- [x] manual(01-스케줄.md·06-설정.md)에서 "숨김 바" 관련 설명 갱신.

## 결정 사항 (작성: Claude)
- 대결 컬럼도 다른 컬럼들과 완전히 동일한 방식(Settings 전역 토글)으로 통일 — 스케줄 표 자체에는 더 이상 "복원" 기능이 없고, 숨긴 컬럼은 전부 Settings에서만 다시 켠다.
