# 요청 #0074 — Tickets 탭에서 티켓 바로 수정

---

## 요청 (작성: Spinouetter)

Tickets에서 티켓 바로 수정할 수 있게 변경해줘

---

## 정리 (작성: Claude)

- 지금까지 Tickets 카드를 누르면 Schedule 탭으로 이동해 해당 행으로 스크롤한 뒤에야 편집할 수 있었음(맨 위 티켓=Schedule 팝오버, 추가 좌석=관리창 열기만 하고 티켓 선택은 다시 눌러야 함, 취소·양도 이력=이동만 하고 편집 UI 없음).
- 세 종류 모두 **탭 이동 없이 Tickets 안에서 곧바로 편집**할 수 있게 바꾼다. 맨 위 티켓은 Tickets 탭 안의 팝오버(신규), 추가 좌석은 다중 티켓 관리창을 그 티켓의 편집 상태로 바로 열기, 취소·양도 이력은 이력 창을 그 기록의 편집 상태로 바로 열기.

## 할 일 (작성: Claude)
- [x] 상태 `ticketsEditIdx`(-1=닫힘) 추가. 다른 탭으로 이동하면 자동으로 닫힘(`activateTab`).
- [x] `renderTickets()`: `ticketsEditIdx>=0`이면 `buildTicketPopover(idx, grade, topTicketObj(p), {bookingField:true, showAddTicket:!multiTicketMode})`를 오버레이(`.tix-pop-overlay`, 기존 `.tm-pop-overlay` 스타일 재사용)로 표시. 좌석 필드는 없음(맨 위 티켓의 좌석은 그 공연에 고정) — 대신 예매일을 팝오버 안에서 바로 입력(취소료 계산 패널을 거치지 않고 즉시 편집).
- [x] `wireTicketsEditPopover()`: 관리창 팝오버와 동일한 최소 배선(옵션 변경 시 가격 요약 갱신, 저장/취소/삭제, ＋중복 티켓 관리). 저장 시 `renderTickets/renderSchedule/renderStats/renderSeatMap/saveState` 모두 갱신(다른 화면과 데이터 공유).
- [x] 이력(`ticketHistory`) 항목에 배열 인덱스(`hi`) 추가해 카드에 `data-hi`로 노출.
- [x] 카드 클릭 핸들러 재작성: 맨 위 티켓=`ticketsEditIdx` 토글, 추가 좌석=`openTicketManager` 연 뒤 좌석으로 찾은 티켓의 `tmEditTi`를 바로 세팅해 편집 팝오버 표시, 취소·양도=`openTicketHistory` 연 뒤 `thEditHi=hi`로 바로 편집 팝오버 표시. 더 이상 Schedule 탭으로 이동하지 않음.
- [x] 캐시버스터 app.js v194.

## 결정 사항 (작성: Claude)
- 맨 위 티켓 편집 팝오버는 **취소/양도/숨김/이력 같은 라이프사이클 액션은 포함하지 않음**(Schedule의 거대한 배선 로직을 그대로 복제하는 대신, 이미 0073에서 검증된 "관리창 팝오버" 수준의 가벼운 배선을 재사용). 그런 액션이 필요하면 기존처럼 Schedule 탭이나 이력 창에서 계속 가능— Tickets 탭 카드에서는 "가격/예매일 등 바로 고치기"에 집중.
- 예매일 입력은 팝오버 안에 직접 넣어(0073에서 만든 seatField 없는 `bookingField` 단독 줄 재사용) 별도 "취소료 계산" 패널·시계 아이콘 흐름을 거치지 않게 함 — 클릭 한 번으로 예매일까지 바로 편집.
- 추가 좌석·이력 편집은 이미 있는 관리창/이력창을 그대로 쓰되, 여는 즉시 해당 항목의 편집 팝오버를 띄워 클릭 한 번으로 편집에 도달하게 함(이전엔 창을 연 뒤 항목을 한 번 더 눌러야 했음).
