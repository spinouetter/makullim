---

## 요청 (작성: Spinouetter)

설정 탭에서 저장 초기화 기능 분화
  - 개발 중인 기능에 의해 localstorage에 저장된 값이 비정상 동작을 만들 수 있음
  - 현재는 이를 초기화하면 좌석 정보까지 다 지워짐

새로운 버튼
  - 설정 초기화
  - 좌석 데이터 삭제

---

## 정리 (작성: Claude)

기존 "모든 저장 데이터 삭제"(localStorage 전체 삭제 → 좌석까지 사라짐)를 두 버튼으로 분리:
- **설정 초기화**: 화면 설정·필터·테마·통계 구성 등만 초기화, **좌석·티켓·메모는 유지**. (개발 중 기능으로 저장값이 꼬였을 때 안전하게)
- **좌석 데이터 삭제**: 입력한 **좌석·티켓·메모만** 삭제, 설정은 유지.

## 할 일 (작성: Claude)
- [x] 설정 탭 버튼 2개로 분리(설정 초기화 / 좌석 데이터 삭제) + 설명 (`?v=33`)
- [x] 설정 초기화: `{performances}`만 남기고 저장 후 reload(나머지 설정 기본값)
- [x] 좌석 데이터 삭제: 공연별 seat/ticket/note/discount/extra 비우고 저장 후 reload
- [x] 헤드리스 검증: 설정초기화→좌석유지·필터/테마 리셋 / 좌석삭제→좌석만 삭제·설정 유지

## 결정 사항 (작성: Claude)
- 매치 결과(p.match)는 서버 제공 데이터라 둘 중 어느 쪽으로도 지우지 않음(좌석 데이터 = seat/ticket/memo).
- 설정 초기화는 현재 좌석값을 buildStateSnapshot에서 가져와 `{performances}`만 저장 후 reload → 좌석 보존, 나머지 기본값.
