## 정리 (작성: Claude)

- 공연 회차 ID(sid)는 지금까지 로드 시 `app.js`가 배열 순번으로 `s1, s2…`를 암묵 부여했다.
  이 값은 좌석 JSON 내보내기/가져오기의 키, `grades.json`의 `sids`/`excludeSids`
  (티켓 적용 회차 지정)에서 쓰이므로, **스케줄 중간에 회차를 추가·삭제하면 뒤 회차들의
  sid가 전부 밀려** 데이터가 어긋날 수 있었다.
- 이제 각 공연 `schedule.json`의 회차마다 `"sid"`를 **명시 필드**(첫 키)로 기록한다.
  값은 기존 암묵 값과 동일(`s{순번}`) — 저장된 좌석 파일·grades 참조와 100% 호환.
- `app.js`는 명시 sid를 우선 사용하고, 없으면 기존처럼 순번으로 부여(구버전 데이터 호환).
  중복 sid는 콘솔 경고로 알린다(데이터 오류 감지).

## 결정 사항 (작성: Claude)

- **값을 바꾸지 않고 명시화만** — sid는 localStorage 좌석 내보내기 파일·grades.json
  참조의 키이므로 기존 값(`s{순번}`)을 그대로 기록. 마이그레이션 불필요.
- 앞으로 **회차를 중간에 추가할 때는 새 sid를 뒤 번호와 겹치지 않게**(예: 마지막
  번호 다음 번호) 부여하면 기존 회차들의 sid가 밀리지 않는다. 순번과 sid는 이제
  독립이다.
- 폴백(순번 부여)은 유지 — sid 없는 회차가 섞여도 동작하되, 명시 sid와 충돌하면
  콘솔 경고가 뜬다.
