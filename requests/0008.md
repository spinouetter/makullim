## 요청 (작성: Spinouetter)

경기 승부 (match) 기능

- 뮤지컬 안에서 배역 간의 경기가 있을 경우 이를 나타낼 수 있는 기능
- json 내용 (키는 영어로)
  - 대결의 이름
  - 대결이 발생하는 지점 (1~10까지 정해준 배역의 비중 기반)
    예를 들어 4이 값이 4였음
    - A날에는 김승주,3 / 김우진,7이었음. 이날은 김우진이 대결에 참여
    - B날에는 김승주,7 / 김우진,3이었음. 이날은 김승주가 대결에 참여
  - 대결에 참여하는 배역
  - 승패가 없는 경우 임의로 승리했다고 간주하는 배역
    - 통계화면의 옵션으로 (잘모를시 배역 승리 간주) 추가

  - 공연정보 json
    - 승패여부를 저장하는 항목이 추가됨. 필드는 배역과 메모가 존재함

  - 스케줄 탭
    - 메모와 배우 사이에 대결의 결과를 표시하는 열 추가. 값이 없는 경우 끝난 공연은 모름. 다음 공연은 미정

  - 통계화면
    - <이름> 대결 <배역> 승리
      배우, 전체 (완료 공연 횟수), 승리, 패배, 무승부, 승율
    - <이름> 대결 <주배역x서브배역> 승리
      주배우, 서브배우, 전체 (둘이 함께 공연 횟수), 주배역 승리 수, 주배역 패배수, 무승부 수, 주배역 승률

## 정리 (작성: Claude)

뮤지컬 안의 "대결(match)"(예: 빌리 엘리어트의 **슈즈** = 빌리 vs 마이클)을 회차별로 기록·통계.

**1) 매치 정의 (JSON, 영어 키)** — 신규 파일 `json/matches.json` 배열 제안:
```json
[{ "name":"슈즈", "point":4, "roles":["빌리","마이클"], "defaultWinner":"빌리" }]
```
- `name` 대결 이름 / `point` 발생 지점(1~10) / `roles` 참여 배역 / `defaultWinner` 승패 미기록 시 승리로 간주할 배역.

**2) 지점 → 참여 배우 결정 (비중 기반)**
- 각 배역의 그날 캐스팅 비중을 **누적**해, `point`를 덮는 배우가 그날 그 배역의 참여자.
- 예) point=4 — `김승주3 / 김우진7` → 1~3 김승주, 4~10 김우진 → **김우진** 참여. `김승주7 / 김우진3` → 1~7 김승주 → **김승주** 참여.
- 단일 배우(비중 10/이름만)면 그 배우가 참여. 그 배역에 캐스팅이 없으면 매치 성립 안 함.

**3) 공연정보에 승패 저장** — 공연별 매치 결과: **승리 배역 + 메모**. (무승부/미기록 표현 포함)

**4) 스케줄 탭** — 메모와 배역(cast) 사이에 "대결 결과" 열 추가. 값 없으면 끝난 공연="모름", 다음 공연="미정". (셀 클릭으로 입력)

**5) 통계화면**
- 배우별: `<슈즈> 대결 <빌리> 승리` — 배우 / 전체(완료 공연) / 승 / 패 / 무 / 승률.
- 조합: `<슈즈> 대결 <빌리×마이클> 승리` — 주배우, 서브배우 / 전체(함께 공연) / 주배역 승 / 패 / 무 / 주배역 승률.
- 옵션: "(잘 모를 시 배역 승리 간주)" 토글 — 켜면 완료공연 미기록을 `defaultWinner` 승리로 간주.

**기본 데이터:** 슈즈 / 지점 4 / 배역 빌리·마이클 / 승리간주 빌리.

## 결정 사항 (작성: Claude) — Spinouetter 답변 반영
- **승패는 서버가 JSON으로 제공. 직접 입력 UI 없음.** 스케줄 결과 열은 **표시 전용**. 지금은 임시 데이터로 채움:
  과거(종료) 공연 일부에 승리 배역/무승부 기록, **일부는 비워 '모름'**, **미래 공연엔 결과 없음('미정')**.
- 승패 저장 위치 = 공연정보(schedule.json) 공연별 필드. 한 필드에 **배역명 또는 "무승부"** + 메모. (무승부 별도 열 없음)
- 매치 정의는 신규 `json/matches.json`(배열, 다중 매치 가능). 키 영어.
- 통계 옵션 "(잘 모를 시 배역 승리 간주)": **OFF면 결과 미기록 완료공연은 전체엔 포함하되 승/패/무 미반영**, ON이면 `defaultWinner` 승리로 간주.
- 승률 = 승 / (승+패+무) (미반영 제외) — 표기 %. (확정 시 조정 가능)
- 매치 결과는 데이터(JSON) 소유라 localStorage 사용자 상태에는 저장 안 함.

### 확정 (Spinouetter)
- **지점→참여 배우 = 누적 구간이 지점을 포함하는 배우.** 이름(캐스팅) 순서대로 비중 누적, point가 속한 구간의 배우. 예: [가2,나3,다5] 지점4 → 가1~2,나3~5,다6~10 → **나**. (단일 배우면 그 배우)
