## 요청 (작성: Spinouetter)

내보내기 기능 수정
- 설정 내보내기
  - 전체 공연 목록은 서버에서 받기 때문에 저장할 필요가 없음
  - 설명에 설정과 좌석이 모두 저장되고 업데이트 된다는 설명 추가 필요. (내부 구조는 설명할 필요 없음)
  - 좌석 정보는 포함함 datetime은 YYMMDD_hh:mm 으로 KST 기준
  - 파일 이름을 makollim-settings-seats-<datetime>.json 으로 변경

- 좌석 내보내기
  - 좌석 정보는 포함함 datetime은 YYMMDD_hh:mm 으로 KST 기준
  - 파일 이름을 makollim-seats-<datetime>.json 으로 변경


- 이 이외에도 musicaltracker 가 남아있으면 makollim으로 변경
  (로컬 스토리지 구조 변경해도 됨)

## 정리 (작성: Claude)
- 설정 내보내기: 전체 공연 목록(index 배열) 대신 좌석 정보를 sid 기준(`seats`)으로 포함. 설명 텍스트 갱신. 파일명 `makollim-settings-seats-YYMMDD_hhmm.json`.
- 좌석 내보내기: 파일명 `makollim-seats-YYMMDD_hhmm.json`.
- datetime = `YYMMDD_hhmm` (KST).
- 코드 내 musicaltracker 잔존 없음(0019에서 정리 완료).

## 결정 사항 (작성: Claude)
- localStorage 스냅샷은 그대로 두고 **내보내기 시에만** performances 제거·seats 부착(서버가 공연 목록 제공 → 좌석만 오버레이). 가져오기는 신/구 포맷 모두 처리.
- 파일명 datetime은 `YYMMDD_hhmm`(콜론 제거 — 파일명 호환성).
- 공통 다운로드 헬퍼 `downloadJSON`으로 정리.
