## 정리 (작성: Claude)

앱은 한 벌 유지, 공연 데이터만 `shows/` 아래로 분리한다.

### 새 구조

```
shows/
  index.json                 # 레지스트리: { "default": "betm-skorea-2026", "shows": [ ...id들 ] }
  betm-skorea-2026.json      # 공연 정의(구 json/meta.json) + "slug": "billy"
  betm-skorea-2026/          # 공연 데이터 폴더 (id와 같은 이름)
    casts.json  schedule.json  grades.json  matches.json  casting_by_date.json
    finale-boards.json  finale-boards/…
    images/                  # 배우 사진 + finale-board.svg (+ CI 생성 finale-preview.png)
data/holidays.json           # 공용(한국 공휴일 — 공연과 무관)
theatres/                    # 공용(변경 없음) — 공연 정의의 theatre가 절대경로로 연결
js/ css/ fonts/ index.html   # 공용 앱(한 벌)
```

### 접속 경로 동작

- **`/billy/`(slug 주소)** — GitHub Action이 배포 시 `index.html`을 복사해 `billy/index.html` 스텁을 생성(커밋 안 함).
  스텁에는 `window.MAKOLLIM_SHOW_ID="betm-skorea-2026"`이 주입되어 앱이 바로 해당 공연을 로드.
- **`/`(루트)** — 앱이 `shows/index.json`의 `default` 공연을 읽고, 배포 도메인에서는 그 공연의 slug 주소로
  `location.replace`. 로컬 개발(localhost/127.0.0.1)에서는 리다이렉트 없이 default 공연을 그대로 렌더.
- **로컬에서 다른 공연 보기** — `/?show=<id|slug>` 쿼리 지원(스텁은 배포에만 존재하므로).

### 경로 해석 규칙

- 공연 파일(공연 정의·finale-boards·보드 def) **안의 상대경로 = 그 공연 폴더(`/shows/<id>/`) 기준**.
- **`/`로 시작하면 사이트 루트 기준** — 공용 자원(`/theatres/…`, `/fonts/…`)은 절대경로로 적는다.
- 페이지가 `/billy/`에서 뜨므로 `index.html`의 css/js 링크와 코드의 공용 fetch는 전부 절대경로로 통일.

## 결정 사항 (작성: Claude)

- **slug ≠ id 분리** — slug는 URL용(언제든 변경 가능), id는 localStorage(`makollim:state:v1:<id>`)·Dropbox
  백업 파일명의 불변 키. **`meta.id`(=파일·폴더명)는 절대 바꾸지 않는다** — 기존 사용자 기록이 그대로 유지됨.
- **스텁은 배포 시 생성** — 저장소에는 slug 디렉토리가 없고, Pages가 Actions 아티팩트 배포라 커밋 없이
  빌드 산출물에만 포함. 새 공연 추가 = `shows/<id>.json` + 데이터 폴더 + 레지스트리 한 줄이면 끝.
- **루트 리다이렉트는 앱(JS)이 수행** — 별도 리다이렉터 파일 없이 부트 로직에서 처리. 로컬 개발·미리보기
  생성기(127.0.0.1)는 리다이렉트를 건너뛰어 기존 개발 방식(`python -m http.server`) 유지.
- **배우 사진은 공연별 폴더** — 전역 공유 풀 대신 `shows/<id>/images/`. 중복 비용은 작고,
  "공연 폴더 하나 = 완결된 데이터 셋"(폴더 복사로 새 공연 세팅)이 유지·이해에 유리.
- **holidays는 공용 `data/`로** — 한국 공휴일은 공연 데이터가 아님.
- **주의(배포 설정)**: Dropbox OAuth redirect URI가 `location.pathname` 기반이라, Dropbox 앱 콘솔에
  `https://<domain>/billy/` 를 redirect URI로 **추가 등록해야** slug 주소에서 백업 연결이 동작한다.
- **`billy-/`(캐스팅 보드 사진 원본)을 .gitignore에 추가** — temp/와 같은 로컬 전용 작업 소스인데
  untracked로 남아 있어 실수 커밋 방지 차원. 추적이 필요하면 이 줄만 빼면 된다.
- 루트→slug 리다이렉트는 쿼리를 보존하지 않는다(`/?board=…` → `/billy/`). 공유 링크는 slug 주소 기준이므로 실사용 영향 없음.
- `casting_by_date.json`(보드 추출 결과, git 추적 중)도 공연 데이터로 보고 `shows/betm-skorea-2026/`로 함께 이동.
