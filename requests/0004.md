# 요청 #0004 — 모바일 시트맵 개선 (좌석 정보 노출)

상태: 완료 <!-- 대기 / 진행 중 / 완료 / 보류 -->

---

## 요청 (작성: Spinouetter)

- 모바일 시트 맵 개선

  - 시트 맵 내부에서는 시트 맵 내부를 움직이도록 드래그가 동작함.
  - 좌석 정보는 시트 맵 페이지를 위로 올려야 볼 수 있는데, 현재 시트 창이 너무 커서 페이지를 드래그 하기가 너무 어려움
  - 동적으로 시트 표시되는 부분을 조정하여 브라우저의 height가 너무 작은 경우 시트 화면의 크기를 줄여서 좌석 정보의 최소한의 부분이 화면에 표시되게 변경

---

## 정리 (작성: Claude)

시트맵(`#svgViewport`)이 `touch-action:none`이라 내부 드래그는 팬/줌으로 동작 → 맵이 화면을 꽉 채우면 **페이지를 스크롤(드래그)할 빈 영역이 없어** 아래의 좌석 정보 패널(`#seatDetail`)에 접근이 어려움.

원인(헤드리스 측정): `.svg-viewport`가 `height:55vh; min-height:300px`. 가시높이가 작아도 **min 300px** 때문에 맵이 안 줄어 좌석정보가 화면 밖으로 밀림.
- 390×900(가시757): 맵 416 → 좌석정보 peek 보임 ✓
- 390×650(가시507): 맵 300(min) → 좌석정보 안 보임 ✗
- 390×560(가시417): 맵 300(min) → 안 보임 ✗
- 비-맵 콘텐츠(층바+컨트롤바+그리드패딩+범례+간격) ≈ 168px.

→ 맵 높이를 **가시높이(`--app-height`) 기반 `clamp()`**로 바꿔 작은 화면에서 자동으로 줄이고, 좌석 정보 일부가 보이도록. `--app-height`는 이미 resize/visualViewport마다 갱신되므로 **동적**으로 반응함.

## 할 일 (작성: Claude)

- [x] `.svg-viewport` 높이를 `clamp(130px, calc(var(--app-height) - var(--header-h) - 240px), 480px)`로 변경 (55vh·min-height 300 제거) (`?v=18`)
- [x] 예약분 240px(비-맵 ~168 + peek ~70), min 130px로 튜닝 → 작은 화면에서도 좌석정보 노출
- [x] 큰 화면 유지: 900px에서 맵 430px(기존 416과 유사), 최대 480px 클램프
- [x] 헤드리스 검증: 560/600/650/900px 모두 좌석정보 패널 보임(seatDetailVisible=true). 650px 스크린샷에 하단 "좌석 정보" 노출 확인

## 결정 사항 (작성: Claude)

- 순수 CSS(clamp + CSS 변수)로 해결 — JS 추가 없이 `--app-height` 변화에 자동 반응(동적).
- 맵 최소 높이는 너무 작아지지 않게 ~150px로 클램프(아주 작은 화면에선 peek가 작아질 수 있음, 요청의 "최소한" 취지에 부합).
- 드래그 충돌(맵 팬 vs 페이지 스크롤)은 맵을 줄여 주변 스크롤 영역(범례 등)을 확보하는 것으로 완화.
